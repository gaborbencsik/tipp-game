# PWA-003: PWA telepítés smart banner + onboarding integráció

## Vezérelv

**A telepítés ott történjen, ahol a user játszik — ne csak az avatar menüben legyen elrejtve.** Egy diszkrét, eldobható banner a Mérkőzések oldalon hozza a legtöbb konverziót; az onboarding utolsó lépésében pedig természetes a felajánlás. Egyik se legyen tolakodó: egyszer látja, eldobja, vége.

## Leírás

Mint **mobilfelhasználó** szeretném, hogy az app maga ajánlja fel a telepítést a megfelelő pillanatban, hogy ne kelljen az avatar menüben turkálnom hozzá.

## Jelenlegi helyzet

- A telepítés tutorial ([[PWA-002]]) a `/app/install` URL-en élesben van, és a UserMenuButton dropdown-ból érhető el.
- Nincs proaktív promó: a user csak akkor találja meg, ha kifejezetten keres rá.
- `installPromptEvent` és `isStandalone` már globálisan elérhető a `src/lib/install-prompt.ts`-ből — erre lehet építeni.

## Hatókör

### Benne van

1. **Smart banner komponens** (`InstallPromoBanner.vue`)
   - Csak akkor jelenik meg, ha **mind** teljesül:
     - `!isStandalone` (még nincs telepítve)
     - mobil viewport (`md:hidden` vagy UA-alapú detektálás)
     - user nem dismissolta korábban (`localStorage` flag, pl. `pwa-install-dismissed`)
     - legalább 1 tippet leadott már (engagement signal — különben nem érti, miért ajánljuk)
   - Tartalom: rövid felirat („Telepítsd appként a főképernyődre"), elsődleges CTA, ✕ dismiss
   - Ha `installPromptEvent` elérhető → CTA gomb azonnal hív `triggerInstall()`-t
   - Ha nincs (iOS) → CTA átnavigál `/app/install`-ra
2. **Banner placement**: `MatchesView.vue` tetején (a fő használati képernyőn), `<DayNavigator />` előtt
3. **Onboarding lépés** (`OnboardingOverlay.vue`)
   - Új lépés a meglévő flow utolsó hub-előtti pozíciójában
   - Cím: „Telepítsd appként", leírás: 1-2 mondat, CTA: „Telepítés most" (ha `installPromptEvent`) vagy „Mutasd a lépéseket" (átnavigál `/app/install`-ra)
   - Skip-elhető — a meglévő onboarding skip mintát követi
4. **Dismiss perzisztálás**:
   - Banner ✕ → `localStorage.setItem('pwa-install-dismissed', '1')` + nem mutatja újra
   - `appinstalled` esemény → automatikus dismiss (nem jelenik meg többet)
   - Onboarding skip nem dismissolja a bannert (két különböző belépés)
5. **i18n**: `install.banner.*` és `install.onboarding.*` kulcsok hu+en
6. **Tesztek**:
   - Unit: `InstallPromoBanner.spec.ts` — feltételes megjelenés (4 feltétel kombinációi), CTA viselkedés (prompt elérhető vs. nem), dismiss perzisztálás
   - Onboarding lépés egy meglévő test bővítményeként, nem új teszt fájl

### Nincs benne

- A/B teszt vagy időzítés (pl. „mutasd 3 nappal az első tipp után") — túl korai
- Push notification a telepítés ösztönzésére
- Telepítési arány tracking (külön analytics story lenne)

## Tervezett UI (banner)

```
┌──────────────────────────────────────────────┐
│  📱 Telepítsd appként a főképernyődre   ✕   │
│     Egy kattintás, és úgy fut, mint egy app. │
│  [ Telepítés most ]                          │
└──────────────────────────────────────────────┘
```

- Háttér: `bg-gradient-to-br from-blue-50 to-white`, `border border-blue-200`
- Szöveg: `text-sm` (cím félkövér), `text-xs text-gray-600` leírás
- Gomb: a meglévő `bg-blue-600 hover:bg-blue-700 text-white` mintát követi
- ✕: jobb felső sarok, `text-gray-400 hover:text-gray-600`

## Tervezett UI (onboarding lépés)

A `OnboardingOverlay.vue` jelenlegi kártyastruktúráját követi (icon + cím + leírás + CTA-k).

## Elfogadási kritériumok

1. ✅ Mobil viewport + nincs telepítve + van leadott tipp + nem dismissolt → banner megjelenik a `MatchesView` tetején
2. ✅ Bármelyik fenti feltétel hamis → banner nem jelenik meg
3. ✅ Banner CTA `installPromptEvent` mellett → rendszer telepítő dialógus
4. ✅ Banner CTA `installPromptEvent` nélkül → router push `/app/install`-ra
5. ✅ Banner ✕ → eltűnik, oldal újratöltés után se jön vissza
6. ✅ `appinstalled` esemény → banner soha többé nem jelenik meg (auto-dismiss)
7. ✅ Onboarding új lépésben CTA működik (prompt + fallback), skip-elhető
8. ✅ Frontend typecheck + unit tesztek zöldek
9. ✅ E2E: meglévő tesztek nem törnek (banner nem zavarja a desktop layoutot)

## Tesztelési stratégia

- Unit teszt mock `installPromptEvent` ref + mock `localStorage` + mock `usePredictionsStore` (engagement)
- Manual smoke test: Chrome DevTools mobile mode + clear localStorage / standalone mode toggle

## Vezérelv (összefoglaló)

> A telepítés-promó ott legyen, ahol természetes — ne tolja, ne ismételje, és tűnjön el azonnal, ha már nincs rá szükség.
