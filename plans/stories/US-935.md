# US-935: Torna szintű statisztikai bonus tippek (5 globális típus seedelése)

> Forrás: Tornaspecifikus bonus tippek bevezetés

## User Story

Mint platform admin, szeretném, hogy a tornához előzetesen **5 globális bonus tipp típus** kerüljön seedelve az adatbázisba, hogy a csoportadminok csoportonként feliratkozhassanak rá, és a tagok a torna alatt tippeljenek ezekre a csapatszintű és játékosszintű statisztikákra.

## Jelenlegi helyzet

A `special_prediction_types` tábla már támogatja:
- `isGlobal: true` és `groupId: null` — globális típusok,
- `inputType: 'team_select' | 'player_select'` — csapatokra és játékosokra tippeléshez,
- `points: smallint` — típusonként konfigurálható pontérték.

A `groupGlobalTypeSubscriptions` tábla már lehetővé teszi, hogy csoportok feliratkozzanak globális típusokra. A frontend `SpecialPredictionsTab.vue` automatikusan megjeleníti a globális típusokat, ha a csoport feliratkozott rájuk.

A 5 új típus **statisztikai bonusz tippek**, melyeket a torna alatt nyomon követünk, majd az admin a torna végén beállítja a helyes választ.

## Elfogadási kritériumok

### Backend

1. **Seed: 5 új globális special prediction type**
   - Az alábbi táblázatban szereplő típusok kerülnek be az `specialPredictionTypes` táblára.
   - `isGlobal: true`, `groupId: NULL`, `isActive: true` — mindegyik.
   - `deadline`: a torna első csoportmeccsének kickoff ideje (YYYY-MM-DD HH:MM:SSZ formátumban, placeholder: `<első csoportmeccs kickoff>`).
   - A seed **idempotens**: legalább kétszer futtatva sem hoz létre duplikátumot.
     - Módszer 1 (migration + ON CONFLICT): Drizzle raw query vagy dedikált migration `INSERT ... ON CONFLICT DO NOTHING`.
     - Módszer 2 (seed script): npm run `seed:tournament-bonus-types` vagy init-based trigger.
     - A választott módszer dokumentálva van a technikai megjegyzésekben.

   | # | Név (HU) | Leírás | inputType | points | Megjegyzés |
   |---|----------|--------|-----------|--------|-----------|
   | 1 | Csoportkör – legtöbb gólt szerző csapat | Melyik csapat szerzett a legtöbb gólt a csoportkörben? | `team_select` | 3 | Felhasználó csapatnevet választ |
   | 2 | Csoportkör – legkevesebb gólt szerző csapat | Melyik csapat szerzett a legkevesebb gólt a csoportkörben? | `team_select` | 3 | Felhasználó csapatnevet választ |
   | 3 | Csoportkör – legtöbb gólt kapó csapat | Melyik csapat kapott a legtöbb gólt a csoportkörben? | `team_select` | 3 | Felhasználó csapatnevet választ |
   | 4 | Csoportkör – legkevesebb gólt kapó csapat | Melyik csapat kapott a legkevesebb gólt a csoportkörben? | `team_select` | 3 | Felhasználó csapatnevet választ |
   | 5 | Torna gólkirálya | Ki szerzett a legtöbb gólt a teljes tornán? | `player_select` | 5 | Felhasználó játékosnevet választ |

2. **Seed idempotenciája**
   - A seed script/migration legalább kétszer futtatható anélkül, hogy duplikátumok keletkeznének.
   - Az idempotencia megvalósítása: az alábbiak közül válassz:
     - **INSERT ... ON CONFLICT DO NOTHING**: a típus name + isGlobal kombinációs egyedisége alapján (vagy az UUID-n, ha a rekordok már léteznek).
     - **Előzetes SELECT + conditional INSERT**: a seed script ellenőrzi, hogy az 5 típus már létezik-e, és ha igen, nem ír.

3. **Unit teszt (backend)**
   - Teszt: seed script legalább 3 forgatókönyvet lefed:
     1. **Első futtatás**: 5 típus kerül be, `insertedCount === 5`.
     2. **Második futtatás** (ugyanazon DB-re): `insertedCount === 0` vagy ugyanaz az 5 típus marad, duplikátum nincs.
     3. **Típusok attribútumai**: random tipo választás után ellenőriz: `inputType in ['team_select', 'player_select']`, `points in [3, 5]`, `isGlobal === true`, `groupId === null`.
   - Teszt módszere: vitest + db fixture.
   - Teszt fájl: `packages/backend/src/db/seeds/*.test.ts` vagy `packages/backend/src/scripts/*.test.ts`.

4. **Seed futtathatósága**
   - Az npm script definiálva van: pl. `npm run seed:tournament-bonus-types` vagy `npm run migrate:seed` részeként.
   - A seed lokálisan (`docker-compose`) és prod. környezetben egyaránt futtatható.
   - A script naplózza az eredményt (hány típus kerül be, hány már létezik, stb.).

### Frontend

5. **Automatikus feliratkozás vagy manual opt-in (döntés szükséges)**
   - **Lehetőség A**: az US-932 logikája kiterjesztendő — minden új csoport **automatikusan feliratkozik** az 5 új globális típusra a csoport létrehozáskor.
   - **Lehetőség B**: a csoportadmin az admin panelen manuálisan kiválasztja ezeket a típusokat (UI már létezik az `AdminGroupSpecialTypesView.vue`-ben).
   - A story **nyitott kérdésként** hagyja: "Az 5 bonus típusra automatikus feliratkozás legyen-e minden új csoportnál (A), vagy manual opt-in (B)?"
   - A frontend egyéb részen **nincs változás** — ha a csoport feliratkozott, az 5 típus automatikusan jelenik meg a "Torna tippek" tabon.

## Technikai megjegyzések

### Backend Seed Megvalósítás

Javasolt: **Drizzle migration + raw SQL** (gyors, megismételendő, Production-ready).

```typescript
// packages/backend/src/db/migrations/0XXX_seed_tournament_bonus_types.sql
-- Idempotens seed: ON CONFLICT DO NOTHING az (name, is_global) páron
INSERT INTO special_prediction_types 
  (id, name, description, input_type, deadline, points, is_global, is_active, created_at, updated_at)
VALUES 
  (gen_random_uuid(), 'Csoportkör – legtöbb gólt szerző csapat', 'Melyik csapat szerzett a legtöbb gólt a csoportkörben?', 'team_select', '<első csoportmeccs kickoff>', 3, true, true, now(), now()),
  (gen_random_uuid(), 'Csoportkör – legkevesebb gólt szerző csapat', 'Melyik csapat szerzett a legkevesebb gólt a csoportkörben?', 'team_select', '<első csoportmeccs kickoff>', 3, true, true, now(), now()),
  (gen_random_uuid(), 'Csoportkör – legtöbb gólt kapó csapat', 'Melyik csapat kapott a legtöbb gólt a csoportkörben?', 'team_select', '<első csoportmeccs kickoff>', 3, true, true, now(), now()),
  (gen_random_uuid(), 'Csoportkör – legkevesebb gólt kapó csapat', 'Melyik csapat kapott a legkevesebb gólt a csoportkörben?', 'team_select', '<első csoportmeccs kickoff>', 3, true, true, now(), now()),
  (gen_random_uuid(), 'Torna gólkirálya', 'Ki szerzett a legtöbb gólt a teljes tornán?', 'player_select', '<első csoportmeccs kickoff>', 5, true, true, now(), now())
ON CONFLICT DO NOTHING;
```

**Alternatíva**: TypeScript-alapú seed script a `packages/backend/src/scripts/seed-tournament-types.ts`-ben, amely:
- Drizzle ORM-mel kérdez (type-safe),
- EXISTS check után INSERT-el,
- npm run scripten keresztül futtatható.

### Teszt szerkezet

```typescript
// packages/backend/src/db/seeds/tournament-types.test.ts
import { describe, it, expect, beforeEach } from 'vitest'
import { db } from '../client'
import { specialPredictionTypes } from '../schema'
import seedTournamentBonusTypes from './tournament-types.seed'

describe('seedTournamentBonusTypes', () => {
  beforeEach(async () => {
    // DB fixture: clean start
    await db.delete(specialPredictionTypes).where(eq(specialPredictionTypes.isGlobal, true))
  })

  it('Should insert 5 tournament bonus types on first run', async () => {
    const count = await seedTournamentBonusTypes()
    expect(count).toBe(5)
  })

  it('Should be idempotent – second run inserts nothing', async () => {
    await seedTournamentBonusTypes()
    const count2 = await seedTournamentBonusTypes()
    expect(count2).toBe(0)
  })

  it('Should have correct attributes for all types', async () => {
    await seedTournamentBonusTypes()
    const types = await db.select().from(specialPredictionTypes).where(eq(specialPredictionTypes.isGlobal, true))
    expect(types).toHaveLength(5)
    types.forEach(t => {
      expect(['team_select', 'player_select']).toContain(t.inputType)
      expect([3, 5]).toContain(t.points)
      expect(t.isGlobal).toBe(true)
      expect(t.groupId).toBeNull()
    })
  })
})
```

## Edge case-ek

| Helyzet | Elvárt viselkedés |
|---------|-------------------|
| Seed futtatása üres adatbázison | 5 típus kerül be, no errors. |
| Seed futtatása már meglévő 5 típussal | Idempotens — no duplicates, no errors. |
| Seed futtatása részleges adatokkal (pl. 3 típus már létezik, 2 nincs) | A 2 hiányzó típus kerül be, 3 meglévő untouched. |
| `deadline` helyesen van-e tárolva | TIMESTAMPTZ-ben a torna első meccse, mindegyik típusnál azonos. |
| `groupId` minden típusnál NULL | Megerősített: globális típusok, nem csoportspecifikus. |
| Egy csoport feliratkozott egy bonus típusra, majd megtörli | A `groupGlobalTypeSubscriptions` rekord törlődik, a típus a csoportnál már nem jelenik meg. |

## Kizárások

- **Nem** a torna statisztikáinak **valós időben** számolása (az adminok manuálisan állítják a helyes választ).
- **Nem** új `inputType` bevezetése — csak meglévő `team_select` és `player_select` használata.
- **Nem** a frontend felhasználói felületének módosítása — az 5 típus ugyanúgy jelenik meg, mint bármelyik speciális tipp.
- **Nem** a csoportos feliratkozási logika módosítása (`groupGlobalTypeSubscriptions`) — ez már létezik (US-926).
- **Nem** a kiértékelés rendszere — a meglévő `scoring.service.ts` kezel egy szól.
