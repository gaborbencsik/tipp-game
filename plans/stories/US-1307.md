# US-1307: Match Pulse – csapat-piaci értékek megjelenítése

## Leírás

Mint **bejelentkezett felhasználó**, szeretném **a Match Pulse szekcióban a két csapat aktuális Transfermarkt szerinti keret-piaci értékét egymás melletti horizontális sávokkal összehasonlítva látni**, hogy **gyors objektív viszonyítási pontom legyen a meccs előtt — kicsoda a papíron erősebb keret**.

## Elfogadási kritériumok

### Backend — API kiterjesztés

- [ ] A `GET /api/matches/:matchId/insight` endpoint válasza kiegészül egy `marketValues` objektummal:
  ```ts
  marketValues: {
    home: { teamId: string; teamName: string; valueEur: number | null }
    away: { teamId: string; teamName: string; valueEur: number | null }
  } | null
  ```
- [ ] A `valueEur` egész euróban, kerekítve (a DB `squad_market_value` cent érték / 100, `Math.round`)
- [ ] **Esetkezelés (mind a négy esetet le kell fedni):**
  - Mindkét csapatnak van értéke → mindkét `valueEur` szám
  - Csak hazai van → `home.valueEur` szám, `away.valueEur` null
  - Csak vendég van → `home.valueEur` null, `away.valueEur` szám
  - Egyik csapatnak sincs (mindkettő NULL) → `marketValues: null`
- [ ] A `204 No Content` válasz csak akkor jön, ha a meccshez nincs insight rekord — a market value nem befolyásolja az endpoint formáját
- [ ] Unit teszt mind a négy esetkezelési ágra

### Frontend — megjelenítés a Match Pulse szekcióban

- [ ] A "Csapatértékek" mini-blokk a Match Pulse wrapperen belül, az insight szöveg **FÖLÖTT** jelenik meg (gyors objektív horgony, mielőtt az AI próza absorption-t igényelne)
- [ ] **Vizuális minta (Polymarket-stílusú horizontális sávok, US-1501 mintájára):**
  - 2 sor, soronként: csapat zászló + név (vagy 3-betűs rövidítés mobilon) + horizontális bar + érték a bar végén
  - Mindkét bar szélessége a csapat értékének aránya a két érték összegéhez képest (`home / (home + away) * 100%`, ill. fordítva)
  - Hazai sor: primary szín (kék), vendég sor: secondary (narancs/zöld) — a meglévő app palettával konzisztens
  - A bar mögött halvány track (pl. `bg-gray-100`), hogy a 0-közeli érték is látható legyen
- [ ] **Numerikus erőviszony chip:** ha az arány > 1.5× az erősebb csapat sorának végén kis chip jelenik meg: `2,4× erősebb` (1 tizedesjegy). Egyenlőbb értékeknél (≤1.5×) chip nincs. Ez kiváltja a korábbi "min 5% bar" logikát — a sávok valódi arányt mutatnak, az erőviszonyt szöveges tag közvetíti.
- [ ] **Reveal-gated (US-1306):** a teljes mini-blokk CSAK akkor látható, ha `revealed === true`. Ez szándékos: a reveal-metrika méri, hány felhasználó nézi meg az elemzést egyáltalán. A blur a piaci értékre is vonatkozik, hogy a reveal-trigger egységes legyen.
- [ ] **Részleges/üres adat kezelés:**
  - Egyik csapat `valueEur` null → annál a sornál `—` jelenik meg az érték helyén, bar-track látszik de kitöltés nincs, chip nincs
  - `marketValues === null` → a teljes mini-blokk rejtve, nincs üres terület, nincs hibaüzenet
- [ ] **Forrás-attribution:** a blokk alján apró szürke szöveg: `Forrás: Transfermarkt` — a "Transfermarkt" szó link `https://www.transfermarkt.com/`-ra (`target="_blank"`, `rel="noopener noreferrer"`). ToS-megfelelés.
- [ ] **Accessibility:**
  - Minden bar `role="img"` + `aria-label="<csapatnév> piaci értéke <formázott érték>"`
  - A chip szöveges (nem csak színes), color-vakok számára is olvasható
  - A csapatnév/rövidítés látszik a sor elején — nem csak szín különbözteti meg a két oldalt
- [ ] Vitest teszt a komponensre:
  - Mindkét érték → mindkét sor + bar arányosan + chip ha >1.5×
  - Egyik null → `—` az adott sornál, üres bar, chip nincs
  - `marketValues: null` → blokk rejtve (`v-if`)
  - Pontos arány render: 70/30 → bar szélesség 70%/30% (nem clamp-elt)

### Formázás

- [ ] Új utility függvény: `formatMarketValueEur(eur: number, locale: 'hu' | 'en'): string`
  - **HU (kisbetű, sportsajtó konvenció):** `1_200_000_000` → `"1,2 mrd €"`, `850_000_000` → `"850 M €"`, `45_000_000` → `"45 M €"`, `0 < x < 1_000_000` → `"<1 M €"`, `0` → `"0 €"`
  - **EN:** `1_200_000_000` → `"€1.2B"`, `850_000_000` → `"€850M"`, `0` → `"€0"`
  - Szóköz a szám és az egység között HU-ban, közvetlen összevonás EN-ben (Transfermarkt EN konvenció)
- [ ] Unit teszt: határértékek (0, <1M, 1M, 999M, 1mrd, 999mrd), HU + EN

## Technikai megjegyzések

- A meccs lekérdezése egyetlen JOIN-nal történjen a `matches` és `teams` táblák között (a `home_team_id` és `away_team_id` alapján) — nincs külön `getTeamMarketValue` hívás
- A `teams.squad_market_value` mezőt az US-1406 sync tölti, BIGINT EUR cent (a JS `Number` 9_007 billió €-ig pontos, így biztonságos)
- A horizontális sáv egyszerű flexbox/grid: minden sor `<div :style="{width: pct + '%'}">` — nincs külön chart library
- Az insight `body` és a market value együtt jelennek meg ugyanazon a fetch-en — egyszerre frissül, nincs külön loading state
- **Mobil (<375px):** csapatnév 3-betűs rövidítésre (pl. `HUN`, `GER`), az érték és bar egy sorban marad, font-size csökken (`text-xs`)
- **Skeleton (US-1304 loader része):** 2× sor-skeleton (zászló placeholder 16×12 + szöveg placeholder ~80px + bar placeholder full-width 8px magas + érték placeholder ~60px), fix `min-height` a wrappernek hogy reveal után ne ugorjon a layout

## Nem tartalmazza

- Játékos szintű piaci értékek (ki a legértékesebb tag a kezdő 11-ben) — külön story lehet
- Történeti grafikon (értékváltozás az időben)
- Manuális admin override a megjelenített értékre
- Push értesítés értékváltozáskor
- A Match Pulse szekción kívüli megjelenítés (pl. csoport leaderboard mellett)
- Logaritmikus skála — két szám lineáris arányát mutatjuk, az extrém eltérést a numerikus chip kommunikálja

## Komplexitás

S — backend: 1 endpoint kiterjesztés (JOIN + 4-ágú esetkezelés) + utility, frontend: 1 mini-komponens (2 sor + chip + reveal-gate) + 1 formátum util. A `squad_market_value` adatforrás US-1406-tal már betöltve.
