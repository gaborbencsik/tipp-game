# US-940: Pending speciális tipp összesítő composable (`usePendingSpecialTips`)

> Forrás: special-tippek-banner-stories.md

### Cím

Pending speciális tipp összesítő composable (`usePendingSpecialTips`)

### User Story

Mint fejlesztő, szeretném, hogy a Mérkőzések oldalon legyen egy izolált, tesztelhető composable, amely kiszámolja, hogy a bejelentkezett usernek hány kitöltetlen, le nem járt speciális tippje van csoportonként, hogy ezt az adatot a banner és bármely jövőbeli UI elem egyszerűen fogyaszthassa.

### Elfogadási kritériumok

1. Új fájl: `packages/frontend/src/composables/usePendingSpecialTips.ts`
2. A composable neve `usePendingSpecialTips()`, visszatérési értéke:
   ```ts
   interface PendingGroupSummary {
     readonly groupId: string
     readonly groupName: string
     readonly pendingCount: number
     readonly nearestDeadline: string | null  // ISO string, a legközelebbi le nem járt határidő
   }

   interface UsePendingSpecialTipsReturn {
     readonly pendingGroups: ComputedRef<PendingGroupSummary[]>
     readonly totalPendingCount: ComputedRef<number>
     readonly now: Ref<number>
   }
   ```
3. A `pendingGroups` computed: iterálja a `useGroupsStore().groups` listát, minden csoporthoz megnézi a `specialPredictionsMap[groupId]` értékét a store-ból. Ha egy csoporthoz nincs még betöltve adat (`undefined`), az a csoport nem szerepel az eredményben (nem számít be a `totalPendingCount`-ba sem).
4. Egy `SpecialPredictionWithType` elem "pending"-nek számít, ha: `sp.answer === null AND new Date(sp.deadline).getTime() > now`.
5. A `nearestDeadline` az adott csoport pending elemeinek legkisebb deadline értéke (ISO string), vagy `null`, ha nincs pending elem.
6. A `totalPendingCount` a `pendingGroups` értékeiben lévő `pendingCount` összege.
7. A `now` ref értékét `Date.now()`-val inicializálja, és 60 másodpercenként frissíti egy `setInterval`-lal. Az interval `onUnmounted`-ban törlődik (a composable a Vue lifecycle horgokra támaszkodik).
8. A `now` ref publikusan exportált (hogy a banner komponens át tudja adni a `formatRelativeDeadline`-nak).
9. Unit teszt (`usePendingSpecialTips.test.ts`): legalább 4 eset:
   - Üres store (`groups = []`): `totalPendingCount === 0`, `pendingGroups === []`.
   - 1 csoport, 2 pending tipp: `pendingCount === 2`, `nearestDeadline` a 2 deadline közül a kisebb.
   - 1 csoport, minden tipp leadva (`answer !== null`): `pendingCount === 0`, a csoport nem szerepel `pendingGroups`-ban.
   - 1 csoport, minden tipp lejárt (`deadline < now`): `pendingCount === 0`.

### Edge case-ek

| Helyzet | Elvárt viselkedés |
|---------|-------------------|
| `specialPredictionsMap[groupId]` undefined (nem töltöttük be) | A csoport kihagyva — nem kerül a `pendingGroups`-ba |
| Csoport betöltve, de 0 pending elem | A csoport nem szerepel `pendingGroups`-ban (ne adj vissza `pendingCount: 0` sorokat) |
| Több csoport, csak az egyiknek van pending tippje | Csak az érintett csoport szerepel |
| A user egyszerre több csoportban tag | Minden érintett csoport saját sorral szerepel |
| A `deadline` pontosan `now`-val egyenlő | Nem pending (lejárt, `>` feltétel) |

### Technikai megjegyzések

- A composable a `useGroupsStore()` store-t használja — nem vesz fel prop-ot, nem hív API-t közvetlenül.
- A store `specialPredictionsMap` reaktív ref, a computed automatikusan frissül, ha a store-ban megváltozik.
- A `now` frissítése nem az interval belső értéke alapján számol, hanem `Date.now()`-t hív minden ticknél — így sosem csúszik el.
- Tesztfájl: `packages/frontend/src/composables/usePendingSpecialTips.test.ts`
- A tesztek mock store-t kapjanak (`vi.mocked` + Pinia test setup), ne hívják az éles API-t.
- Az `onUnmounted` hook miatt a tesztet egy ideiglenes Vue komponensbe kell burkolni, vagy a `getCurrentInstance()` guard-ot elkerülni. Alternatíva: a lifecycle hookot opcionális paraméterként injektálni (de az egyszerűbb a `withSetup` helper megközelítés, amit más spec fájlok is használnak).

### Hatókörön kívül

- Push értesítés.
- LocalStorage perzisztencia.
