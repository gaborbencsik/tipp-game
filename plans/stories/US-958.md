# US-958: Liga-típus (league/cup/mixed) + fázis-szűrő láthatóság

> Epic: E-Admin – Liga- és tornakezelés

## Háttér

A Mérkőzések oldal fázis-szűrő fülsora (csoportkör vs. egyenes kiesés) jelenleg minden ligánál látható, függetlenül attól, hogy az adott liga vegyes típusú-e. Az igazi VB liga valójában vegyes struktúrájú (csoportkör + egyenes kiesés), de sok ligának csak egy szakasza van (pld. egy egyenes kiesési kupa, vagy egy puro liga). A fázis-szűrőnek csak a vegyes ligáknál van értelme.

**Kódbázis-valóság (fontos, ne vizsgáld újra):**

- **Séma:** `packages/backend/src/db/schema/index.ts`. Az enum-ok `pgEnum`-mal vannak definiálva a fájl tetején (pl. `leagueStatusEnum` L57). A `leagues` tábla L164-184: mezők `id, name, shortName, startsAt, status (leagueStatusEnum default 'active'), syncEnabled, externalId, season, syncFrom, syncTo, fixtureAllowlist, createdAt, updatedAt`. Új enum szükséges: `leagueTypeEnum` értékekkel: `'league'`, `'cup'`, `'mixed'`.
- **Frontend League típus:** `packages/frontend/src/types/index.ts` L558-573 (`interface League`, readonly mezők). Bővítendő: `readonly type: 'league' | 'cup' | 'mixed'`. `LeagueInput` L575+ szintén.
- **Admin liga-form:** `packages/frontend/src/views/AdminLeaguesView.vue` — a create/edit form (`formData` ref) tartalmaz `name`, `shortName`, `externalId`, `season`, `status`, `syncEnabled` mezőket, `data-testid="league-form-*"` konvencióval. Új `<select data-testid="league-form-type">` mező kell (liga / kupa / vegyes opciók).
- **Backend liga-CRUD service:** `packages/backend/src/services/leagues.service.ts` — `/api/admin/leagues` route-okat kezel. A `createLeague` és `updateLeague` az `insertedLeagues`/`updatedLeagues` table-re írnak; kiterjesztendő `type` mezővel.
- **Fázis-szűrő (a lényeg):** `packages/frontend/src/views/MatchesView.vue`. A `stageFilterOptions` computed (L479-483): `[{allStages,null},{groupStage,'group'},{knockout,'knockout'}]`. A `SegmentedControl` (desktop L33-37) és a mobil `<select>` (L55-61) `matchesStore.stageFilter`-t köti. A store: `packages/frontend/src/stores/matches.store.ts` — `stageFilter = ref<StageFilterValue|null>(null)` (L34), `StageFilterValue = MatchStage | 'knockout'` (L13), `leagueFilter = ref<string|null>(null)` (L36). A `userLeagues` lista a MatchesView-ban elérhető (L40, 47, 63, 70 hivatkozza; `useLeagueFilter` composable L453). **Feltételes megjelenítés:** a fülsor (desktop SegmentedControl + mobil select) csak akkor rendereljön, ha `userLeagues` között van legalább egy `type==='mixed'`. Egyébként a fülsor rejtett **és** a `stageFilter` null-ra visszaesik.
- **Backend match/league API:** a Mérkőzések meccsei a `matches` táblából jönnek, a meccs `league` objektummal (FE `Match.league?.id`, `matches.store.ts` L47). A liga `type` mezőt a meccs-lista / user-leagues API válaszában is vissza kell adni, hogy a frontend tudja szűrni.
- **i18n:** `packages/frontend/src/locales/hu.json` és `en.json`. Meglévő kulcsok: `matches.allStages/groupStage/knockout` (hu L83-85), `admin.leagues.*` (form mezők `admin.leagues.form.*`). Új kulcsok kellenek: `admin.leagues.form.type` + a három opció label (`typeLeague` / `typeCup` / `typeMixed`).
- **Migráció:** a repo **kézzel írt** SQL migrációkat használ egyedi migrátorral (`src/db/migrator.ts`, `--> statement-breakpoint` szeparátor), NEM `drizzle-kit generate`. Új migráció mintája: `0075_league_sync_run_audit.sql` + bejegyzés a `src/db/migrations/meta/_journal.json`-ba (növekvő idx + tag). Az enum-hozzáadás mintája: `CREATE TYPE league_type AS ENUM ('league', 'cup', 'mixed')` + `ALTER TABLE leagues ADD COLUMN type league_type NOT NULL DEFAULT 'league'`.

**Terméki döntések (már eldöntött — NE kérdőjelezd meg):**

1. A fülsor akkor jelenik meg, ha a felhasználó ligái közül legalább egy `mixed` típusú. Az "Összes saját liga" nézetben is látszik, ha bármelyik saját liga vegyes. Ha egyik saját liga sem vegyes → a fülsor rejtett és a `stageFilter` null-ra esik vissza.
2. A meglévő ligák a migrációnál `league` default értéket kapnak (NEM `mixed`). Tehát a fülsor alapból eltűnik minden meglévő ligánál, amíg admin be nem állít vegyeset.
3. Az aktuális VB liga valójában vegyes (csoportkör + egyenes kiesés) — ezt az admin utólag állítja `mixed`-re; az adat-utómunka (admin manuális update vagy SQL) nem része ennek a story-nak, de jegyzendő.

## Story

Mint **adminisztrátor**, szeretném **egy liga-típus mezőt (liga / kupa / vegyes) beállítani az admin UI-ból**, hogy **a Mérkőzések oldal fázis-szűrője csak a vegyes ligák esetén jelenjen meg, és így a felhasználók csak a szükséges ligáknál látsanak fázis-szűrési lehetőséget**, és hogy a meglévő ligák alapértelmezetten elrejtik a szűrőt, amíg az admin be nem állítja vegyesnek.

## Elfogadási kritériumok

### Backend — DB

- [ ] **Új enum:** `leagueTypeEnum` az `auditActionEnum` mellé a `packages/backend/src/db/schema/index.ts` tetején: `pgEnum('league_type', ['league', 'cup', 'mixed'])`
- [ ] **Új oszlop:** `leagues` tábla bővítése: `type: leagueTypeEnum('league_type').notNull().default('league')`
- [ ] **Migráció:** kézzel írott SQL migráció a `packages/backend/src/db/migrations/` mappában: enum létrehozás + `ALTER TABLE leagues ADD COLUMN type` + default 'league'. Bejegyzés az `_journal.json`-ba (id, tag).

### Backend — CRUD service

- [ ] A `createLeague` a `formData.type` mezőt elfogadja és az adatbázisba írja.
- [ ] Az `updateLeague` módosítja a ligát, beleértve a `type` mezőt.
- [ ] A `getLeague(id)` és a liga-lista API (`getLeaguesForUser`, `getAllLeaguesAsAdmin` stb.) visszaadja a `type` mezőt.
- [ ] A `Match` API-válasz a meccs `league` objektumban tartalmazza a `type` mezőt.

### Frontend — Admin forma

- [ ] Az `AdminLeaguesView.vue` create/edit formon (`data-testid="league-form-type"`) új `<select>` mező jelenik meg:
  - Opciók: Liga, Kupa, Vegyes (i18n kulcsok alapján)
  - Default: "Liga" (új liga) / meglévő érték (edit)
  - Obligatorikus (nem üres)
  - Sikeres mentés után a liga objektum `type` mező frissül.

### Frontend — Mérkőzések oldal (fázis-szűrő láthatóság)

- [ ] A `MatchesView.vue` a `userLeagues`-ot átjárja (`computed`-ben vagy effect-ben) és kiszámítja: **van-e legalább egy vegyes liga?**
- [ ] Ha **nem** van vegyes liga: a fázis-szűrő (SegmentedControl desktop + select mobil) **rejtett** és a `stageFilter` **null-ra reset**.
- [ ] Ha **van** vegyes liga: a fázis-szűrő **látható**.
- [ ] Az "Összes saját liga" nézetben is ugyanez: ha egy saját liga sem vegyes, fülsor rejtett; ha legalább egy vegyes, fülsor látható.
- [ ] A szűrő alapértelmezésben az **összes fázist** mutatja (null stageFilter).

### i18n

- [ ] Magyar és angol locale fájlok:
  - `admin.leagues.form.type` = "Liga típusa"
  - `admin.leagues.form.typeLeague` = "Liga"
  - `admin.leagues.form.typeCup` = "Kupa"
  - `admin.leagues.form.typeMixed` = "Vegyes"

### Tesztek

- [ ] **Unit teszt (backend):** `createLeague` és `updateLeague` az `type` mezőt helyesen menti és olvassa (default 'league' új ligánál, érték módosítható).
- [ ] **Unit teszt (backend API):** a liga-lista és a Match-lista API válasz tartalmazza a `type` mezőt.
- [ ] **Frontend egységteszt:** az admin formában a `league-form-type` select az opciók közül választható és a `formData.type` frissül.
- [ ] **Frontend komponens teszt (MatchesView):** 
  - Ha a `userLeagues` közül **nincs** vegyes típus: a SegmentedControl és mobil select **rejtett**; `stageFilter` null.
  - Ha a `userLeagues` közül **van** vegyes típus: az elem **látható**.
  - Ha egy vegyes ligáról nem-vegyes ligára váltunk a szűrőben: a `stageFilter` null-ra reset (vagy marad — ha null, akkor OK; ha értéke volt, vegyes ligánál volt értelme).
- [ ] **Regressziós teszt:** a meglévő szűrés (league/cup szűrés a dropdown-ból) viselkedése változatlan; a `stage` filter a vegyes ligáknál a korábbi módon működik.

## Mit ne csináljunk ebben a story-ban

- **Ne** módosítsd a globális szinkron-logikát (US-956 `runAllLeagues` érintetlen).
- **Ne** vezess be szinkron-aktiválási vagy korlátozást a `type` alapján — a `type` csak a UI-szűrést vezérli, a szinkron a `sync_enabled` és `status` alapján fut.
- **Ne** írd újra a fázis-szűrő kereséslogikáját — a feltételes megjelenítés az UI-ban, a keresési logika érintetlen.
- **Ne** adj hozzá weitere félkész adat-migrációs logikát az aktuális VB liga `type`-jének beállításához; ez manual admin munka vagy utólagos adatmigrációs story lesz.
- **Ne** tárold az i18n kulcsokat sorokként (`admin.leagues.typeMixed` stb.) — csak a `type` value-k enum stringek, az i18n a komponensben szótárazódik fel.

## Technikai megjegyzések

- **Enum konvenciók:** az `leagueTypeEnum` az `auditActionEnum` mintáját követi (`pgEnum(...)` + `.notNull().default(...)`).
- **API szerződés:** a `Match` és `League` típusok (frontend + backend DTO-k) kiterjesztésére van szükség, de az üzleti logika a szűrés feltételét nem érinti — pár helyen kell csak `type` mező.
- **Fázis-szűrő reset logika:** ha `userLeagues` üres → fülsor rejtett (és `stageFilter` null). Ha `userLeagues` nem üres, de egyik sem vegyes → szintén rejtett és null. Ha legalább egy vegyes → látható. A state-reset (`stageFilter = null`) a computed-ben vagy az effect-ben történhet, amikor a `userLeagues` megváltozik.
- **Admin forma:** az `type` select opciói az `['league', 'cup', 'mixed']` enum-értékek lefordítása i18n-nel; nem hardcoded szöveg.

## Komplexitás

**M** — az enum és migráció egyenes (SQL sablon), a backend CRUD-bővítés szintén (egy mező hozzáadása). Az admin forma `<select>` egyszerű komponens. A fázis-szűrő láthatósága a MatchesView-ban egy `hasHasMixedType` computed és egy `v-if`-s feltételes renderelés + `stageFilter` reset. Viszont a függőség (userLeagues feltöltöttsége, szűrő reset logika) kissé trükkös lehet; közepes komplexitás.
