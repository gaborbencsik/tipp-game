# US-611: Mérkőzések automatikus szűrése a felhasználó csoport-ligái alapján

> Epic: E1 – Mérkőzések / E6 – Csoportok

**Story:**
Mint **felhasználó**, szeretném hogy **a mérkőzések oldal alapértelmezetten csak azokat a meccseket mutassa, amelyek a csoportjaim ligáihoz tartoznak**, hogy **ne kelljen irreleváns meccsek között keresgélnem**.

**Kontextus:**
Az US-609 bevezeti a `group_leagues` junction táblát (csoportok → ligák hozzárendelés). Erre a story-ra építve a matches oldal alapértelmezett szűrése a user csoport-ligáira vonatkozik. Ha a user nincs csoportban, minden meccset lát. Az US-610 liga select megmarad, de az alapértelmezett értéke a user releváns ligáira áll.

**Elfogadási kritériumok:**
- [ ] `GET /api/matches` alapértelmezetten a user csoport-ligái szerint szűr (backend)
- [ ] Ha a user egyetlen csoportban sincs, nincs szűrés (minden meccset lát)
- [ ] Ha a user több csoportban van különböző ligákkal, az UNION-t látja (minden releváns liga)
- [ ] `?scope=all` query param → bypass, minden meccs (az US-610 "Összes liga" opciója ezt használja)
- [ ] A matches oldal liga select default értéke: "Csoportjaim ligái" (nem "Összes")
- [ ] Ha a user csatlakozik/kilép csoportból, a match lista automatikusan frissül
- [ ] Unit tesztek: user 0 csoportban → all, user 1 csoportban 1 ligával → szűrt, user 2 csoportban 2 ligával → union

**Függőség:** US-609 (group_leagues junction), US-610 (liga select UI)

**Komplexitás:** M
**Prioritás:** Should Have
