# OPS-002: Render cron service konfigurálása az adaptive sync-hez

## Probléma

Production-ben az adaptive sync (`/api/internal/sync/tick`) **soha nem fut** automatikusan. A backend kód (gate logika, optimistic lock, scoring trigger) működik, de **nincs külső scheduler ami meghívja** az endpoint-ot.

**Tünet:** meccs eredmények nem frissülnek, pontok nem számolódnak — csak akkor, ha admin manuálisan futtatja a `/admin/sync/run`-t.

**Root cause:** hiányzó cron service Render-en + hiányzó `SYNC_SERVICE_TOKEN` env var.

---

## Story

Mint **adminisztrátor**, szeretnék **automatikus, percenkénti adaptive sync futást production-ben**, hogy **ne kelljen manuálisan futtatnom a sync-et minden meccs alatt**.

---

## Elfogadási kritériumok

1. **Render cron service létrehozva**
   - [ ] `render.yaml`-ban definiált cron service (vagy Render dashboard-on manuálisan)
   - [ ] Schedule: percenként (`* * * * *`) — a backend gate logika dönti el, hogy ténylegesen lefut-e
   - [ ] Command: `curl -X POST -H "Authorization: Bearer $SYNC_SERVICE_TOKEN" $BACKEND_URL/api/internal/sync/tick`
   - [ ] Environment vars: `SYNC_SERVICE_TOKEN`, `BACKEND_URL` beállítva

2. **`SYNC_SERVICE_TOKEN` env var production-ben**
   - [ ] Generálva (kriptográfiailag random, legalább 32 byte)
   - [ ] Beállítva a backend service env vars közé Render-en
   - [ ] Beállítva a cron service env vars közé Render-en (azonos érték)
   - [ ] `.env.example`-be hozzáadva placeholder-rel: `SYNC_SERVICE_TOKEN=replace-me-with-random-secret`

3. **Verifikáció**
   - [ ] Render dashboard cron logs: percenként látszik a curl request
   - [ ] Backend logs (jelenleg console, OPS-001 után strukturált): `sync.tick.received` event minden percben
   - [ ] Élő meccs alatt: `sync.tick.completed` event-ek látszanak, eredmény tényleg frissül

---

## Technikai megjegyzések

A `service-token.middleware.ts` már létezik és validálja a `Authorization: Bearer <token>` header-t. Ha a `SYNC_SERVICE_TOKEN` env var nincs beállítva, az endpoint 401-et ad vissza — ez a jelenlegi production állapot is (még a curl is fail-ne, de nincs curl).

A `sync-gate.ts` és `sync-state.service.ts` már kezeli az interval gate-et és optimistic lock-ot — percenkénti hívás biztonságos.
