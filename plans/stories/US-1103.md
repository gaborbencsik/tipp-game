# US-1103: Admin-triggered donation popup

> Epic: E11 – Támogatás

**Story:**
Mint **admin**, szeretnék **egy gombnyomással donation popupot küldeni minden felhasználónak**, hogy **emlékeztessem őket a támogatási lehetőségre anélkül, hogy tolakodó lennék**.

---

## Elfogadási kritériumok

### Admin oldal

- [ ] Új admin route: `/admin/settings` — általános "Beállítások" gyűjtő oldal
- [ ] A Beállítások oldalon "Donation popup küldése" szekció:
  - Egy "Popup küldése mindenkinek" gomb
  - Kattintásra megerősítő dialog: "Ez minden felhasználónál egyszer megjeleníti a donation popupot. Folytatod?"
  - Sikeres küldés után: megerősítés + "Utoljára kiküldve: [dátum]" megjelenítés
- [ ] Az admin nav-ban új "Beállítások" menüpont (⚙️ ikon)

### User-facing viselkedés

- [ ] Admin aktiválja → user következő oldalbetöltésekor (hard refresh VAGY első app mount) a DonationModal automatikusan megnyílik
- [ ] **2-3 másodperc delay** az oldal betöltése után — ne legyen azonnali
- [ ] A popup EGYSZER jelenik meg kampányonként — bezárás után localStorage-be mentjük a promptId-t
- [ ] Ha az admin újra megnyomja a gombot → új promptId → mindenkinél újra megjelenik egyszer
- [ ] **Route-aware suppression:** NE jelenjen meg, ha a user prediction oldalon van vagy meccs deadline <30 percen belül van — ilyenkor a következő "biztonságos" oldalon jelenik meg
- [ ] **"Ne kérdezd többé"** opció: kis szöveglink a modal alján — permanent localStorage suppress flag, ami felülírja az admin triggert

### Dismiss copy

- [ ] Fő dismiss: "Köszi, most nem" (nem "Talán később" — kevesebb guilt-trip)
- [ ] Permanent suppress: "Ne kérdezd többé" (kis, szürke szöveglink)

### Technikai elvárások

- [ ] Ha a user manuálisan megnyitja a sidebar donation gombot ÉS az auto-popup is aktív → ne legyen dupla modal (shared `isOpen` ref)
- [ ] localStorage hozzáférés try/catch-ben — private browsing módban ne törjön el
- [ ] Az API hívás (`GET /api/donation-prompt`) egyszer történik app mount-kor, nem minden route-váltásnál
- [ ] Ha az API hívás sikertelen → silently skip, ne dobjon hibát a usernek

---

## Technikai megjegyzések

### Backend

**Új tábla: `app_config`**
```sql
CREATE TABLE app_config (
  key VARCHAR(100) PRIMARY KEY,
  value JSONB NOT NULL,
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

**Endpointok:**
- `POST /api/admin/donation-prompt` — admin only, generál új UUID-t, upsert `app_config` key=`donation_prompt_id`
  - Response: `{ promptId: string, activatedAt: string }`
- `GET /api/donation-prompt` — auth required
  - Response: `{ promptId: string | null }`
  - `Cache-Control: no-store` header

**Alternatíva (egyszerűbb):** A `promptId`-t piggyback-elni a meglévő auth/session API response-ra (0 extra request). Eldöntendő implementációkor.

### Frontend

**Új composable: `useDonationPrompt`**
- App mount-kor hívja az API-t (egyszer)
- Logic:
  1. Ha `localStorage.getItem('donation-prompt-never')` === `'true'` → skip (permanent suppress)
  2. Ha `promptId` truthy ÉS `localStorage.getItem('donation-prompt-seen') !== promptId` → `shouldShow = true`
  3. Route check: ha current route prediction-related vagy deadline <30 perc → defer (watch route, show on next safe page)
  4. 2-3s setTimeout delay
- Close handler: `localStorage.setItem('donation-prompt-seen', promptId)`
- "Ne kérdezd többé" handler: `localStorage.setItem('donation-prompt-never', 'true')`

**AppLayout.vue módosítás:**
- `useDonationPrompt` composable használata
- Shared `showDonationModal` ref amit a sidebar gomb ÉS az auto-popup is használ

**Új view: `AdminSettingsView.vue`**
- Általános admin beállítások gyűjtő oldal
- Első szekció: Donation popup trigger + utolsó kiküldés dátuma

### Edge case-ek kezelése

| Edge case | Kezelés |
|-----------|---------|
| Multiple tabs | localStorage shared → ha egy tabban bezárja, más tabban nem nyílik meg újra. storage event listener nem szükséges ennél a méret |
| Private browsing | try/catch localStorage körül, fallback: session-scoped variable |
| promptId null + régi localStorage érték | Csak truthy promptId-ra nyitunk — `if (promptId && promptId !== stored)` |
| Sidebar klikk + auto-popup egyszerre | Shared isOpen ref, mindkét trigger ugyanazt írja |
| API failure | Silent skip, no retry |
| User prediction közben | Route-aware suppress: `/app/predictions`, `/app/matches/:id/predict` stb. |
