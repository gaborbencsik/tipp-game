# US-1301: Match Pulse – adatréteg és csapat KPI-ok gyűjtése

## Leírás

Mint **rendszer**, szeretnék **minden mérkőzéspárhoz strukturált csapat-statisztikákat összegyűjteni és tárolni**, hogy **az AI insight-generáláshoz rendelkezésre álljanak a releváns adatok**.

## POC eredmények (US-1301-POC)

- API stratégia: `GET /fixtures?team={externalId}&season={year}` — 3 season (2024, 2025, 2026), nem from/to
- 3 API hívás / csapat, 48 VB-csapatra 144 hívás (≈15 perc free tier-en)
- A `/teams/statistics` endpoint nem szükséges — a fixtures response-ból minden KPI számítható
- Team externalId párosítás megoldva (name-based fallback)

## Elfogadási kritériumok

### DB migráció

- [ ] A `match_insights` tábla migrációja lefut hibátlanul:
  - `id UUID PK`, `match_id UUID FK → matches.id (cascade delete)`, `type insight_type_enum NOT NULL`, `data JSONB NOT NULL`, `summary VARCHAR(200)`, `generated_at TIMESTAMPTZ DEFAULT NOW()`, `version SMALLINT DEFAULT 1`, `created_at TIMESTAMPTZ`, `updated_at TIMESTAMPTZ`
  - Unique index: `(match_id, type)`
  - Index: `match_id`
- [ ] Az `insight_type_enum` értékei: `defense`, `attack`, `form`, `set_pieces`, `key_matchup`, `fatigue`, `historical`

### FootballApiClient bővítés

- [ ] Új metódus: `fetchTeamFixtures(params: { teamId: number; season: number }): Promise<ApiFootballResponse<ApiFootballFixture>>`
- [ ] Az implementáció a `GET /fixtures?team={teamId}&season={season}` endpointot hívja

### Stats collector service

- [ ] `packages/backend/src/services/insights/stats-collector.service.ts`
- [ ] `collectTeamStats(externalId: number): Promise<TeamStats>` — 3 season lekérdezése, 24 hónapos ablakra szűrés, KPI aggregálás
- [ ] KPI-ok: totalMatches, wins/draws/losses, winRate, goalsScored/PerMatch, goalsConceded/PerMatch, cleanSheets/Rate, formString (utolsó 5 meccs)
- [ ] Befejezett meccs statusok: `FT`, `AET`, `PEN` — minden más kiszűrve
- [ ] Null goals fixture-ök skip-elve
- [ ] Ha nincs befejezett meccs → nullázott KPI-ok, üres formString, nem dob hibát
- [ ] Ha API 4xx/5xx → `StatsCollectionError` a hiba részleteivel

### Match-szintű gyűjtés

- [ ] `collectMatchStats(matchId: string): Promise<RawMatchStats>` — lekéri a hazai és vendég csapat `externalId`-jét a DB-ből, mindkettőre meghívja `collectTeamStats`-ot
- [ ] `RawMatchStats` típus: `{ homeTeam: TeamStats; awayTeam: TeamStats }`
- [ ] Ha egy csapatnak nincs `externalId` → `StatsCollectionError` (nem silent fail)

### Perzisztálás

- [ ] `saveMatchStats(matchId: string, stats: RawMatchStats): Promise<void>` — a `match_insights` táblába ment `type: 'raw_stats'`-ként, JSONB data mezőbe
- [ ] Upsert: `(match_id, type)` conflict → `data`, `generated_at`, `updated_at` frissül

### Unit tesztek

- [ ] `collectTeamStats`: vegyes eredmények, üres válasz, API hiba, null goals, kevés meccs
- [ ] `collectMatchStats`: sikeres (2 csapat), hiányzó externalId
- [ ] Legalább 8 teszteset

## Technikai megjegyzések

- SEASONS konstans: `[2024, 2025, 2026]` — a teljes US-1301-ben ez dinamikussá tehető (pl. current year - 2..current year)
- `external_id` a `teams` táblában elérhető (US-1202 + sync name-based fallback)
- HTTP hívásokhoz a meglévő `FootballApiClient` wrapper használandó
- Fájl helye: `packages/backend/src/services/insights/stats-collector.service.ts`
- Típusok: `packages/backend/src/services/insights/stats.types.ts`
