# US-1403: Automatikus játékos-szinkronizáció futball API-ból

## Leírás

Mint **adminisztrátor**, szeretném, hogy a `players` tábla automatikusan feltöltődjön a futball API-ból az összes `national` típusú csapat játékosaival és statisztikáival, hogy a `player_select` típusú speciális tippek kérdéseinél valós adatokból tudjon a felhasználó választani.

## Jelenlegi helyzet (implementáció után)

- `players` tábla: automatikusan feltölthető `syncPlayers()` hívással
- `player_stats` tábla: játékos statisztikákat tárol (gól, assziszt, sárga lap stb.) liganév + szezon + játékos kombinációra
- `football-api.service.ts`: `fetchSquad` (keret, `/players/squads`) és `fetchPlayers` (statisztikák paginálva, `/players`) metódusok megvannak
- `player-sync.service.ts`: `syncPlayers(client)` — összes `national` teamre iterál
- `sync.routes.ts`: `POST /api/admin/sync/players-run` — admin manuális trigger
- `internal-sync.routes.ts`: `POST /api/internal/sync/player-tick` — cron trigger, 24h rate limiterrel és `playerSyncEnabled` flag-gel
- `sync-state.service.ts`: `playerSyncEnabled`, `lastPlayerSyncAt`, `setPlayerSyncEnabled`, `markPlayerSyncFinished` megvannak
- `AdminSyncView.vue`: játékos szinkron toggle + "Játékosok szinkronizálása most" gomb megvan
- **HIÁNY:** `sync-runner.ts` nem integrálja a `syncPlayers`-t — a `/player-tick` cron végpont önálló route-ként létezik, de a Render Cron job-on csak a `/tick` fut

## Technikai megjegyzések

### Két API végpont, két fázis csapatonként

1. `GET /players/squads?team={teamId}` — keretek szinkronizálása (nincs season paraméter)
   - Visszaadja a squad player listát: `id`, `name`, `number`, `position`
2. `GET /players?team={teamId}&season={season}&page={n}` — statisztikák (paginált, season-onként)
   - SEASONS = `[2025, 2026]` hardcode-olva a service-ben

### player_stats unique key — leagueName alapján

Az eredeti tervben `league_id` volt az egyedi kulcs, de az api-football `/players/squads` nem ad vissza `league.id`-t. A tényleges implementáció `(playerId, season, leagueName)` kombinációt használ, ahol `leagueName` a `stat.league.name` string mezőből jön. A `filterValidStats` kiszűri azokat a sorokat, ahol `leagueName` null vagy üres.

### Upsert pattern (player_stats)

```typescript
.onConflictDoUpdate({
  target: [playerStats.playerId, playerStats.season, playerStats.leagueName],
  set: { appearances: sql`excluded.appearances`, ..., updatedAt: sql`now()` },
})
```

### Rate limit

- `FootballApiRateLimitError` dobódik exponenciális backoff (max 3 retry) után
- `syncPlayers` catch blokkja: rate limit → `errors.push(...)`, `skipped++`, `continue` (következő csapat)
- Párhuzamos hívás nincs — csapatonként szekvenciális iteráció

### Render Cron — nyitott teendő

A `/player-tick` végpont létezik és működik, de a Render Dashboardon külön Cron job-ot kell felvenni rá (pl. napi 1× éjjel), vagy a meglévő `/tick` logikájába kell beágyazni úgy, hogy ne ütközzön a napi API limittel.
