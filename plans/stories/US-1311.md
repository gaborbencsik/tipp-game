# US-1311: Admin-vezérelt torna-tipp kiértékelés (lightweight MVP)

> Epic: E5 – Insights & UX

## Háttér

A torna végén az adminnak manuálisan kell beírnia a helyes válaszokat a globális speciális tippekhez (például „Torna győztese", „Gólkirály", „Csoport végeredménye", „Kieséses fázis továbbjutottai"). Ezt az MVP-t úgy építjük, hogy minden típus **szeletenként külön kiértékelve** legyen: csoport tippek csoportonként, kieséses tippek körönként. A meglévő `scoreBracketProgression` placeholder függvény helyére bekerül az igazi halmaz-alapú pontozás. Ez az implementáció nem blokkolja a PLAN-001 terv későbbi pontosítása, de már működőképes alap.

## Story

Mint **admin**, szeretném, hogy a torna lezárásakor az admin felületen **minden globális speciális tipp típushoz beírjam a helyes választ**, és **szeletenként (pl. A csoport, negyeddöntő körök) külön gombokkal lefutassam a pontkalkulációt**, hogy **minden felhasználó automatikusan megkapja a helyesen számított pontokat a torna-tippjeiből**, és **azok azonnal megjelennek a leaderboard-on**.

## Hatókör (MVP scope)

### LE FED
- Globális (`is_global = true`) speciális típusok admin-vezérelt **manuális** kiértékelése.
- **Szeletenként** futtatható kalkuláció:
  - Egyéb típusok (text, dropdown, number, team_select, player_select): 1 input + 1 gomb (teljes típus).
  - Biztos kieső (multi_team_weighted): jelenlegi UI, 1 gomb.
  - Csoport tippek (all_groups_standing): **12 csoport** → 12 szelet, szeletenként 1 gomb + 1 „Mind" gomb. A **8 legjobb 3. helyezett** (`best3rds`) szelet **nem külön input** — automatikusan a (12 csoport 3. helyezettje) ∩ (bracket `last_32` résztvevői) halmazként számolódik, amint mindkét forrás kitöltött.
  - Kieséses tippek (bracket_progression): **6 kör** (last_32, last_16, qf, sf, final, bronze) → 6 gomb + 1 „Mind" gomb.
- Pontok visszaírása `special_predictions.points`-ba, leaderboard automatikus frissítése.
- Idempotencia: többszöri futtatás → ugyanaz az eredmény (overwrite, nem additív).
- Audit log (`audit_logs`, `action: 'result_set'`) minden kiértékelésre.
- Admin felület: `/admin/tournament-evaluation` vagy meglévő admin nézet bővítése.

### NEM FED
- Automatikus levezetés `match_results`-ból (gólkirály, torna győztes) — marad PLAN-001-re.
- Csoport-szintű (`is_global = false`) típusok — marad a meglévő `setCorrectAnswer`.
- Visszavonás (undo), rész-pont a csoport tippnél, párosítás-helyességi rész-pont a bracketnél.
- UI teljes redesign — csak admin widget-ek read-only módban.
- Ütemezés (cron, SSE push) — csak gombnyomás.

## Elfogadási kritériumok

### Backend

- [ ] **`scoreBracketProgression` implementáció:** halmaz-alapú pontozás körönként. Bemenet: `predicted` (tippelt sor), `correct` (helyes sor), `pointsPerMatch` (egy csapat pontja). Logika: körönként számolja, hány tippelt csapat van meg a helyes halmazban, függetlenül a párosítástól. Képlet: `(found_count / correct_count) * points_per_round`, vagy egyszerűbben: `points_per_match` per talált csapat. Output: teljes pont a körre, ha minden csapat talál, 0 ha semmi, arányos pont között. Teszt: tipp = helyes → max pont; tipp ⊆ helyes (halmaz) → arányos pont; tipp ∩ helyes = ∅ → 0.

- [ ] **Új service függvény:** `evaluateGlobalTypeSlice(typeId: string, slice?: string): Promise<EvaluationResult>`. Bemenet: típus ID, opcionális `slice` szöveg (pl. `'group_A'`, `'last_16'`). Logika: `correctAnswer` már be van írva a típusnál; végigmegy az aktív, nem törölt felhasználókon, kiszámolja `scorePrediction`-nal a pontot, visszaírja `special_predictions.points`-ba. Output: `{ evaluatedCount: number, totalPoints: number, lastRunAt: ISO8601 }`. Edge case: üres slice = teljes típus; csoport tippnél konkrét csoport (pl. `'group_A'`); kieséses tippnél konkrét kör (pl. `'last_16'`). Soft-deleted user-eket kihagyja (`deletedAt IS NOT NULL` vagy `users.isActive = false`).

- [ ] **Új admin-only PATCH endpoint:** `PATCH /api/admin/special-types/:typeId/correct-answer`. Body: `{ correctAnswer: string }`. Jogosultság: admin-only (meglévő middleware). Validáció: típus létezeit, `is_global = true`. Response: frissített `specialPredictionType` objektum. Hiba: 404 típus nem létezik, 400 érvénytelen correctAnswer (pl. nem parseable JSON a strukturált típusoknál).

- [ ] **Új admin-only POST endpoint:** `POST /api/admin/special-types/:typeId/evaluate`. Body: opcionális `{ slice?: string }`. Jogosultság: admin-only. Logika: hívja az `evaluateGlobalTypeSlice` service-t. Response: `{ evaluatedCount, totalPoints, lastRunAt }`. Hiba: 404 típus nem létezik, 400 érvénytelen slice (pl. nem létező csoport vagy kör).

- [ ] **Audit log:** minden `POST /admin/special-types/:typeId/evaluate` hívásra új rekord az `audit_logs` táblába. Mező: `action = 'result_set'`, `userId = requesterId`, `resourceType = 'special_prediction_type'`, `resourceId = typeId`, `details = { slice?, evaluatedCount, totalPoints }` JSON. Timestamp: `createdAt` az aktuális időpont.

- [ ] **Unit teszt `scoreBracketProgression`-ra:** 3 szcenárió: (1) teljes találat → max pont; (2) részleges találat (pl. 3 a 4 csapatból) → 75% pont; (3) 0 találat → 0. Edge case: üres `winners` objektum.

- [ ] **Unit teszt csoport-tipp pontozásra:** „pontos egyezés = teljes pont, bármely eltérés = 0" szabály. Teszt: (1) sorrend pontosan egyezik (4 csapat) → teljes pont; (2) 1 csapat rossz helyen → 0; (3) 3 csapat jó, 1 hiányzik → 0.

- [ ] **Unit teszt `best3rds` származtatásra:** (1) 12 csoport 3. helyezettje + last_32 lista adott → metszet pontosan 8 csapat → ezt használja a pontozás; (2) ha a metszet < 8 (last_32 hiányos) → `evaluateGlobalTypeSlice('best3rds')` érvénytelen, hibaüzenet; (3) felhasználói tipp `best3rds` halmaza ⊆ származtatott halmaz → találatok száma × egy-csapat-pont; (4) sorrend a tippben nem számít.

- [ ] **Unit teszt `evaluateGlobalTypeSlice`-ra:** (1) csoportos slice (`'group_A'`) → csak az A csoport tippjeit értékeli; (2) körös slice (`'last_16'`) → csak a last_16 kör csapatait számolja; (3) üres slice → teljes típus; (4) idempotencia: kétszer futtatva → ugyanaz az érték; (5) soft-deleted user kihagyása.

- [ ] **Integrációs teszt új endpoint-okra:** (1) `/admin/special-types/:typeId/evaluate` - 200 OK, visszatér `{ evaluatedCount, totalPoints }`; (2) jogosultság: non-admin → 403; (3) nem létező típus → 404; (4) `correctAnswer` még nincs beírva → működik (0 pont mindenkinek vagy hibaüzenet?).

### Frontend

- [ ] **Új admin nézet vagy meglévő admin oldal bővítése:** `/admin/tournament-evaluation`. Kilistázza az aktív, globális (`is_global = true`) speciális típusokat táblázatban: név, deadline, pontok, `correctAnswer` input mezője, gombsor.

- [ ] **„Egyéb" típusok (text, dropdown, number, team_select, player_select):** soronként 1 input mező a helyes válaszra (meglévő tippelő widget read-only módban vagy input), 1 „Kiértékelés futtatása" gomb. Gomb klikk: `POST /api/admin/special-types/:typeId/evaluate`, toast visszajelzés: „X felhasználó tippje értékelve, Y pont".

- [ ] **Biztos kieső (multi_team_weighted):** meglévő tippelő UI admin-módban (kiesett csapatok vizuális input), 1 „Kiértékelés futtatása" gomb.

- [ ] **Csoport tippek (all_groups_standing):** meglévő csoport-tábla UI admin-módban (12 csoport sorrend input). **Szeletenként 12 gomb**: A–L csoport (12 × „Kiértékelés: <csoport>") + 1 „Mind kiértékelése" (összesítő). Gomb klikk: `POST /api/admin/special-types/:typeId/evaluate` body: `{ slice: 'group_A' }` stb. A **8 legjobb 3. helyezett** szelete a UI-on is megjelenik **automatikusan származtatott, read-only** listaként, amint a 12 csoport 3. helyezettje és a `bracket_progression` típus `last_32` köre is ki van töltve. Ha valamelyik forrás hiányzik, a UI explicit jelzi: „A 8 legjobb 3. értékeléséhez add meg a 12 csoport sorrendjét és a `bracket_progression` last_32 csapatait." A `best3rds` szelet **nem önálló admin input** és **nincs külön „Kiértékelés" gomb** — a `'group_*'` szeletek lefutása után automatikusan kiértékelődik (lásd Backend pont).

- [ ] **Kieséses tippek (bracket_progression):** meglévő bracket UI admin-módban (6 kör tippei). **Szeletenként 6 gomb**: last_32, last_16, qf, sf, final, bronze → körönként 1 „Kiértékelés: <kör>" + 1 „Mind kiértékelése" (összesítő). Gomb klikk: `POST /api/admin/special-types/:typeId/evaluate` body: `{ slice: 'last_16' }` stb.

- [ ] **Válasz-beírás elmentése és UI feedback:** `correctAnswer` PATCH az input megváltozásakor (debounce) vagy „Mentés" gomb. Szeletenként: toast / inline status megjelenítése kiértékelés után: „<csoport>: 24 felhasználó, 480 pont", „Legutóbb: 2026-06-11 14:32 UTC".

- [ ] **Komponens teszt:** (1) admin nézet render-el összes globális típust; (2) szeletenként gombsor megjelenik; (3) gomb klikk meghívja az API-t; (4) toast/status megjelenés.

### Tesztelés

- [ ] **E2E teszt (Playwright):** admin belépés → `/admin/tournament-evaluation` → egy globális típus értékelése (pl. csoport A) → 200 OK, pont szinkronizálódik. Leaderboard: pont változik valóban.

- [ ] **Backend integrációs teszt:** `/admin/special-types/:typeId/evaluate` endpoint — admin user admin gombnyomás → 200, `evaluatedCount > 0`; non-admin → 403; nem létező típus → 404.

## Technikai megjegyzések

- **Schema:** nincs migráció szükséges; meglévő `special_predictions.points` használandó.
- **Pontozási logika:**
  - **Egyéb típusok:** meglévő `evaluateSpecialPrediction` (string normalizáció, teljes/0 pont).
  - **Biztos kieső:** meglévő `scoreUpsetSpecial` (súlyozás).
  - **Csoport tippek:** csoportonként **pontos sorrend egyezés szükséges** (4 csapat pozíciója), eltérés = 0. Implementáció: `parseAllGroupsStandingAnswer(correctAnswer)` az előzetes, `parseAllGroupsStandingAnswer(answer)` a tipp, összehasonlítás: `groups[X][0..3]` == `groups[X][0..3]`, true = teljes csoport-pont, false = 0.
  - **8 legjobb 3. helyezett (best3rds, származtatott):** az `all_groups_standing` típus `correctAnswer`-jébe **NEM kell** kézzel beírni a `best3rds` listát. A kiértékelő automatikusan kiszámolja: `derivedBest3rds = (12 csoport 3. helyezettje a correctAnswer-ben) ∩ (bracket_progression típus correctAnswer last_32 csapatai)`. Ha a metszet 8 csapat → ezzel pontoz; ha kevesebb (mert még nincs minden last_32 input) → a slice futtatása érvénytelen, hibaüzenet: „Hiányzó adat: bracket_progression last_32 még nem teljes". A felhasználói tipp `best3rds` mezőjének pontozása: **csapatonként 1 pont a 8-ból a metszetben**, a bracket körönkénti logikával konzisztensen (halmaz-egyezés, sorrend nem számít). A `'best3rds'` slice **automatikusan lefut**, amint mindkét típus megfelelő szelete kiértékelt — külön gomb nem kell.
  - **Kieséses tippek:** körönként halmaz-egyezés — `scoreBracketProgression` lecserélve halmaz-alapúra. Köri szeletelés: a `BracketProgressionAnswer.winners` map-ot körönként csoportosítva, majd az adott kör csapatai a tippelt winners helyiből kiolvasva (parse), és a helyes winners helyiből. Találatok száma / kör mérete * kör pont = az adott körre eső pont. Összesen: összes kör pontja.
- **Slice szintaxis:**
  - Csoport: `'group_A'`, `'group_B'`, … `'group_L'`. (`'best3rds'` nem külön admin slice — automatikusan származtatott, lásd fent.)
  - Kör: `'last_32'`, `'last_16'`, `'qf'`, `'sf'`, `'final'`, `'bronze'`.
  - Üres / null slice = teljes típus.
- **API:**
  - PATCH `/api/admin/special-types/:typeId/correct-answer` — `correctAnswer` szöveg beírása.
  - POST `/api/admin/special-types/:typeId/evaluate` — szeletenként vagy teljes kiértékelés.

## Kizárások

- Automatikus levezetés `match_results` vagy `player_stats`-ból — PLAN-001 terv később dönt.
- Csoport-szintű típusok — marad az eddig meglévő logika.
- Visszavonás, rész-pont, párosítás-helyesség — MVP-ből kimaradnak.
- Teljes UI redesign — csak admin widget-ek read-only konfigurációja.
