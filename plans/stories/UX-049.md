# UX-049: Pontozási szabályzat feature toggle — config flag-gel elrejtés

## Leírás

Mint **platform-admin**, szeretném, hogy a **pontozási szabályzat ("scoring explainer") modalja és annak összes belépési pontja statikus build-time config flag-gel legyen elrejthetővé**, hogy **szükség esetén az egész funkciót deaktiválhassam a UI-ból, deploy nélkül nem kell a kódot módosítanom — csak az env-et**.

## Jelenlegi helyzet

Az UX-028 (scoring explainer) modal és a `ScoringExplainerTrigger` komponens implementálva van. 5 belépési pont létezik a frontenden:
1. **AppLayout.vue L130-139**: bal menü nav item (`openScoringExplainer()` függvény + nav item elem)
2. **MatchDetailView.vue L88**: `<ScoringExplainerTrigger source="match-tip" variant="icon" />`
3. **TournamentTipsView.vue L7**: `<ScoringExplainerTrigger source="special-tip" variant="icon" />`
4. **GroupDetailView.vue L15**: `<ScoringExplainerTrigger source="group" variant="icon" />`
5. **LeaderboardView.vue L6**: `<ScoringExplainerTrigger source="leaderboard" variant="link" />`

A modal maga az **App.vue L4-ben** mount-olva: `<ScoringExplainerModal />` (import L9).

Jelenleg nincs lehetőség ezeket feature-toggle alapon elrejteni.

## Elfogadási kritériumok

### Config és composable

1. **Build-time env flag létrehozása**
   - Flag neve: `VITE_SCORING_RULES_ENABLED` (Vite env variable)
   - Default érték: `true` (az aktuális viselkedés marad az alapértelmezett)
   - Úgy működik: `.env` vagy `.env.production` fájlban definiálható
   - TypeScript-safe: nincs `any` típus, explicit `boolean` érték

2. **`useScoringRulesConfig()` composable létrehozása**
   - Hely: `packages/frontend/src/composables/useScoringRulesConfig.ts`
   - Exportál egy `isScoringRulesEnabled: boolean` reactive state-et vagy plain boolean-t
   - Olvassa az `import.meta.env.VITE_SCORING_RULES_ENABLED`-et (Vite magic)
   - Single source of truth: env-ből egy helyen olvasunk, az egész app ezt használja
   - Implicit érték: ha env undefined, az default `true`
   - Teszt-mock-olható: a composable importálható, env-je mock-olható

### Frontend — belépési pontok rejtése

3. **AppLayout.vue — nav menüpont elrejtése**
   - Feltétel: flag OFF → a nav item (`data-testid="nav-scoring-explainer"`) nem renderelődik (`v-if="isScoringRulesEnabled"`)
   - Feltétel: flag ON → a nav item normal módon megjelenik és működik

4. **Öt trigger komponens rejtése (MatchDetailView, TournamentTipsView, GroupDetailView, LeaderboardView)**
   - Feltétel: flag OFF → egyetlen `<ScoringExplainerTrigger>` se renderelődik (`v-if="isScoringRulesEnabled"`)
   - Feltétel: flag ON → összes trigger normal módon működik
   - **ScoringExplainerTrigger komponens** önmagában guard-olható: ha ezt nem renderelünk, a szülő `v-if` redundáns, de opcionális

5. **Modal (`App.vue`) elrejtése**
   - Feltétel: flag OFF → `<ScoringExplainerModal />` nem kerül be a DOM-ba (`v-if="isScoringRulesEnabled"`)
   - Feltétel: flag ON → modal normal módon mount-olva van
   - Plusz: ha valakinek URL-ből vagy store-ból közvetlen hozzáférése lenne a modalhoz, az se nyíljon meg → `ScoringExplainerStore.open()` no-op vagy warning (optional, de jó gyakorlat)

### Backend — N/A

- **Ez egy frontend-only feature toggle** — nincs backend módosítás
- Ha a flag OFF, a klient nem fogja le a scoring explainer-t → nem hívja az API-t sem (mert a komponens nem renderelődik)

### Tesztek (TDD)

6. **Unit teszt: `useScoringRulesConfig.test.ts`**
   - Teszt 1: `isScoringRulesEnabled === true` (default érték, env undefined-kor)
   - Teszt 2: env = `"true"` → `isScoringRulesEnabled === true` (string "true" parse-olódik)
   - Teszt 3: env = `"false"` → `isScoringRulesEnabled === false` (string "false" parse-olódik)
   - Teszt 4: `typeof isScoringRulesEnabled === 'boolean'` (explicit típus, nincs `any`)

7. **Komponens tesztek: trigger-ek és menü rejtése**
   - Teszt: AppLayout nav item (`data-testid="nav-scoring-explainer"`) **nem jelenik meg**, ha flag OFF
   - Teszt: AppLayout nav item **megjelenik**, ha flag ON
   - Teszt: `<ScoringExplainerTrigger>` 5 előfordulása (MatchDetailView, TournamentTipsView, GroupDetailView, LeaderboardView) — közös mock-kal tesztelhető (composable mock-olása)
   - Teszt: App.vue `<ScoringExplainerModal>` **nem renderelődik**, ha flag OFF
   - Teszt: App.vue `<ScoringExplainerModal>` **renderelődik**, ha flag ON

8. **Build-time ellenőrzés (opcionális, de hasznos)**
   - CI/CD: `.env.production` fájl legalább tartalmazza a `VITE_SCORING_RULES_ENABLED` definiálását (warning, ha nincs)
   - Default CI build: `VITE_SCORING_RULES_ENABLED=true` (az alapértelmezett behavior marad)

### Végfelhasználó szempont

9. **Flag OFF — semmilyen scoring explainer UI nem érhető el**
   - Nav menüpont: eltűnik
   - Trigger ikonok (5 helyen): eltűnnek
   - Modal: nem érhető el UI-ból
   - Ha valaki direkt URL-ből próbálna hozzáférni, az nem lesz probléma (modal nem nyílik meg)
   - **Viselkedés:** láthatatlan, de nem okoz hibát

10. **Flag ON (default) — mindent normál módon működik**
    - Nav menüpont: megjelenik, kattintható
    - Trigger ikonok: megjelennek, kattinthatók
    - Modal: nyitható, funkcionális
    - **Viselkedés:** jelenlegi UX-028 működés

## Technikai megjegyzések

### Composable és env olvasás

```typescript
// packages/frontend/src/composables/useScoringRulesConfig.ts

export function useScoringRulesConfig(): { isScoringRulesEnabled: boolean } {
  // Vite env-ből olvasás: string "true"/"false" vagy undefined
  const envValue = import.meta.env.VITE_SCORING_RULES_ENABLED;
  
  // Parse: undefined vagy "true" → true, "false" → false
  const isScoringRulesEnabled = envValue === undefined || envValue === "true";
  
  return {
    isScoringRulesEnabled
  };
}
```

**Megjegyzés:** `import.meta.env` Vite magic, build-time érték, type-safe. Nincs runtime lookup, nincs `any`.

### Env file (.env, .env.production, .env.local)

```bash
# Default: true (scoring explainer engedélyezett)
VITE_SCORING_RULES_ENABLED=true

# Ha ki kell kapcsolni:
# VITE_SCORING_RULES_ENABLED=false
```

### Komponens integrációk

#### AppLayout.vue

```vue
<template>
  <!-- nav item -->
  <button
    v-if="isScoringRulesEnabled"
    @click="openScoringExplainer()"
    data-testid="nav-scoring-explainer"
    class="..."
  >
    {{ label }}
  </button>
</template>

<script setup lang="ts">
import { useScoringRulesConfig } from '@/composables/useScoringRulesConfig';

const { isScoringRulesEnabled } = useScoringRulesConfig();
// ...
</script>
```

#### ScoringExplainerTrigger.vue (opcionális inner guard, de javasolt külső `v-if`)

```vue
<template>
  <button
    v-if="isScoringRulesEnabled"
    @click="openModal"
    :class="triggerClasses"
  >
    <!-- icon / label -->
  </button>
</template>

<script setup lang="ts">
import { useScoringRulesConfig } from '@/composables/useScoringRulesConfig';

const { isScoringRulesEnabled } = useScoringRulesConfig();
// ...
</script>
```

**Vagy:** minden szülő (`MatchDetailView`, `TournamentTipsView`, stb.) guard-olja a trigger-t:

```vue
<ScoringExplainerTrigger
  v-if="isScoringRulesEnabled"
  source="match-tip"
  variant="icon"
/>
```

**Javaslat:** a szülő `v-if`-et használj a trigger előtt (tiszta separation of concerns), az inner guard-ot ne is add hozzá.

#### App.vue

```vue
<template>
  <ScoringExplainerModal v-if="isScoringRulesEnabled" />
</template>

<script setup lang="ts">
import { useScoringRulesConfig } from '@/composables/useScoringRulesConfig';

const { isScoringRulesEnabled } = useScoringRulesConfig();
</script>
```

### TypeScript szigorúság

- Nincs `any`, nincs `as unknown`, nincs `@ts-ignore`
- `isScoringRulesEnabled: boolean` explicit típus
- Env-ből olvasott érték konvertálása boolean-né egyértelműen definiált

### Store-integrációs lehetőség (opcionális, de jó gyakorlat)

Ha a store-nak is szüksége lehet a flag-re, add hozzá az `scoringExplainerStore`-hoz:

```typescript
// stores/scoring-explainer.store.ts

export const useScoringExplainerStore = defineStore('scoringExplainer', () => {
  const { isScoringRulesEnabled } = useScoringRulesConfig();
  
  const open = (source: ScoringExplainerSource) => {
    if (!isScoringRulesEnabled) {
      console.debug('[ScoringExplainer] disabled, ignoring open() call');
      return;
    }
    // normal open logic
  };
  
  return { open, isScoringRulesEnabled };
});
```
