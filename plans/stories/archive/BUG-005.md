# BUG-005: PlayerSelectCombobox és TeamSelectDropdown nyers UUID-et mutat ha a csoport ligájában nincs játékos/csapat

## Leírás

Mint **csoporttag**, szeretném, hogy **a leadott `player_select` és `team_select` típusú speciális tipp az ID helyett a játékos/csapat nevét mutassa az input mezőben akkor is, ha a csoport ligájához (pl. pre-VB edzőmeccs liga) nincs feltöltött játékos vagy csapat lista**, hogy **ne UUID-et lássak a "Leadva" állapotú kártyán**.

## Reprodukció

1. Csoport, amely egy pre-VB / edzőmeccs ligához kapcsolódik (`groupLeagues.leagueId` = pre-VB liga UUID-je)
2. A speciális tipp típus globális (pl. "Gólkirály", `player_select`), a felhasználó már leadta (UUID-et tárolva a `special_predictions.answer` mezőben — pl. egy VB-keret játékos UUID-jét)
3. Csoport oldalon megnyitva a "Leadva" kártya az input mezőjében nyers UUID látszik a játékosnév helyett
4. Ugyanez `team_select` típusnál is, ha a kiválasztott csapat nincs a csoport ligájához rendelt csapatok listájában

## Gyökérok

- `PlayerSelectCombobox.vue` (`packages/frontend/src/components/predictions/PlayerSelectCombobox.vue`):
  - A `loadPlayers()` a csoport ligájához (`props.leagueId`) tartozó játékosokat kéri (`api.players.list(token, leagueId)`).
  - Ha a liga játékos listája üres, a `freeTextMode` computed `true` lesz (`loaded && playersList.length === 0`), és az input `:value="modelValue ?? ''"` — azaz a tárolt UUID-et közvetlenül szövegként megjeleníti.
  - Nem freeTextMode-ban a `displayText` `selectedPlayer?.name ?? ''`, ami szintén üres marad, ha a játékos nincs a `playersList`-ben.
- `TeamSelectDropdown.vue`: a `<select>` csak a `teamsList` opcióit tartalmazza. Ha a `modelValue` UUID nincs köztük, a select natív viselkedése miatt vagy üresen, vagy az adott UUID-et szövegként mutatja.
- A backend (`getMyPredictions`) a `players` ill. `teams` táblából liga-szűrés nélkül oldja fel a nevet és visszaadja `answerLabel` mezőben — tehát az **autoritatív név megvan**, csak a frontend komponensek nem használják fallbackként.

## Elfogadási kritériumok

### `PlayerSelectCombobox.vue`

- [ ] Új opcionális prop: `answerLabel?: string | null` (a backend `SpecialPredictionWithType.answerLabel` mezőjéből).
- [ ] `displayText` computed: `selectedPlayer.value?.name ?? props.answerLabel ?? ''` (sorrend: API-ból feloldott név → backend label → üres).
- [ ] `freeTextMode` computed: csak akkor `true`, ha a játékos lista üres **és** `props.answerLabel` is null/üres. Ha az `answerLabel` jelen van, a komponens nem vált free-text módba.
- [ ] FreeTextMode input value: `modelValue ?? answerLabel ?? ''` (ha tényleg free-text módban vagyunk és nincs `answerLabel`, a régi viselkedés marad).
- [ ] Ha a felhasználó beír valamit (free-text módban), az input értéke átveszi a `modelValue`-t (a régi viselkedés).

### `TeamSelectDropdown.vue`

- [ ] Új opcionális prop: `answerLabel?: string | null`.
- [ ] Ha a `modelValue` nem szerepel a `teamsList`-ben **és** `answerLabel` nem null, a `<select>` egy fallback `<option :value="modelValue">{{ answerLabel }}</option>` opciót is renderel (így a select a helyes nevet mutatja).
- [ ] Ha a `modelValue` szerepel a listában, a fallback opció nem jelenik meg (nincs duplikált opció).

### `GroupDetailView.vue`

- [ ] A `<PlayerSelectCombobox>` és `<TeamSelectDropdown>` mindkét helyen (submit edit blokk és set correct answer dialog) megkapja az `:answer-label="sp.answerLabel"` propot (a dialógusban a megfelelő típus `answerLabel`-jét).
- [ ] A "Leadva" állapotú kártya mostantól a játékosnevet/csapatnevet mutatja az input mezőben, nem UUID-et.

## Edge case-ek

| Helyzet | Elvárt viselkedés |
|---------|-------------------|
| `answerLabel` `null` és játékos lista üres | freeTextMode aktív, input a `modelValue`-t mutatja (a jelenlegi pre-VB viselkedés normalizált fallback) |
| `answerLabel` jelen van, de a `modelValue` `null` | input üres, placeholder látszik |
| A felhasználó beírja a saját szövegét free-text módban | `modelValue` az új szöveg lesz (a régi viselkedés) |
| A backend nem ad vissza `answerLabel`-t (régi cache) | input `modelValue`-t (UUID) mutat — a fix nem ront semmit |

## Hatókörön kívül

- A backend `getMyPredictions` szerver oldali változtatás: az `answerLabel` resolve már működik (lásd `special-predictions.service.ts:98-110`).
- A `PlayerSelectCombobox` általános refaktorálása (pl. egységesítés `TeamSelectDropdown`-nal).
- Globális típus létrehozási folyamat változtatása (admin oldal).

## Technikai megjegyzések

- A `sp.answerLabel` a backend felől már kötelezően jön (lásd `SpecialPredictionWithType` interface mindkét csomagban). A frontend csak nem használta fallbackként.
- Nem szükséges új API endpoint, nincs DB migráció.
- A változtatások visszafelé kompatibilisek: az új prop opcionális, alapértelmezett értéke `null`.
