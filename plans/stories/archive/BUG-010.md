# BUG-010 – Csoport "Group is full" a 20 tag elérésekor

## Tünet

A user a `FMYGHUBC` meghívó kóddal nem tudott csatlakozni a csoporthoz, "Group is full" hibát kapott, pedig a termékben nincs tervezett csoport-méret limit.

## Gyökérok

`packages/backend/src/services/groups.service.ts` `joinGroup` függvénye a per-user `MAX_GROUPS_JOINED = 20` konstanst használta a csoport tagjainak számára vetített ellenőrzéshez:

```ts
const existing = await db.select().from(groupMembers).where(eq(groupMembers.groupId, group.id))
if (existing.length >= MAX_GROUPS_JOINED) {
  throw new AppError(422, 'Group is full')
}
```

A `MAX_GROUPS_JOINED` szemantikusan azt jelenti, hogy egy user max hány csoportba léphet be — nem azt, hogy egy csoportnak hány tagja lehet. Sehol sem volt definiálva csoport-méret limit (sem schema-ban, sem konfigban), tehát ez egy másolás-eredetű bug volt.

## Javítás

- A teljes member-listát betöltő SELECT + count-check kikerült.
- Helyette célzott `userId`-szűrés egyetlen rekordra a "már tag" ellenőrzéshez.
- A `memberCount` a `toApiGroup` válaszhoz dedikált `count(*)` lekérdezésből jön.
- Per-user `MAX_GROUPS_JOINED` korlát változatlan.
