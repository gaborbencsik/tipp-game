# UX-041: Torna tippek – Biztos kiesők (Upset Special) kiértékelt tipp vizuális visszajelzés + részpont-bontás

## Háttér

Az UX-039 és UX-040 bevezette a Csoport végeredmény tipphez a zöld/piros vizuális visszajelzést és a per-csoport részpont-bontást. A Biztos kiesők (`multi_team_weighted`, Upset Special) kiértékelt nézete jelenleg csak az `answerLabel` és `correctAnswerLabel` szöveget mutatja — a felhasználó nem látja, melyik picke érte a teljes pontszámot és mennyit ért egyenként.

Az Upset Special pontozása:
- a `correctAnswer` egy JSON array a **ténylegesen kiesett** csapat-UUID-ekből (a `options.choices` poolból);
- minden user-pick, amely benne van az `eliminated` halmazban, **a saját `choice.points` súlyát kapja** (lásd `scoreUpsetSpecial`);
- a többi pick 0 pont.

## Hatókör

- Csak az `multi_team_weighted` típus kiértékelt módja.
- Frontend-only — a backend `scoreUpsetSpecial` változatlan; a `correctAnswer` és `points` már a frontre kerül.
- A picker `readOnly` propot már most is támogatja (US-936) — kibővítjük `correctAnswer` proppal.

## Elfogadási kritériumok

### Frontend – `UpsetSpecialPicker`

1. Új prop: `correctAnswer?: string | null` (JSON array az ténylegesen kiesett team UUID-ek).
2. Parse-olás: ugyanazzal a sémával mint az `answer` (string-array).
3. Ha `correctAnswer` jelen van **és** a picker `readOnly` módban van, a chips per-csapat vizuális visszajelzést kapnak:
   - **tipped + tényleg kiesett (helyes pick)** → zöld border (`border-emerald-500 bg-emerald-50`) + ✓ ikon + a chip alján zöld pirula `+Np`;
   - **tipped + NEM esett ki (helytelen pick)** → piros border (`border-rose-400 bg-rose-50`) + ✗ ikon + `0 pont`;
   - **NEM tipped + tényleg kiesett** → szaggatott zöld border + halvány `Tényleges` badge a chip alatt (mint a Best 3rd picker UX-039-ben);
   - **NEM tipped + NEM esett ki** → halvány szürke (`opacity-60`).
4. A picker tetején (jelenlegi instruction copy helyén kiértékelt módban) egy összegző sáv jelenik meg:
   - "Részpont-bontás: 3 helyes pick → **24 pont**" (példa);
   - "Tényleges kiesők: 12 csapat" megjegyzés a halmazról.
5. Az alsó "X / max kiválasztva" sor `readOnlyScored` esetben elrejtődik (helyén az összegző sáv).
6. Az `@submit` interakció `readOnly` módban már most blokkolt — változatlan.

### Frontend – `TournamentTipsView`

7. Ha `sp.points !== null && sp.inputType === 'multi_team_weighted'`:
   - rendereljen `UpsetSpecialPicker`-t `readOnly` és `correctAnswer={sp.correctAnswer}` proppal **a meglévő pontszám-blokk mellett**, helyettesítve a szöveges "Helyes válasz: …" sort.
   - A "Te tipped: …" + összesített pontszám marad a kártya tetején.
8. Ha az `options` nem érvényes `MultiTeamWeightedOptions`, fallback a meglévő szöveges összegzéshez.

### i18n

9. Új kulcsok hu/en mindkét lokálban: `upsetSpecial.scoredSummary`, `upsetSpecial.actualEliminatedCount`, `upsetSpecial.actualBadge`, `upsetSpecial.pointsZero`.

### Tesztelés

- Unit: `UpsetSpecialPicker`:
  - helyes pick → zöld classok + chip pirula a `+Np`-vel;
  - helytelen pick → piros classok + `0 pont`;
  - kiesett, de nem tipped → szaggatott zöld + "Tényleges" badge;
  - összegző sáv pontszáma a helyes pickek `points` súlyozott összege;
  - readOnly-ban a click nem emit `submit`-et.
- `TournamentTipsView`: scored `multi_team_weighted` → picker readOnly+correctAnswer renderel, szöveges fallback nem jelenik meg.

## Kizárások

- A picker olvasói módjának egyéb csiszolása (animáció, gestures) — out of scope.
- A `correctAnswer` szerver-oldali auto-derive — már megoldva US-1311-ben.
