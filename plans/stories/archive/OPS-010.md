# OPS-010: Frontend bundle + asset optimalizáció

## Probléma

A Matches oldal **prod-ban 8.3 MB / 917 request** az első látogatáskor. Cél: ~1-2 MB / kevesebb mint 50 request.

A vizsgálat:

| Probléma | Méret | Request |
|----------|-------|---------|
| `flag-icons` 271 ország × 4x3 + 1x1 SVG | **5.4 MB** | **542 request** |
| PWA service worker `globPatterns: '**/*.{js,css,svg,...}'` | mindent precache-el az induláskor | a többiek többszöröse |
| Vendor (vue, supabase, i18n) egy main chunk-ban | nincs hosszú-távú cache hit revenue-ban | — |
| Lazy locale-ok | hu+en mindkettő bundle-ben | ~ |

A torna csak **48 országot** használ (Hungary VB 2026 + barátságos meccsek), nem 271-et.

---

## Story

Mint **felhasználó**, szeretném, hogy **az alkalmazás első betöltése legalább 5-10x gyorsabb legyen**, mert lassú mobil hálón a 8 MB elviselhetetlen.

---

## Elfogadási kritériumok

### A — flag-icons szűkítése

1. [ ] Saját kis CSS modul: `src/styles/flags.css`, ami csak a torna 48 országára tartalmaz `.fi-xx` szabályt
2. [ ] Az SVG-k `import.meta.glob` segítségével dinamikusan beemelve a `flag-icons` packageből, csak a használt countryCode-ok
3. [ ] A `style.css`-ből kivenni a `@import "flag-icons/css/flag-icons.min.css"`-t

### B — PWA precache szűkítés

1. [ ] `vite.config.ts` `injectManifest.globPatterns` szűkítve: csak `index.html`, `manifest.webmanifest`, `**/*.css`, fő `*.js` (nem **/*.{js,svg,png,...})
2. [ ] `globIgnores` a flag SVG-kre, vendor split asset-ekre

### C — Vendor manualChunks

1. [ ] `vite.config.ts` `build.rollupOptions.output.manualChunks`:
   - `vue` (vue, vue-router, pinia)
   - `i18n` (vue-i18n)
   - `supabase` (@supabase/supabase-js)
2. [ ] Saját `views/` route-szintű splitting már működik (`() => import(...)`), nem kell változtatás

### D — Verifikáció

1. [ ] `npm run build` után a `dist/` méret `<` 3 MB
2. [ ] Lokálisan a Network tab-ban a Matches oldal kevesebb mint **100 request / 2 MB**
3. [ ] Teszt: `npm test` zöld marad (785/785)
4. [ ] Typecheck: `npx vue-tsc --noEmit` clean

## Mit NE csinálj

- Ne dobd ki teljesen a flag-icons NPM packaget — a bundlolt SVG-k onnan jönnek
- Ne erőltesd a route splittingt jobban (már működik)
- Ne nyúlj a Vue komponens kódhoz (csak CSS / config / asset szint)
