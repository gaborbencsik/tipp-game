# UX-043: Knockout meccsek – döntetlen tipp visuális megjelenítése kiértékelés után

## Háttér

Az egyenes kiesés (knockout) meccseken tippelőknek a végeredmény tipp mellé egy "döntetlen esetén ki jut tovább" mezőt is ki kell tölteniük (`predictions.outcomeAfterDraw` — továbbmentő csapat, extra_time_home/away vagy penalties_home/away). A pontozási szabályok már támogatják ezt: ha döntetlen lett és a user helyesen tippelte a továbbmentőt, plusz pontot (`extraTimeBonusPoints`) kap.

**Jelenlegi probléma:** A saját tipp és a "mások tippjei" szekció kiértékelés után nem mutatja meg a döntetlen-tipp mezőt, illetve nem jelenítik meg vizuálisan, hogy:
- A meccs döntetlen volt-e
- A user helyesen tippelte-e a továbbmentőt (vagy nem volt döntetlen, így nem aktiválódott)
- Hány pont járt a döntetlen-tippért

## Story

Mint **bejelentkezett felhasználó**, szeretném, hogy **egy knockout meccs MatchCard / MatchDetailView-jában a kiértékelés után látssza a döntetlen-tipp messze** (továbbmentő csapat, és hogy elért-e érte pontot), hogy **visszautalást kapjak az eredményre és a pontozásra.**

Illetve mint **csoporttag**, szeretném a "mások tippjei" szekciónál is **látni a többi user döntetlen-tippjét** (melyik csapatra tippeltek, és hogy jó-e volt), hogy **jobban megértsem, miként tippelt másokban az adott meccsre.**

## Jelenlegi helyzet

- Backend: `predictions.outcomeAfterDraw` és `match_results.outcomeAfterDraw` mezőkben tárolódik a döntetlen-tipp és az eredmény.
- Pontozás (`scoring.service.ts`): ha `result.homeGoals === result.awayGoals` és `prediction.outcomeAfterDraw === result.outcomeAfterDraw`, `extraTimeBonusPoints`-ot ad (alapjárműen 1 pont).
- Frontend: a MatchCard/MatchDetailView meccs leadása után megjeleníti a saját tippet, de a `outcomeAfterDraw` mezőt nem mutatja.
- Backend API: a `getMatchPredictions()` endpoint (mások tippjei) nem adja vissza az `outcomeAfterDraw` mezőt — csak `homeGoals`, `awayGoals`, `pointsGlobal`, `scorerBonusPoints` van benne.

## Elfogadási kritériumok

### 1. Saját tipp megjelenítése kiértékelés után (MatchCard / MatchDetailView)

1. **Knockout meccs után:** a tipp összefoglalóban megjelenik a "Döntetlen esetén:" mező:
   - Csapatnév badge + zászló (pl. "Döntetlen esetén: Magyarország")
   - Csak knockout meccsekre (stage: `round_of_32`, `round_of_16`, ..., `final`).

2. **Vizuális státusz az eredmény alapján:**
   - **Döntetlen + helyes tipp** → zöld pipa ✓ + "+1 pont" label (ha `extraTimeBonusPoints === 1`).
   - **Döntetlen + rossz tipp** → szürkén/ áthúzva X / szürke label "Nem jött be".
   - **Nem döntetlen** → szürke, deinaktivált jelzés: "Nem volt döntetlen" / könnyű szürke ikon.

3. **Pozícó a UI-ban:** a "Döntetlen esetén" mező a normál tipp (pl. "Magyarország 1-1 Szerbia") alatt vagy mellett, függőlegesen szeparálva.

### 2. Mások tippjei lista – döntetlen-tipp megjelenítése

1. **Backend API bővítés:** a `GET /api/matches/:matchId/predictions` endpoint (`getMatchPredictions()`) adja vissza az `outcomeAfterDraw` mezőt is (csapat ID vagy enum érték aszerint, amit a DB tárol — ellenőrizd).

2. **Frontend "mások tippjei" komponens:** minden user sorában megjelenik:
   - Normál tipp (pl. "1-1")
   - Plusz sor vagy badge: "Döntetlen esetén: {csapatnév}"
   - Ugyanaz a vizuális státusz, mint a saját tipp esetén (zöld/szürke/X).

3. **Sor szövege pl.:** "Alice: 1-1 (Döntetlen esetén: Szerbia) ✓ +1"

### 3. Vizuális feedback és pontjelzés

1. **Badge / szöveg minden esetben:**
   - Helyes: zöld hátterű "Eltaláltad +1" vagy ✓ ikon.
   - Rossz: szürke hátterű "Nem jött be" vagy X ikon.
   - Nem aktiválódott: szürke háttérű "Nem volt döntetlen".

2. **A pont számot az `extraTimeBonusPoints` konfig határozza meg** — a Badge ezt a értéket mutatja (alapjárműen "+1", de csoport override esetén eltérő lehet).

### 4. Csak knockout meccsek

1. A feature csak akkor aktiválódik, ha `match.stage` a knockout szakaszokra érvényes (nem `group`).
2. Csoportkörön NEM jelenik meg a "Döntetlen esetén" mező.

### 5. Backend validáció

1. `getMatchPredictions()` (`packages/backend/src/services/predictions.service.ts`) szekció, ahol az `outcomeAfterDraw` lekérdezésre kerül:
   - Csak knockout meccsekre (`stage !== 'group'`) adja vissza az `outcomeAfterDraw` mezőt (opcionális: Frontend validálja, Backend megadja az összes esetben).
   - Dummy/null érték, ha nincs adat.

### 6. Tesztelés

1. **Unit teszt (Frontend):**
   - Knockout meccs + döntetlen + helyes tipp → zöld badge + "+1".
   - Knockout meccs + döntetlen + rossz tipp → szürke badge + "Nem jött be".
   - Knockout meccs + nem döntetlen → szürke jelzés + "Nem volt döntetlen".
   - Csoportkör → nincs "Döntetlen esetén" mező.
   - Mások tippjei listája: az `outcomeAfterDraw` mező mindegyik felhasználónál megjelenik ugyanezzel a státusz logikával.

2. **E2E teszt:**
   - Létrehozunk egy knockout meccset (stage: `round_of_16`).
   - Felhasználó A tippel: 0-0 + döntetlen esetén Magyarország.
   - Felhasználó B tippel: 0-0 + döntetlen esetén Szerbia.
   - Admin beállítja az eredményt: 0-0 döntetlen, továbbjutó: Magyarország.
   - MatchDetailView-n ellenőrizzük:
     - A saját tippet (0-0 + Magyarország) zöld ✓-vel és "+1"-gyel.
     - "Mások tippjei" listán B User sorát szürkével és "Nem jött be"-vel.

## Technikai megjegyzések

### Backend (`packages/backend/src/services/predictions.service.ts`)

- **getMatchPredictions() bővítés:**
  - Az `outcomeAfterDraw` mező lekérdezésre kerül.
  - Ha a match stage `group`, az `outcomeAfterDraw` elhagyható / null-as.
  - Vagy: mindig adódjon vissza, Frontend szűri.

- **MatchPrediction típus:** (`packages/backend/src/types/index.ts`)
  ```typescript
  interface MatchPrediction {
    userId: string
    displayName: string
    homeGoals: number
    awayGoals: number
    pointsGlobal: number | null
    pointsResult: number | null
    scorerPickPlayerId: string | null
    scorerPlayerNameSnapshot: string | null
    scorerBonusPoints: 0 | 1 | null
    outcomeAfterDraw?: string | null  // NEW
    isSupporter: boolean
    isPaid?: boolean
  }
  ```

### Frontend komponensek

- **MatchCard, MatchDetailView** (saját tipp szekció):
  - Döntetlen-tipp badge: csak knockout meccsekre.
  - Pontozás logika: `calculateBonusStatus(result, prediction, config)` helper:
    ```typescript
    function calculateBonusStatus(
      result: MatchResult,
      prediction: Prediction,
      config: ScoringConfig,
    ): 'correct' | 'incorrect' | 'inactive' | null {
      if (result.homeGoals !== result.awayGoals) return 'inactive'
      if (prediction.outcomeAfterDraw === null) return 'inactive'
      if (prediction.outcomeAfterDraw === result.outcomeAfterDraw) return 'correct'
      return 'incorrect'
    }
    ```

- **"Mások tippjei" lista komponens:**
  - Az `MatchPrediction.outcomeAfterDraw`-t renderelni.
  - Ugyanaz a `calculateBonusStatus` logika.

### Adatábázis

- Nincs új séma / migration.
- Meglévő mezők: `predictions.outcomeAfterDraw`, `match_results.outcomeAfterDraw`.

## Kizárások

- NEM szükséges az "elrejt" / "mutasd" toggle a "Döntetlen esetén" mező számára.
- NEM szükséges az eredmény megváltoztatása után a tipp pontjainak retroaktív frissítése — ezt az admin már kezel.
- NEM szükséges az adott user teljes pontstatisztikájában a döntetlen-pont külön megjelenítése (csak a meccs szintjén).

## Komplexitás

L (1-2 nap) — Backend API bővítés (pequeño), Frontend komponens logika (több helyen), vizuális design (badge-ek, státusz), tesztek.
