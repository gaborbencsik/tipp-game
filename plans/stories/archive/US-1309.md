# US-1309: Odds-sáv felfedése — reveal mechanizmus

## Leírás

Mint **bejelentkezett felhasználó**, szeretnék az **elmosódott Polymarket odds-sáv felfedésére egy "Elemzés felfedése" gombbal**, hogy **az odds megtekintése az én aktív döntésem legyen, és következő látogatásra is megmaradjon**.

A `?showInsights=true` query string logika teljes egészében helyettesítendő egy adatbázis-alapú, RLS-sel védett reveal mechanizmussal. Ez a story az **odds-sávra vonatkozik** — a teljes Match Pulse insight-tartalmakra való kiterjesztés az **US-1306** felelőssége.

## Jelenlegi helyzet

- `MatchDetailView.vue` 176-187: `?showInsights=true` query string logika (URL-trükk, nem perzisztált)
- Nincs adatbázis-tárolás, nincs felhasználói akció, nincs telemetria
- Az odds `MatchOddsBar` komponens elmosódva (`blur-sm`), statikus "Hamarosan" overlay-el

## Elfogadási kritériumok

### Adatbázis & RLS

1. **insight_reveals tábla létezik**
   - `id` UUID PK
   - `user_id` UUID NOT NULL FK → `users.id`
   - `match_id` UUID NOT NULL FK → `matches.id`
   - `revealed_at` TIMESTAMPTZ NOT NULL DEFAULT NOW()
   - UNIQUE constraint: `(user_id, match_id)`
   - RLS engedélyezve: autentikált felhasználó saját `user_id`-jére írhat; SELECT csak saját sorokat; admin minden sort lát

2. **Drizzle séma + migráció**
   - `insight_reveals` a `packages/backend/src/db/schema/index.ts`-ben definiálva
   - Migráció létrehozva és verzionálva

### Backend — Reveal endpoint

3. **POST /api/matches/:matchId/insights/reveal — idempotens**
   - Autentifikáció kötelező (auth.middleware)
   - Ha már létezik sor az `(user_id, match_id)` párra, `200 OK` + `{ revealed: true }` (nem hiba)
   - Ha `matchId` nem létezik, `404 Not Found`
   - Response: `{ revealed: true }`
   - Rate limit: max 10 / perc / felhasználó

4. **Reveal állapot lekérése**
   - Opció A: A `GET /api/matches/:matchId/odds` endpoint kiegészül egy `revealed: boolean` mezővel
   - Opció B: Külön `GET /api/matches/:matchId/reveal-status` endpoint
   - A felelős: a story implementálójában indokolni kell, melyik választás (javaslat: opció A, mert az odds és a reveal szorosan összekapcsolt)

### Frontend — Blur + felfedés flow

5. **Query string logika eltávolítva**
   - Az `?showInsights=true` query string keresése **teljesen eltávolítva** a kódból
   - `MatchDetailView.vue` az odds-sáv felfedésére az API-ból tölti a `revealed` státuszt

6. **Blur + gomb UI alapértelmezetten**
   - Az odds-sáv elmosódva (`blur-sm`) jelenik meg, ha `revealed === false`
   - Az elmosódott szekció felett "Elemzés felfedése" gomb
   - Gomb megnyomásakor: `POST /api/matches/:matchId/insights/reveal` hívás
   - Sikeres válasz után: blur animáltan eltűnik (CSS `filter: blur(0)` 300ms ease-in-out transition), gomb eltűnik

7. **Hiba kezelés**
   - API hívás sikertelen (5xx, hálózati hiba): gomb aktív marad, "Nem sikerült felfedni, próbáld újra" üzenet 4 mp után auto-eltűnik

8. **Már felfedett állapot**
   - Ha `revealed === true` az oldalbetöltéskor, az odds-sáv azonnal blur nélkül, gomb nélkül jelenik meg
   - Nincs animáció

9. **Állapot perzisztálása**
   - Felfedett állapot Pinia store-ban cache-elve; oldalfrissítés a DB-ből tölti vissza
   - **NEM** localStorage — kizárólag DB az igazságforrás

### Tesztek

10. **Backend unit tesztek**
    - Reveal service: idempotens insert (már meglévő sor → nem dob hibát)
    - `POST .../reveal`: nem létező matchId → 404
    - `POST .../reveal`: sikeres eset → `{ revealed: true }`
    - Rate limit teszt: 11 hívás / perc → 429 Too Many Requests

11. **Frontend Vitest tesztek**
    - `useReveal` composable (vagy insights store): `revealed: false` → gomb jelenik meg, blur alkalmaz
    - Gomb klikkelés → API hívás → sikeres válasz → `revealed: true`, blur eltűnik, gomb eltűnik
    - API hiba eset → gomb aktív, hibaüzenet jelenik meg
    - `revealed: true` inicializáláskor → nincs gomb, nincs blur

## Technikai megjegyzések

- **Endpoint**: `POST /api/matches/:matchId/insights/reveal` az `packages/backend/src/routes/matches.ts`-ben vagy dedikált `packages/backend/src/routes/insights.ts` fájlban
- **Service**: `revealInsight(userId, matchId): Promise<{ revealed: boolean }>` az `packages/backend/src/services/insights/reveal.service.ts`-ben
- **Drizzle upsert**: `ON CONFLICT (user_id, match_id) DO NOTHING`
- **RLS**: `ALTER TABLE insight_reveals ENABLE ROW LEVEL SECURITY` + politikák:
  ```sql
  CREATE POLICY "Users can insert own reveals" ON insight_reveals
    FOR INSERT WITH CHECK (auth.uid()::uuid = user_id);
  
  CREATE POLICY "Users can select own reveals" ON insight_reveals
    FOR SELECT USING (auth.uid()::uuid = user_id OR (SELECT role FROM users WHERE id = auth.uid()::uuid) = 'admin');
  ```
- **Frontend composable**: `const { revealed, isLoading, error, reveal } = useReveal(matchId)` — Pinia store-ban cache-elve
- **CSS transition**: `filter: blur(8px)` → `filter: blur(0)` 300ms ease-in-out
- **Pozicionálás**: `relative` wrapper a szekción, `absolute` overlay a blur felett — szekció tartalma ne ugorjon el
