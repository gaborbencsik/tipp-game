# BUG-006: Leaderboard tipp-számláló nem szűrte ki a soft-delete-elt meccseket

## Leírás

Mint **csoporttag**, szeretném, hogy **a csoport ranglistájának "Tipp" oszlopa csak az aktív (nem soft-delete-elt) meccsekre adott tippjeimet számolja**, hogy **a megjelenített érték konzisztens legyen a tényleges leadott tippekkel**.

## Reprodukció

1. Felhasználó tippet adott le egy meccsre.
2. Az admin (vagy a sync) később soft-törölte a meccset (`matches.deleted_at` ≠ NULL).
3. A csoport ranglistáján (és a globális ranglistán) a "Tipp" oszlop továbbra is beleszámolta a törölt meccs tippjét.

Konkrét eset: `gabor.bencsik13@gmail.com` a "Pre VB edzömeccsek -1-" csoportban 1 érvényes tippet adott le, de a UI 2-t mutatott (a 2026-05-21-i soft-delete-elt meccs tippje is bekerült).

## Gyökérok

- `packages/backend/src/services/group-leaderboard.service.ts`: a query egyik ágában sem szerepelt `isNull(matches.deletedAt)` szűrés. A `useGroupPoints=true` ágban a `groupPredictionPoints.id`-t számolta `matches`-szűrés nélkül, a `useGroupPoints=false` ágban csak akkor join-olta a `matches` táblát, ha `leagueId` is volt — soft-delete szűrés sehol.
- `packages/backend/src/services/leaderboard.service.ts`: a globális ranglista nem is join-olta a `matches` táblát, így a `predictions.id` count szintén beleszámolta a soft-delete-elt meccsek tippjeit.
- A CLAUDE.md kötelezi: *"Lekérdezéseknél mindig: `.where(isNull(table.deletedAt))`"*.

## Elfogadási kritériumok

### `group-leaderboard.service.ts`

- [x] Mindkét scoring ág (group-points és global-points) mindig `leftJoin`-olja a `matches` táblát a `predictions.matchId`-n keresztül.
- [x] A `WHERE` feltétel: `(predictions.id IS NULL OR (matches.deleted_at IS NULL [AND matches.league_id = leagueId::uuid]))` — ahol a leagueId rész csak akkor kerül be, ha a csoportnak van ligája.
- [x] A user-rekord akkor is megjelenik a ranglistán, ha 0 tippje van (LEFT JOIN tartja meg, az `OR predictions.id IS NULL` ág engedi át).

### `leaderboard.service.ts` (globális ranglista)

- [x] `leftJoin(matches, eq(matches.id, predictions.matchId))` hozzáadva.
- [x] `WHERE`: `isNull(users.deletedAt) AND (predictions.id IS NULL OR matches.deleted_at IS NULL)`.

### Tesztek

- [x] `group-leaderboard.service.test.ts` mock chain frissítve — minden scoring ágban mindig szerepel a `matches` leftJoin (akár van leagueId, akár nincs).
- [x] 3 új regressziós teszt: a `sql` template tag interpolált argumentumai között szerepel a `matches.deletedAt` mind a group-points ágban, mind a global-points ágban, mind a no-league esetben.
- [x] Backend tesztsorozat: 526/526 zöld, `tsc --noEmit` tiszta.

## Hatókörön kívül

- `users.deletedAt` szűrés a csoport-leaderboard-ban (groupMembers-en keresztül érkezünk, a tagsági kapcsolat eltávolítja a deleted user-eket).
- A `predictions` táblának nincs `deleted_at` mezője — ott nincs mit szűrni.

- API szerződés nem változik.
