# UX-015: Kedvenc csapat zászló a ranglistán

## Leírás

Mint **csoport tag**, szeretném **látni a többi tag kedvenc csapatának zászlóját a ranglistán**, hogy **áttekintsem, kinek van duplázási esélye az adott meccseken**.

## Elfogadási kritériumok

- [x] A zászló csak olyan csoportokban jelenik meg, ahol a `groups.favoriteTeamDoublePoints = true`
- [x] A zászló a ranglista soron a felhasználó neve mellett, attól jobbra jelenik meg
- [x] A zászló mérete 16x12px, a meglévő `flag-icons` könyvtár `fi fi-{countryCode}` osztályaival renderelve
- [x] Hover esetén tooltip jelenik meg a csapat teljes nevével (pl. "Magyarország")
- [x] Ha a felhasználó nem állított be kedvenc csapatot, a zászló helye üres — nincs placeholder vagy hibajelzés
- [x] A `GET /api/groups/:groupId/leaderboard` válasz tartalmaz minden taghoz egy `favoriteTeam: { countryCode, name } | null` mezőt
- [x] A kedvenc csapat lekérdezésekor az adott csoport `leagueId`-jéhez tartozó `user_league_favorites` sor kerül felhasználásra
- [x] Ha egy csoporthoz több liga tartozik, az első (vagy egyetlen) liga kedvence jelenik meg — többes liga esetén a viselkedés dokumentált
- [x] A leaderboard tesztek lefedik a `favoriteTeam: null` és a `favoriteTeam: { countryCode, name }` eseteket egyaránt

## Technikai jegyzetek

**Backend:**
- `GET /api/groups/:groupId/leaderboard` bővítése: a lekérdezés JOIN-olja a `user_league_favorites` táblát a csoport `leagueId`-je alapján, majd JOIN-olja a `teams` táblát a `countryCode` és `name` mezőkhöz
- Response shape bővítése: `entries[].favoriteTeam: { countryCode: string; name: string } | null`
- A `favoriteTeamDoublePoints` flag-et is vissza kell adni a response-ban, hogy a frontend feltételesen renderelhessen

**Frontend:**
- Érintett komponens: `LeaderboardTab.vue` (vagy a ranglista sort renderelő child komponens)
- Ha `group.favoriteTeamDoublePoints === false`, a `favoriteTeam` mező nem kerül renderelésre (v-if)
- Tooltip: natív HTML `title` attribútum elegendő — külső tooltip könyvtár nem szükséges
- A `flag-icons` CSS már importálva van, új dependency nem kell

**Adatfolyam:**
1. `GroupDetailView` betölti a ranglista adatot a meglévő leaderboard store-on keresztül
2. A store a bővített response-t tárolja
3. `LeaderboardRow` komponens (vagy inline) rendereli a zászlót a `countryCode` alapján

## Komplexitás

**M** — Backend JOIN bővítés + frontend feltételes render + tooltip

## Prioritás

Should Have

## Függőségek

- `user_league_favorites` tábla létezik és populált (feltételezett)
- `teams.countryCode` mező létezik a Drizzle schemában
- `groups.favoriteTeamDoublePoints` boolean mező létezik
- `flag-icons` könyvtár integrálva van a frontenden
