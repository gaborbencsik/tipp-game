# US-936: Upset Special – multi-csapatos, súlyozott bonus tipp típus

> Forrás: Tornaspecifikus bonus tippek – csoportkörös kiesések szórakoztató predikcióját lehetővé tevő új típus.

## User Story

Mint csoporttag, szeretném, hogy a torna **24 favoritja** közül pontosan **2-t kiválasszak**, akikről úgy gondolom, hogy **NEM** jutnak túl a csoportkörön, úgy hogy ha valamelyik tényleg kiesik, a hozzá rendelt pontérték jár nekem (pl. Spain 18p, Canada 2p), hogy szórakoztató módon wagerozzhessek a meglepetésekre.

## Jelenlegi helyzet

A `specialPredictionTypes` tábla támogatja:
- `inputType: 'team_select' | 'player_select' | 'text' | 'dropdown' | 'number'` — az Upset Special-hoz **multi-pick, súlyozott** opció hiányzik.
- `points: smallint` — egyetlen pontérték a típusra — az Upset Special-hoz csapatonkénti súlyozás kell.
- `specialPredictions.answer: text` — egy szöveges érték — az Upset Special-hoz 2 csapatnév/ID tárolásához nem alkalmas.
- `specialPredictionTypes.correctAnswer: text` — egy szöveges érték — az Upset Special-hoz a kiesett csapatok **listáját** kell tárolni.

Az `evaluateGlobalType()` scoring logika a meglévő típusokat kezel; az Upset Special-hoz új, tiszta függvény kell, amely azt számítja: „mely pickeik estek ki, és mennyi pont".

## Elfogadási kritériumok

### Backend

1. **Új inputType enum érték: `multi_team_weighted`**
   - Az `specialPredictionInputTypeEnum` Drizzle schemájában bővítendő.
   - Drizzle migration szükséges (up + down).
   - A meglévő `team_select`, `player_select`, stb. viselkedése **változatlan** (regression-mentes).

2. **`specialPredictionTypes.options` JSONB tartalma az új típushoz**
   - Új típus létrehozáskor az `options` JSONB-ben tárolódik az alábbi séma:
     ```json
     {
       "maxPicks": 2,
       "minPicks": 2,
       "choices": [
         { "teamId": "uuid-spain", "points": 18 },
         { "teamId": "uuid-france", "points": 18 },
         { "teamId": "uuid-england", "points": 17 },
         ...24 csapat összesen
       ]
     }
     ```
   - Az `options` mező meglévő az `specialPredictionTypes` táblán; serdülő típusoknál (mint a `team_select`) üres vagy `null` — most az Upset Special-nál töltött.

3. **Validáció típus létrehozáskor (admin endpoint)**
   - Az `options.maxPicks` és `minPicks` teljesítésére ellenőrzés (számlálás a `choices` array hosszán).
   - Az `options.choices` array 24 elemből áll; mindegyiknek `teamId` (UUID) és `points` (smallint, 1–18 közötti) értéke van.
   - Mindegyik `teamId` létezik az `teams` táblán (no orphan UUIDs, aktuális torna csapatai).
   - Ha validáció sikertelen, `400 Bad Request` a hibaüzenettel.

4. **User válasz tárolása (`specialPredictions.answer: text`)**
   - **A kiválasztott opció: JSON-encoded tömb** — `'["uuid-spain", "uuid-canada"]'` formátumban (szövegként tárolva az `answer` mezőben).
   - Validáció tippeléskor:
     - Pontosan `minPicks` (=2) csapat ki van választva.
     - Minden csapat az `options.choices` listában szerepel (no out-of-scope picks).
     - Nincs ismétlődés (ugyanaz a csapat nem választható kétszer).
     - Ha validáció sikertelen, `400 Bad Request`.

5. **Admin `correctAnswer` formátum**
   - Az `specialPredictionTypes.correctAnswer: text` mezőbe JSON-encoded tömb kerül: `'["uuid-canada", "uuid-austria", "uuid-ecuador", ...]'` — a **csoportkörben kiesett csapatok** ID-listája.
   - Validáció beállításkor (admin endpoint):
     - Minden ID az `options.choices` listában szerepel.
     - Ha validáció sikertelen, `400 Bad Request`.
   - Ha a `correctAnswer` még nem állított (NULL), a tipp még **kiértékelésre vár** (`points: null`).

6. **Új scoring függvény: `scoreUpsetSpecial()`**
   - Helyét: `packages/backend/src/services/special-predictions.service.ts` (vagy dedikált helper).
   - Szignatúra: `scoreUpsetSpecial(userPicks: string[], eliminatedTeams: string[], choices: Choice[]): number`
   - Logika:
     ```typescript
     // Pseudo-code
     const eliminatedSet = new Set(eliminatedTeams)
     return userPicks
       .filter(pick => eliminatedSet.has(pick))
       .map(pick => choices.find(c => c.teamId === pick)?.points ?? 0)
       .reduce((sum, p) => sum + p, 0)
     ```
   - **Tiszta függvény**: nincs DB-hívás, unit testelhető (CLAUDE.md előírás).
   - Tesztelendő esetek:
     1. Mindkét pick kiesik → 36p (max).
     2. Egy pick kiesik → az adott csapat pontja.
     3. Egyik sem esik ki → 0p.
     4. Üres pick tömb → 0p.
     5. Érvénytelen teamId (nem az `options.choices` listában) → 0p (fallback).

7. **Seed: 1 új globális típus az Upset Special-lal**
   - Típus neve: `"Upset Special – kiesett csapatok"`
   - Leírás: `"Melyik 2 favoritnak végét ér a csoportköre? Válassz 2 csapatot, akik nem jutnak túl a csoportkörön. Pontérték csapatonként változó (2–18p)."`
   - `inputType: 'multi_team_weighted'`
   - `isGlobal: true`, `groupId: NULL`, `isActive: true`
   - **`deadline`: a torna első csoportmeccsének kickoff ideje** (placeholder: `<első csoportmeccs kickoff>`, mint az US-935-ben). Ez a határidő **közös az összes bonus tipp típussal** (US-935 statisztikai típusok, US-937 progression stage-ek): a "Torna tippek" tab a VB kezdetekor egységesen lezárul.
   - `points: NULL` (a tipikus `points` mező itt nem használatos — a súlyozás az `options`-ban van).
   - `options` JSONB:
     ```json
     {
       "maxPicks": 2,
       "minPicks": 2,
       "choices": [
         { "teamId": "<uuid-spain>", "points": 18 },
         { "teamId": "<uuid-france>", "points": 18 },
         ...24 csapat összesen a táblázat alapján
       ]
     }
     ```
   - Seed idempotens (mint az US-935-ben: `INSERT ... ON CONFLICT DO NOTHING` vagy conditional insert).
   - Az npm script: meglévő `npm run seed:tournament-types` vagy új `npm run seed:upset-special` (dönts az implementáció során).

8. **Unit tesztek (backend)**
   - `scoreUpsetSpecial()` legalább 5 case-el (lásd 6. pont).
   - Seed idempotencia (lásd US-935 minta).
   - Validációs tesztek: típus létrehozás érvényes/érvénytelen `options`-szel; tippelés 0/1/2 pick-kel; admin correctAnswer beállítása.

### Frontend

9. **Új komponens: `UpsetSpecialPicker.vue`**
   - Megjelenítés: 24 csapat **chip/toggle gombként**, mindegyiken:
     - Csapatzászló (ikonként, pl. `flag-es` Tailwind/custom icon set).
     - Csapatkód vagy rövid név (pl. `ESP`, `CAN`).
     - Pontérték szöveg: `18p`, `2p` — szürke, apró.
   - Kiválasztási logika: **checkbox-szerű** vagy **toggle-gomb-szerű** UI — az aktívak tükröznek.
   - **Pontosan 2 pick kötelező**: a submit gomb (`Add picks` / `Mentés`) **blokolt**, amíg `selected.length !== 2`; ha 1 vagy 0, tiltott állapot, tooltip: `"Válassz pontosan 2 csapatot"`.
   - Visuális feedback: aktív pick `bg-blue-100 ring-2 ring-blue-400`, inaktív `bg-gray-50 hover:bg-gray-100`.
   - Mobilon: grid layout, 3–4 oszlop, természetes törés.

10. **Integráció a `SpecialPredictionsTab.vue`-be**
    - Amikor az `inputType === 'multi_team_weighted'`, a tab megjelenít egy `<UpsetSpecialPicker />` komponenst az adott típushoz.
    - Határidő előtt: picker aktív, submit gomb engedélyez.
    - Határidő után: read-only megjelenítés (lásd 11. pont).

11. **Read-only megjelenítés határidő után**
    - Az 2 pick megjelenítése **chip-ek formátumában**, mindegyiken:
      - Zászló + csapatkód/név.
      - Pontérték.
      - Az `answer` (beadott tipp) megjelenítéséhez az `options.choices` array-ből kell kikeresni a pontértékeket.
    - Kiértékelés után (`points !== null`):
      - Minden pick mellett egy **zöld pipa** vagy **piros X** jelzi, hogy kiesett-e.
      - Az összpontszám a kártyán megjelenítendő (mint bármelyik más típusnál).

12. **Frontend unit teszt (`UpsetSpecialPicker.vue`)**
    - Vitest + Vue Test Utils.
    - Tesztek:

      2. 0 pick: submit blokkolt.
      3. 1 pick: submit blokkolt.

## Technikai megjegyzések

### Adatbázis

- Migration: új enum érték az `specialPredictionInputTypeEnum`-hez.
- Nem szükséges új tábla vagy új oszlop — az `options` JSONB már létezik az `specialPredictionTypes`-on.
- Az `answer: text` meglévő; JSON stringként tárolva (backward compatible).

### Backend Seed

```typescript
// packages/backend/src/scripts/seed-upset-special.ts (vagy ugyanez az US-935 seed-hez hozzáadva)
const upsetSpecialType = {
  name: 'Upset Special – kiesett csapatok',
  description: 'Melyik 2 favoritnak végét ér a csoportköre? Válassz 2 csapatot, akik nem jutnak túl a csoportkörön. Pontérték csapatonként változó (2–18p).',
  inputType: 'multi_team_weighted',
  isGlobal: true,
  isActive: true,
  deadline: '<első csoportmeccs kickoff>',
  options: {
    maxPicks: 2,
    minPicks: 2,
    choices: [
      { teamId: '<uuid-spain>', points: 18 },
      { teamId: '<uuid-france>', points: 18 },
      // ... 24 csapat
    ]
  }
}
```

Team ID lookup: a seed az `teams` tábla alapján kérdez (WHERE name = 'Spain' stb.), vagy a torna megadott csapatlistájából olvas (pl. EU/CA/LATAM/AFR csapatonként filtrálva).

### Service logika

```typescript
// packages/backend/src/services/special-predictions.service.ts
export function scoreUpsetSpecial(
  userPicks: string[],
  eliminatedTeams: string[],
  choices: Array<{ teamId: string; points: number }>
): number {
  const eliminatedSet = new Set(eliminatedTeams)
  return userPicks
    .filter(pick => eliminatedSet.has(pick))
    .map(pick => choices.find(c => c.teamId === pick)?.points ?? 0)
    .reduce((sum, p) => sum + p, 0)
}
```

Integrálás az `evaluateGlobalType()` vagy `evaluateSpecialType()`-ba:
```typescript
if (type.inputType === 'multi_team_weighted') {
  const userPicks = JSON.parse(prediction.answer)
  const eliminatedTeams = JSON.parse(type.correctAnswer ?? '[]')
  const choices = type.options?.choices ?? []
  const score = scoreUpsetSpecial(userPicks, eliminatedTeams, choices)
  return score
}
```

## Edge case-ek

| Helyzet | Elvárt viselkedés |
|---------|-------------------|
| Felhasználó 0 csapatot választ | Frontend submit blokkolt; backend 400 |
| Felhasználó 1 csapatot választ | Frontend submit blokkolt; backend 400 |
| Felhasználó 3 csapatot választ | Frontend UI megelőzi; backend 400 |
| Felhasználó ugyanazt a csapatot 2x választja | Frontend megelőzi; backend duplikátum-detektálás 400 |
| Felhasználó olyan csapatot választ ami nincs az `options.choices`-ban | Backend 400 |
| Admin olyan csapatot állít be `correctAnswer`-be ami nincs az `options.choices`-ban | Backend 400 |
| `correctAnswer` még nem állított (NULL) | Típus kiértékelésre vár; UI `points: null` jelez |
| Mindkét pick kiesik a csoportkörben | 36p jár (18 + 18) |
| Egy pick kiesik, a másik nem | Csak az egyik pontja jár (pl. 18p) |
| Egyik sem esik ki | 0p (nem null!) |
| A meglévő `team_select` típusok tippelése | Működik az eddig ismert módon; az Upset Special nem érinti |
| Határidő lejárt | Tippelés nem lehetséges; read-only chip-ek megjelennek |
| Seed futtatása 2x | Idempotens — duplikátum nincs |

## Kizárások

- **Nem** dinamikus pontérték a FIFA/aktuális ranking alapján — a 24 csapat és pontérték az Upset Special seed-ben hardcoded.
- **Nem** más torna (már csak ez az 1 típus a meglévő 5-höz: US-935).
- **Nem** automatikus pontszámítás a tényleges kiesésből — az admin manuálisan állítja be a `correctAnswer`-t a csoportkör után.
- **Nem** aggregált csoportbontás az Upset Special-hez (US-934 minta) — ez post-MVP feladatként kerülhet a backlogra.
- **Nem** élő statisztika vagy valós idejű frissítés az SSE-en.
- **Nem** a csapatviszonyok vagy csoportallokáció módosítása — csak az Upset Special típus.

## Nyitott kérdések

1. **User answer storage A vagy B:**
   - **A**: `answer: text` + `JSON.stringify()` (current choice, backward compatible).
   - **B**: Új `answer_json: jsonb` oszlop a `specialPredictions`-on (type-safe, de migrációigényes, és más inputType-okat is érinthetne).
   - **Ajánlás**: A opció — kisebb blast radius.

2. **Dinamikus maxPicks/minPicks:**
   - Jelenleg: hardcoded 2 (Upset Special = 2 pick).
   - Lehetőség: az `options`-ban paraméterezhető (1–3 vagy 1–N).
   - **Ajánlás**: Jelenleg hardcoded 2 (vagy `minPicks=maxPicks=2` az `options`-ban); post-MVP flexibilizálás ha szükséges.

3. **Auto-subscribe az Upset Special-ra:**
   - Az 5 globális típushoz (US-935) hasonlóan: automatikus feliratkozás minden új csoportra?
   - **Ajánlás**: Igen (következetesség az US-935-tel; manual opt-out később lehetséges).

4. **Csapatzászló a pickerben:**
   - Kötelező-e a flagga icon, vagy elég a csapatkód + név?
   - **Ajánlás**: Zászló + kód (vizuális attraktor, könnyebben azonosítható; icon set: `country-flag` vagy custom SVG).

5. **Határidő után megjelenítés-bontás (US-934 minta):**
   - Az Upset Special-hoz szintén szeretnénk aggregált bontást (pl. "8-an Spainre tippeltek")?
   - **Ajánlás**: Post-MVP; ennek az US-936-nak az elsődleges célja a tippelés és kiértékelés — az aggregálás később (US-934 sablonját követve).
