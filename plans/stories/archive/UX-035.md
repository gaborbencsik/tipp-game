# UX-035: Mérkőzések banner – hiányzó tippek 17:00→07:00 ablak szerint

## Háttér

A mérkőzések oldalon (`packages/frontend/src/views/MatchesView.vue`) egy sárga warning banner (77–83. sor) figyelmeztet: hány meccsen nincs még tippje a felhasználónak. Jelenleg az `untippedTodayCount` computed (717–726. sor) csak a **naptári mai napot** számolja (00:00 → 23:59 helyi idő).

Probléma: késő este nyitva, ha holnap reggel 6:00-kor van egy meccs (pl. amerikai meccsek VB alatt), az nem számít, így a felhasználó lemarad a tippelésről.

A backend `packages/backend/src/jobs/daily-match-review.job.ts` már helyes logikát alkalmaz — egy **14 órás ablak minden este 17:00-tól reggel 7:00-ig (Budapest)** — amely push notifikációkat küld az ebben az időablakban kezdődő meccsekre. Ez az ablak DST-tudatos és felöleli a hajnali kezdéseket.

Cél: a frontend banner számolása illeszkedjék a backend logikához, hogy a felhasználó konzisztens, teljes képet kapjon a határidőig hiányzó tippekről.

Duplikáció: ugyanez a logika szükséges a `packages/frontend/src/components/AppLayout.vue` sidebar badge-ében is (197–207. sor).

## Story

Mint **felhasználó**, szeretném, hogy a mérkőzések oldalon a warning banner **holnap reggel 7:00 előtt kezdődő meccseket is számolja**, hogy ne maradjak le a hajnali kezdéseken (pl. amerikai időzóna).

## Elfogadási kritériumok

### Közös composable és logika

1. **Új composable:** `packages/frontend/src/composables/useUntippedTodayCount.ts`
   - Export: `untippedTodayCount` computed, amely a backend `daily-match-review.job.ts` ablak-logikáját követi.
   - Paramétere: `matchesStore` és `predictionsStore` (Pinia store-ok).

2. **Időablak logika (Budapest timezone, DST-tudatos):**
   - Window start hour: **17:00 Budapest**, end hour: **07:00 Budapest (holnap)**.
   - Számítás:
     - Ha a jelenlegi Budapest-idő **< 07:00**, akkor az ablak az **előző nap 17:00 Budapest → ma 07:00 Budapest**.
     - Ha a jelenlegi Budapest-idő **>= 07:00 és < 17:00**, akkor az ablak a **mai 17:00 Budapest → holnap 07:00 Budapest** (jövőbeli).
     - Ha a jelenlegi Budapest-idő **>= 17:00**, akkor az ablak a **mai 17:00 Budapest → holnap 07:00 Budapest** (aktuális).
   - **Minden határidő UTC-ben tárolt** (`scheduledAt`), ezért az ablakot is UTC-re konvertálni kell.
   - **DST-tudatos:** `Intl.DateTimeFormat`-ot használni a Budapest-idő lekéréshez (ne `getTimezoneOffset()`).

3. **Hiányzó tipp feltételei (egy meccs csak akkor számít):**
   - `status === 'scheduled'` (nem finished, nem abandoned).
   - `scheduledAt` az aktív ablak-n belül: `>= windowStartUtc && < windowEndUtc`.
   - `scheduledAt > now` (még nem kezdődött el).
   - Nincs rá tipp: `!predictionsStore.predictionByMatchId(m.id)` (null vagy undefined).

### Kód duplikáció eltávolítása

4. **`MatchesView.vue` (717–726. sor):**
   - Az `untippedTodayCount` computed **törlése**.
   - Az új composable importálása: `const { untippedTodayCount } = useUntippedTodayCount(matchesStore, predictionsStore)`.
   - Megmarad az `isToday()` helper (711–715. sor) — más helyen is használják, nem a banner logikája része.

5. **`AppLayout.vue` (197–207. sor):**
   - Az `untippedTodayCount` computed **törlése**.
   - Az új composable importálása: `const { untippedTodayCount } = useUntippedTodayCount(matchesStore, predictionsStore)`.

### Banner szöveg

6. **UI szöveg marad:** `"{{ untippedTodayCount }} mérkőzésen még nem tippeltél ma — a határidő hamarosan lejár!"`
   - A "ma" szó marad (köznyelvben a "ma este + holnap reggel" együtt "ma" = 1 nap élménye).

### Tesztelés

7. **Vitest unit teszt:** `packages/frontend/src/composables/useUntippedTodayCount.test.ts`
   - Mock: `matchesStore`, `predictionsStore`, `Date.now` (faketime).
   - 8+ test case Budapest időzónában:
     1. **08:00 Budapest, mai 20:00 meccs, nincs tipp** → count = 1.
     2. **08:00 Budapest, holnap 06:00 meccs (window-ban), nincs tipp** → count = 1.
     3. **08:00 Budapest, holnap 09:00 meccs (window-on kívül)** → count = 0.
     4. **02:00 Budapest, ma 06:00 meccs (tegnap 17:00 → ma 07:00 window-ban), nincs tipp** → count = 1.
     5. **18:00 Budapest, ma 17:30 meccs már lezajlott** (`scheduledAt <= now`) → count = 0.
     6. **08:00 Budapest, mai 20:00 meccs, van tipp** → count = 0.
     7. **08:00 Budapest, mai 20:00 meccs `status !== 'scheduled'` (pl. finished)** → count = 0.
     8. **DST transition közeli eset** (opcionális, de jó: október utolsó vasárnap előtt/után) → ablak jól számol.
   - Faketime használat: `vi.useFakeTimers()` + `vi.setSystemTime()` → determinisztikus tesztek.
   - Mock-ok: matchesStore.matches (array), predictionsStore.predictionByMatchId (getter fv.).

8. **E2E teszt:** nem szükséges — kis scope, unit teszt elég.

## Technikai megjegyzések

### Konstansok

```typescript
const WINDOW_START_HOUR_BUDAPEST = 17;
const WINDOW_END_HOUR_BUDAPEST = 7;
const TIMEZONE_NAME = 'Europe/Budapest';
```

- Ezeket a composable-ben definiálni vagy kommentként hivatkozni a backend `daily-match-review.job.ts`-re.
- Ha létezik `packages/shared` workspace és van ott config modul, oda szervezhetők; ha nincs, duplikált konstansok + Backend référence megengedett.

### Függvény vázlat

```typescript
// Számítja az UTC-beli ablak határait (windowStartUtc, windowEndUtc).
function computeWindowBoundsUtc(now: number): { start: number; end: number } {
  // (1) Jelenlegi Budapest-idő lekérése Intl-vel
  // (2) Óra kinyerése
  // (3) Ablak-ágak szerinti logika
  // (4) Visszatérés UTC ms-ben
}

// Composable
export function useUntippedTodayCount(matchesStore, predictionsStore) {
  const untippedTodayCount = computed(() => {
    const now = Date.now();
    const { start, end } = computeWindowBoundsUtc(now);
    
    return matchesStore.matches
      .filter(m =>
        m.status === 'scheduled' &&
        m.scheduledAt >= start &&
        m.scheduledAt < end &&
        m.scheduledAt > now &&
        !predictionsStore.predictionByMatchId(m.id)
      ).length;
  });
  
  return { untippedTodayCount };
}
```

## Mit ne csináljunk

- Ne módosítsuk a backend `daily-match-review.job.ts`-t — az jó, ahogy van.
- Ne változtassuk a banner szövegét vagy stílusát — csak a számolási logika.
- Ne hozzunk létre új `packages/shared` csomagot kizárólag ezért — duplikált konstans + komment elfogadható.
- Ne nyúljunk a meccsek listájának szekció-rendereléseihez ("Ma" / "Holnap" csoportok) — azok külön `isToday()` logikát követnek.
- Ne hozzunk hozzá új API-hívást — minden adat már a Pinia store-okban van.
- Ne adjunk hozzá UI warning-eket vagy extra infó-buborékokat — csak a banner számlálása változik.
