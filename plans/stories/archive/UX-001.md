# UX-001: Tipp input UX javítások

> Epic: E14 – Bugok és stabilitás

**Story:**
Mint **felhasználó**, szeretném, hogy **a tipp beviteli mezők kényelmesebben használhatók legyenek**, hogy **gyorsabban és kevesebb kattintással tudjak tippet rögzíteni**.

**Elfogadási kritériumok:**
- [x] **Autosave debounce:** tipp automatikusan mentésre kerül 2 másodperccel az utolsó gombnyomás után (nincs szükség "Mentés" gombra kattintani)
- [x] **Fókusz → select:** mezőbe kattintáskor / tab-bal belépéskor a tartalom azonnal ki van jelölve (felülírható anélkül, hogy törölni kellene)
- [x] **Üres mező fókuszáláskor → 0:** ha a mező üres volt, fókuszáláskor `0` kerül bele és az ki is van jelölve
- [x] **Szám beütése → következő mező:** ha a user begépel egy számot (0-9), a fókusz automatikusan a másik (away/home) mezőre ugrik
- [x] A "Mentés" gomb maradhat fallback-ként manuális mentéshez
- [x] `saveStatus` visszajelzés ("Tipp elmentve!") az autosave után is megjelenik

**Technikai megjegyzések:**
- Debounce: `setTimeout` + `clearTimeout` az input event handlerben, 2000ms
- Select on focus: `input.select()` a `@focus` event handlerben
- Következő mező: `@keydown` figyelés `0-9` billentyűkre, `nextTick` után `nextInput.focus()` + `nextInput.select()`
- Érintett komponensek: `MatchesView.vue` inline tipp form, `MatchDetailView.vue` tipp form
