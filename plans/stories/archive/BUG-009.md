# BUG-009: Csoport-leaderboard 0 tippet mutat ha favoriteTeamDoublePoints aktív és még nincs lefutott meccs

## Tünet

A "VB Tippjáték 2026" csoport leaderboardján minden tagnál `0` tipp jelent meg, miközben az "Összesített" (globális) ranglistán ugyanazok a userek pozitív tipp-számokkal szerepeltek (pl. Skuli 72, Csukii 72, Bencso13 1). Ez a BUG-008 javítás után is megmaradt — más bug volt mögötte.

## Gyökérok

`packages/backend/src/services/group-leaderboard.service.ts` `useGroupPoints = true` ágában (ami akkor aktív, ha `scoringConfigId !== null` **vagy** `favoriteTeamDoublePoints = true`) a `predictionCount` a `groupPredictionPoints.id`-t számolta:

```sql
count(case when matches.id is not null then group_prediction_points.id end)
```

A `group_prediction_points` tábla rekordjai viszont csak akkor jönnek létre, ha egy meccsnek lett eredménye (`calculateAndSaveGroupPoints` futott rá). A "VB Tippjáték 2026" csoport `favorite_team_double_points = true` beállítással rendelkezik, és a VB ligában még egyetlen meccs sem ért véget (`finished_matches = 0`), ezért a `group_prediction_points` üres → minden user `predictionCount = 0`.

A globális ranglista ezzel szemben a `predictions.id`-t számolja, ami az összes leadott tippet (kiértékelttől függetlenül) lefedi.

## Fix

`predictionCount` mindkét ágon az **összes leadott** tippet számolja a `predictions.id`-n keresztül. A `totalPoints` és `correctCount` továbbra is a `groupPredictionPoints.points`-ot használja a group-points ágban — ezek logikailag csak kiértékelt tippekre értelmesek.

```diff
- predictionCount: count(case when matches.id is not null then group_prediction_points.id end)
+ predictionCount: count(case when matches.id is not null then predictions.id end)
```

Új unit teszt biztosítja, hogy a `useGroupPoints = true` ág `count(...)` kifejezései hivatkozzanak a `predictions.id`-re és **ne** a `groupPredictionPoints.id`-re.

## Elfogadási kritériumok

1. ✅ `favoriteTeamDoublePoints = true` csoportoknál a leaderboard `predictionCount` mezője az összes leadott tippet mutatja, akkor is ha még egy meccs sem ért véget.
2. ✅ `scoringConfigId` beállított csoportoknál ugyanez a viselkedés.
3. ✅ Globális scoring fallback ágban a viselkedés változatlan.
4. ✅ Backend unit tesztek zöldek (719/719).
