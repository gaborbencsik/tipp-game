# US-609: Liga filter csoportonként

> Epic: E6 – Csoportok

**Story:**
Mint **felhasználó**, szeretném **csoport létrehozásakor megadni, hogy melyik liga meccseit számolja a csoport**, hogy **egyértelmű legyen mire tippelünk a csoportban**.

**Kontextus:**
Ez a story az US-1301 (liga entitás bevezetése) után valósítható meg, mert a liga filter liga ID-kra hivatkozik. Amíg egyetlen liga van, a választás automatikus — de a rendszernek akkor is ki kell kényszerítenie a hozzárendelést.

**Elfogadási kritériumok:**
- [ ] `group_leagues` junction tábla: `groupId`, `leagueId` — melyik ligák meccsei számítanak a csoportban
- [ ] Csoport létrehozásakor **kötelező** legalább egy ligát kiválasztani
- [ ] Ha csak egy liga létezik, a frontend automatikusan kiválasztja (nincs felesleges lépés)
- [ ] `POST /api/groups` body bővül: `leagueIds: string[]` (min 1)
- [ ] `GET /api/groups/mine` válasz tartalmazza a csoport ligáit (`leagues: League[]`)
- [ ] `PUT /api/groups/:groupId/leagues` — liga filter utólagos módosítása (csak admin)
- [ ] `calculateGroupPoints()` figyelembe veszi a liga filtert: csak a filterben szereplő ligák meccseit számolja
- [ ] Ha a `group_leagues` tábla üres egy csoportnál (migrációs backward-compat), minden meccs számít
- [ ] Frontend: csoport létrehozás form-on liga multi-select (ha 1 liga van: rejtett, auto-kiválasztva)
- [ ] Frontend: csoport beállításokban liga filter szerkesztés (admin only)
- [ ] Unit tesztek: create with leagues, filter aktív, filter üres fallback, filter módosítás

**Függőség:** US-1301 (liga entitás), US-608 (calculateGroupPoints service)

**Komplexitás:** M
**Prioritás:** Should Have
