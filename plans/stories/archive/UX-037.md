# UX-037: Torna tippek – több helyes válasz támogatása holtverseny esetén

> Epic: E5 – Torna tippek – admin funkciók és pontozási szabályok

## Háttér

A torna-szintű "globális speciális tippek" (`team_select`, `player_select`, `dropdown`, `text` típusok) jelenleg egyetlen `correctAnswer` szöveget tárolnak. A `evaluateSpecialPrediction()` (`packages/backend/src/services/special-prediction-evaluation.service.ts`) ezt egzakt egyezéssel (normalize → case-insensitive, ékezet-mentes) hasonlítja a user tippjéhez.

Probléma: "Csoportkör – legtöbb gólt szerző csapat", "Gólkirály" (player_select) esetén előfordulhat **holtverseny** (több csapat/játékos ugyanannyi góllal). Jelenleg az admin csak egy csapatot/játékost rögzíthet, így a többi, ténylegesen helyes választ tippelő user 0 pontot kap.

Cél: az admin több helyes választ rögzíthet egy torna-tippnél holtverseny esetén. Minden olyan felhasználó megkapja a teljes pontot, aki bármelyik holtversenyben álló opcióra tippelt.

## Story

Mint **admin**, szeretném **több helyes választ rögzíteni egy torna-tippnél holtverseny esetén**, hogy **minden olyan felhasználó megkapja a teljes pontot, aki bármelyik holtversenyben álló opcióra tippelt.**

## Megoldási irány

- **Tárolás:** marad a meglévő `correctAnswer TEXT` mező. A mező két formát fogad el:
  - egyetlen string (mai viselkedés – backward compatible),
  - JSON-tömb: `["uuid1","uuid2",...]` (új – holtverseny támogatása).
- **Pontozás:** "bármely helyes = teljes pont". Ha a user tippje a tömb bármely elemével egyezik (a meglévő normalize szabály szerint), megkapja `maxPoints`-ot, egyébként 0.
- **Hatókör:** `team_select`, `player_select`, `dropdown`, `text`. `multi_team_weighted`, `all_groups_standing`, `bracket_progression` változatlan.

## Elfogadási kritériumok

### Backend (TDD – Red → Green → Refactor)

1. **evaluateSpecialPrediction() viselkedés**
   - Single-string `correctAnswer` (mai viselkedés) változatlanul működik.
   - JSON tömb (`["uuid1","uuid2"]` vagy szöveges opciók) `correctAnswer` esetén: ha user tippje **bármely** elemre normalize-egyezik → `maxPoints`, egyébként 0.
   - Üres tömb (`[]`) → 0 pont mindenkire (defenzív; admin oldalon validáljuk).
   - Tömb 1 elemmel → ekvivalens a single-string viselkedéssel.
   - Tömb whitespace / üres stringeket kiszűri a comparison-ből.

2. **setGlobalCorrectAnswer() validáció** (csak `team_select` / `player_select` / `dropdown` típusra)
   - Ha a `correctAnswer` JSON-tömbként parse-olható → minden elem nem-üres string.
   - `dropdown` esetén minden elem szerepeljen az `options` listában.
   - `team_select` / `player_select` esetén csak alak-ellenőrzés (UUID-szerű).
   - `text` típusnál szöveges opciókat fogad el.

3. **evaluateGlobalTypeSlice() változatlan szerkezetű** — csak a fenti pont-számítás bővül.

4. **Backend unit tesztek** (`packages/backend/tests/special-prediction-evaluation.service.test.ts`):
   - "any match in JSON array → full points (team_select)"
   - "any match in JSON array → full points (player_select)"
   - "any match in JSON array → full points (text)"
   - "any match in JSON array → full points (dropdown, options-aware)"
   - "no match in JSON array → 0 points"
   - "single-element JSON array equivalent to plain string"
   - "empty JSON array → 0 points"
   - "single string correctAnswer continues to work (BC)"

### Frontend – Admin UI

1. **AdminTournamentEvaluationView.vue** támogatja több helyes válasz megadását `text` / `dropdown` / `team_select` / `player_select` ágakban:
   - **team_select / player_select:** egy "Hozzáadás" gomb az aktuálisan kiválasztott elemet egy chip-listához adja; az "X" gomb chipenként eltávolít. Draft: `string[]`.
   - **text / dropdown:** szöveges input + "Hozzáadás" gomb, ugyanazzal a chip-listás kezeléssel.

2. **Mentéskor:**
   - 0 elem → toast hiba ("Először add meg a helyes választ").
   - 1 elem → single-string formátumban (BC: sima szöveg a `correctAnswer` mezőbe).
   - 2+ elem → JSON-tömbként (`JSON.stringify([id1, id2, ...])`).

3. **Kártya teteje:** "Helyes válasz beírva" badge megjelenik, ha `correctAnswer` nem-üres (single vagy JSON-tömb is OK).

4. **Mount-on:** az `answerDrafts`-be a JSON-tömb visszaalakítva `string[]`-ként, single-string esetén `[value]`-ként.

5. **Komponens teszt** (`AdminTournamentEvaluationView.test.ts`):
   - team_select chip-add/remove, single-element save → single-string PATCH body.
   - team_select 2 chip save → `JSON.stringify(array)` PATCH body.
   - Pre-existing JSON-tömb correctAnswer megjelenik chip-listaként mount-on.

### Frontend – User UI (Torna tippek nézet)

1. **TournamentTipsView.vue** kiértékelés után a "helyes válasz" felirat tömbös `correctAnswer` esetén minden elemet jelenít meg:
   - Egy elem → mai megjelenés.
   - Több elem → vesszővel elválasztva, vagy chip-listaként.

2. **Ha a user a tömb bármelyikére tippelt:**
   - "Eltaláltad" jelzés + `maxPoints` (backend már jól számolja).

3. **Komponens teszt** (`TournamentTipsView.test.ts`):
   - `correctAnswerLabels` többes lista → vesszős kijelzés.

### Validáció / Hibák

- [ ] Üres tömb (`[]`) PATCH → 400 "Helyes válasz nem lehet üres".
- [ ] Érvénytelen JSON (csonka) → backend single-string-ként kezeli (BC), frontend admin UI validál előtte.

## Mit ne csináljunk

- Ne adjunk hozzá új DB oszlopot vagy migration-t.
- Ne változtassuk a pontkalkulációt arányos osztásra (`maxPoints / N`) — marad "bármely helyes = teljes pont".
- Ne nyúljunk a `multi_team_weighted`, `all_groups_standing`, `bracket_progression` típusok logikájához.
- Ne adjunk súlyozást a többes helyes válaszokhoz.
