# US-613: Csoportos tippjeim nézet

> Epic: E6 – Csoportok

## User Story

Mint **csoporttag**,
szeretném **egy adott csoporton belül látni, mennyi pontot értek az egyes meccs-tippjeimért a csoport saját pontrendszere szerint**,
hogy **megértsem, miért tér el a csoportos ranglistán elért pontszámom a globális "Saját tippek" nézetben látott értéktől**.

---

## Kontextus

A `predictions` táblán a `pointsGlobal` mező a globális pontrendszer alapján számított értéket tárolja. Ha egy csoport egyedi `scoring_config`-ot használ, vagy engedélyezve van a `favoriteTeamDoublePoints` funkció, a csoport-specifikus pontok a `group_prediction_points` táblában kerülnek tárolásra (`predictionId + groupId → points`). A jelenlegi "Saját tippek" (`/app/predictions`) nézet csak a `pointsGlobal` értéket mutatja, és nem tükrözi a csoport-szintű eltéréseket.

A `GroupDetailView.vue` jelenleg három tabot tartalmaz: **Ranglista**, **Speciális tippek**, **Beállítások**. Ez a story egy negyedik tabot vezet be: **Tippjeim**.

---

## Elfogadási kritériumok

### Backend

- [ ] Új endpoint: `GET /api/groups/:groupId/my-predictions`
  - Auth: `authMiddleware` szükséges (bejelentkezett felhasználó).
  - Tagság ellenőrzés: a kérelmező csak akkor kap adatot, ha tagja a csoportnak (403 egyébként).
  - 404, ha a csoport nem létezik vagy `deleted_at IS NOT NULL`.
  - Csak `status = 'finished'` meccsekhez tartozó tippeket ad vissza (nem kiértékelt tippek nem szerepelnek).
  - A válasz `group_prediction_points` tábla alapján tartalmazza a csoport-specifikus pontszámot (`groupPoints`). Ha a sorban nincs bejegyzés a `group_prediction_points` táblában (pl. a csoport pontrendszere megegyezik a globálissal és a csoport-specifikus pontszámítás nem futott le), a `pointsGlobal` értékkel helyettesítendő (`COALESCE`-logika).
  - A válasz tartalmazza a `doubledByFavorite: boolean` mezőt — `true`, ha a `group_prediction_points` sorban a pontszám pontosan kétszerese annak, amit az alap (nem doubled) szabályok adnának. Lásd a Technikai megjegyzések részt.
  - Rendezés: `match.kickoffTime DESC` (legutóbbi meccsek elöl).
- [ ] Válasz formátum:
  ```json
  {
    "data": [
      {
        "predictionId": "uuid",
        "matchId": "uuid",
        "kickoffTime": "2026-06-14T18:00:00Z",
        "homeTeam": { "id": "uuid", "name": "Argentína", "flagUrl": "..." },
        "awayTeam": { "id": "uuid", "name": "Franciaország", "flagUrl": "..." },
        "result": { "homeScore": 2, "awayScore": 1 },
        "prediction": { "homeScore": 2, "awayScore": 1 },
        "groupPoints": 7,
        "doubledByFavorite": true
      }
    ]
  }
  ```
- [ ] Unit tesztek (backend):
  - Tag kap adatot; visszaad csak `finished` meccsekhez tartozó tippeket.
  - Nem tag 403-at kap.
  - Nem létező csoport 404-et kap.
  - Ha nincs `group_prediction_points` bejegyzés, `pointsGlobal`-t ad vissza `groupPoints`-ként.
  - `doubledByFavorite: true` a megfelelő esetben.
  - Rendezés: legutóbbi meccs az első.

### Frontend

- [ ] A `GroupDetailView.vue`-ban új, negyedik tab: **Tippjeim** (a "Beállítások" tab elé kerül).
- [ ] A tab tartalma **csak akkor töltődik be**, amikor a felhasználó tényleg megnyitja azt (lazy load, `onMounted` helyett a tab-switch handlerben).
- [ ] Új Pinia state a `groups.store.ts`-ben: `myGroupPredictionsMap: ref<Record<string, GroupPredictionRow[]>>({})`.
- [ ] Új API client metódus: `api.groups.myPredictions(token, groupId)`.
- [ ] Minden tipp egy kártyán jelenik meg, amely tartalmazza:
  - A mérkőzés két csapatának nevét és zászlaját.
  - A mérkőzés dátumát (lokalizált, rövid formátum, pl. "2026. jún. 14.").
  - A végleges eredményt (pl. `2 – 1`).
  - A felhasználó tippjét (pl. `2 – 1`).
  - A csoportban szerzett pontszámot, vizuálisan kiemelve (pl. `+7 pont`).
- [ ] Ha `doubledByFavorite === true`, a pontszám mellett egy `×2` jelvény (`badge`) jelenik meg, jelezve a kedvenc csapat duplázást.
- [ ] Ha a listában nincs elem (a felhasználó még nem adott le tippet, vagy egy meccs sem fejeződött be), üres állapot üzenet jelenik meg: `"Még nincs kiértékelt tipped ebben a csoportban."`.
- [ ] Ha az API kérés sikertelen, hibaüzenet jelenik meg: `"Nem sikerült betölteni a tippeket."` — az oldal többi része érintetlen marad.
- [ ] Unit tesztek (frontend):
  - A lista renderel predikciós kártyákat a helyes adatokkal.
  - A `×2` badge megjelenik, ha `doubledByFavorite === true`.
  - A `×2` badge nem jelenik meg, ha `doubledByFavorite === false`.
  - Üres állapot üzenet jelenik meg, ha az adat tömb üres.
  - Az API hiba kezelve van (hibaüzenet, nincs crash).
  - A tab-tartalom nem töltődik be a tab megnyitása előtt (lazy load ellenőrzés).

## Edge case-ek

| Helyzet | Elvárt viselkedés |
|---------|-------------------|
| A csoport nem használ egyedi scoring_config-ot és nincs favoriteTeamDoublePoints | `groupPoints` = `pointsGlobal`, `doubledByFavorite: false` |
| A `group_prediction_points` bejegyzés hiányzik egy tipphez | `groupPoints` a `pointsGlobal` értékét veszi fel |
| A felhasználó nem adott le tippet egyetlen befejezett meccsre sem | Üres lista, üres állapot üzenet |
| Még egy meccs sem fejeződött be a csoportban | Üres lista, üres állapot üzenet |
| A felhasználó elhagyja a csoportot, majd megpróbálja elérni az endpointot | 403 |
| A csoport soft-deleted | 404 |

## Technikai megjegyzések

- **`doubledByFavorite` meghatározása:** A `group_prediction_points` tábla jelenleg csak a végső pontszámot tárolja. A `doubledByFavorite` flag kiszámításához a backend a következő logikát alkalmazza: lekérdezi a meccs alap pontszámát ugyanazon scoring config alapján (duplázás nélkül), majd összehasonlítja a tárolt `group_prediction_points.points` értékkel. Ha pontosan a kétszerese, `doubledByFavorite: true`. Alternatív megközelítés: a `group_prediction_points` táblára egy `doubled_by_favorite BOOLEAN DEFAULT FALSE` oszlop felvétele (migrációval) — ez tisztább és pontosabb. A megvalósító dönt; ha az oszlop felvétele mellett dönt, frissíteni kell a pontszámítás (`scoring.service.ts`) hívási helyeit is.
- Az endpoint a meglévő `assertGroupMember` helper függvénnyel ellenőrzi a tagságot (már létezik a `group.service.ts`-ben).
- Érintett fájlok:

| Réteg | Fájl | Változás |
|-------|------|---------|
| Backend | `packages/backend/src/routes/groups.ts` | Új route: `GET /:groupId/my-predictions` |
| Backend | `packages/backend/src/services/group.service.ts` | Új service függvény: `getMyGroupPredictions(groupId, userId)` |
| Backend | `packages/backend/src/types/index.ts` | Új `GroupPredictionRow` interface |
| Frontend | `packages/frontend/src/api/index.ts` | `api.groups.myPredictions(token, groupId)` |
| Frontend | `packages/frontend/src/stores/groups.store.ts` | `myGroupPredictionsMap` state + fetch action |
| Frontend | `packages/frontend/src/views/GroupDetailView.vue` | Negyedik tab + lazy load trigger |
| Frontend | `packages/frontend/src/components/groups/MyGroupPredictionsList.vue` | Új lista komponens |
| Frontend | `packages/frontend/src/types/index.ts` | `GroupPredictionRow` interface |

- A `scoring.service.ts` `calculatePoints()` függvénye pure marad — nem kap új felelősséget.
- Typecheck (`tsc --noEmit`) mindkét package-ben tiszta kell legyen.
