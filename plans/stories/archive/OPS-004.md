# OPS-004: Virtual points polling gate – csak élő meccs alatt, 60 mp

## Probléma

A `GroupDetailView.vue` "Tippjeim" tabján a `virtualPoints` endpoint **30 másodpercenként** lekérdezésre kerül, amíg a user a tabon van — függetlenül attól, hogy van-e éppen folyamatban lévő meccs. Ez a fő egress-szivattyú a Supabase Free Plan kvótán (8.7 / 5 GB egress, 174%).

**Számszerűen:** napi 24 órából tipikusan 2-4 órányi időablakban van élő meccs. A maradék 20+ órában a polling **zéró új információt** hoz vissza (a virtuális pontszám csak élő meccs gólváltozására változik), miközben minden hívás teljes choice‐ranglistát hoz vissza per user, per group.

A backend payload mérete + 30 mp-es ráta + N user × M perc tab-aktivitás = **napi GB nagyságrendű felesleges egress**.

---

## Story

Mint **felhasználó és platform-üzemeltető**, szeretném, hogy **a virtuális pontszám polling csak akkor fusson, ha legalább egy meccs éppen folyamatban van, és akkor is csak 60 mp-enként**, hogy **ne pazaroljunk Supabase egress kvótát olyan adatokra, amik úgyis változatlanok**.

---

## Elfogadási kritériumok

### A — Live-match gate

1. [ ] A `GroupDetailView.vue` polling **csak akkor indul el**, ha mindkét feltétel teljesül:
   - `activeTab === 'my-predictions'`
   - Van legalább egy meccs `status === 'live'` állapotban a `useMatchesStore().matches` listában
2. [ ] Ha nincs élő meccs, a polling **nem indul el**, de **egyszer** lekéri a virtualPoints-t tabra váltáskor, hogy a UI ne legyen üres
3. [ ] Ha az élő meccs befejeződik (státusz `live` → `finished`), a polling **automatikusan leáll** (Vue `watch` reaktívan reagál)
4. [ ] Ha új meccs lép `live` állapotba miközben a user a tabon van, a polling **automatikusan elindul**

### B — Polling interval

1. [ ] Az interval **30 mp-ről 60 mp-re** változik
2. [ ] Konstansként deklarálva a komponensen belül (`const VIRTUAL_POINTS_POLL_INTERVAL_MS = 60_000`) — ne legyen mágikus szám

### C — Visibility gate

1. [ ] Ha `document.hidden` (a tab nincs előtérben, pl. zsebben a telefon, vagy másik fülön), a polling **leáll**
2. [ ] Visszatérés esetén (`visibilitychange`) a feltételek (tab + élő meccs) újraértékelődnek, és ha kell, **újraindul**
3. [ ] `onUnmounted`-ben minden `visibilitychange` listener takarítva

### D — Tesztek

1. [ ] **Unit teszt** egy kihelyezett `hasAnyLiveMatch(matches)` helperre: visszaadja `true`-t ha legalább egy meccs `status === 'live'`, különben `false`-t (üres tömb, csak `scheduled`, csak `finished`, vegyes scheduled+finished esetekre)
2. [ ] **Komponens-szintű teszt** (`GroupDetailView.test.ts` vagy hasonló): mock store-ral és fake timer-rel ellenőrzi:
   - Élő meccs nélkül a tab aktiválás után csak **1** API hívás megy ki (a kezdeti fetch), 60s elteltével **nem** lesz újabb
   - Élő meccs esetén 60s-enként új hívás megy
   - `document.hidden = true` esetén a polling leáll
3. [ ] Minden meglévő teszt zöld marad

### E — Memóriaszivárgás megelőzés

1. [ ] `onUnmounted` hook eltakarítja a timer-t **és** a `visibilitychange` listenert
2. [ ] `watch` stop handlerek meghívva, ha külön regisztráltunk

## Technikai megjegyzések

### Helper kihelyezés

```typescript
// packages/frontend/src/composables/useLiveMatchPolling.ts (új fájl) — opcionális
// VAGY: helper függvény a GroupDetailView-ban, ha csak itt használjuk

import type { Match } from '../types/index.js'
export function hasAnyLiveMatch(matches: ReadonlyArray<Match>): boolean {
  return matches.some(m => m.status === 'live')
}
```

A típus (`MatchStatus = 'scheduled' | 'live' | 'finished' | 'cancelled'`) már létezik (`packages/frontend/src/types/index.ts:21`).

### Várt kód-vázlat (`GroupDetailView.vue`)

```typescript
const VIRTUAL_POINTS_POLL_INTERVAL_MS = 60_000

const matchesStore = useMatchesStore()
const hasLiveMatch = computed(() => hasAnyLiveMatch(matchesStore.matches))

watch(
  [activeTab, hasLiveMatch, () => document.hidden],
  ([tab, live, hidden]) => {
    const shouldPoll = tab === 'my-predictions' && live && !hidden
    if (shouldPoll) startVirtualPointsPolling()
    else stopVirtualPointsPolling()
  }
)

function onVisibilityChange(): void {
  if (document.hidden) stopVirtualPointsPolling()
  else if (activeTab.value === 'my-predictions' && hasLiveMatch.value) {
    void fetchVirtualPoints()
    startVirtualPointsPolling()
  }
}

onMounted(() => document.addEventListener('visibilitychange', onVisibilityChange))
onUnmounted(() => {
  document.removeEventListener('visibilitychange', onVisibilityChange)
  stopVirtualPointsPolling()
})
```

## Várt egress-megtakarítás

| Időablak | Jelenlegi (30s polling) | Új (60s + live gate) | Megtakarítás |
|----------|-------------------------|-----------------------|--------------|
| Élő meccs alatt (~3h/nap) | 360 hívás/h × user | 60 hívás/h × user | 83% (intervallum) |
| Nincs élő meccs (~21h/nap) | 360 hívás/h × user | **0** | **100%** |
| Tab nincs előtérben | folyamatos | **0** | **100%** |

Becsült total: **~85-90% csökkenés** ezen az endpointon, ami a fő egress-szivattyú jelenleg.
