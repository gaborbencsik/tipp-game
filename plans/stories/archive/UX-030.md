# UX-030: Match Detail – Transfermarkt csapatértékek blokk

> **Status:** Implementálva 2026-06-10. A "numerikus erőviszony chip" elem törölve a végleges design során (kérésre csak a konkrét formázott értékek jelennek meg). A jobb felső "Transfermarkt ↗" link szintén törölve (a csapatnevek kattinthatóak). Csapatnév-link formátum: `/<slug>/startseite/verein/<id>` (a sima `/-/verein/<id>` nem mindig oldódik fel jól).

## Leírás

Mint **bejelentkezett felhasználó**, szeretném **a match detail oldalon (MatchDetailView.vue), az insights (Polymarket elemzés) ALATT egy önálló vizuális blokkban látni mindkét csapat aktuális Transfermarkt szerinti keret-piaci értékét horizontális sávokkal összehasonlítva**, hogy **gyors objektív viszonyítási pontom legyen a meccs összetételéről és erőviszonyáról — még az AI elemzés olvasása előtt**.

## Jelenlegi helyzet

A `teams.squad_market_value` mező és Transfermarkt szinkronizáció már rendelkezésre áll (US-1406 kész). A `MatchDetailView.vue`-ban a Polymarket insights blokk (`data-testid="insights-section"`) jelenleg az oldalnak ez a vége.

## Elfogadási kritériumok

### Backend — API kiterjesztés

- [x] A `GET /api/matches` és `GET /api/matches/:id` endpoint válaszában mindkét csapat (`homeTeam`, `awayTeam`) kiegészült egy `marketValueEur: number | null` és `transfermarktId: number | null` mezővel.
  - A `marketValueEur` közvetlenül az adatbázis `teams.squad_market_value` mezőjéből jön (egész euróban tárolva — a Transfermarkt API `currentMarketValue.value` mező közvetlen átvétele).
  - A `transfermarktId` közvetlenül a `teams.transfermarkt_id` mezőből.
- [x] Esetkezelés mind a négy esetre (mindkét van / csak hazai / csak vendég / egyik sincs) lefedve unit testtel.

### Frontend — megjelenítés a Match Detail oldalon

- [x] Egy új `<div data-testid="market-values-section">` blokk a `MarketValuesBar.vue` komponensben, közvetlenül a Polymarket odds bar UTÁN, közös reveal-gate alatt.
- [x] A blokk része ugyanannak a reveal-gate-nek mint a Polymarket bar (egyetlen "Elemzés felfedése" gomb fedi fel egyszerre mindkettőt)
  - A wrapper div mindkét blokkot fedi
  - A blur class (`blur-sm select-none pointer-events-none transition-[filter] duration-300`) ugyanazra vonatkozik
  - Az `useInsightsRevealStore` ugyanaz a flag, nincs külön state
- [x] **Vizuális minta (Polymarket-stílusú):**
  - Fejléc: "Csapatértékek" (uppercase, `text-xs font-medium text-gray-500 tracking-wide`). Jobb felső sarok üres (a csapatnevek kattinthatók).
  - Wrapper: `bg-gray-50 border border-gray-200 rounded-lg p-4` (azonos az `MatchOddsBar.vue` stílusával)
  - 2 sor, soronként: csapat zászló (16×12 px) + csapatnév (vagy 3-betűs rövidítés mobilon <375px) + horizontális bar + érték a jobb oldalon
  - Mindkét bar szélessége a csapat értékének aránya a két érték összegéhez képest (`home / (home + away) * 100%`, ill. fordítva) — lineáris arány, nem clamp-elt
  - Hazai sor: `bg-blue-500` (konzisztens a Polymarket home színével), vendég sor: `bg-emerald-500` (konzisztens a Polymarket away színével)
  - A bar mögött halvány track (`bg-gray-100`), hogy a 0-közeli érték is látható legyen
- [~] **Numerikus erőviszony chip:** ELHAGYVA — kérésre csak a konkrét formázott értékek (`100 M €`, `1,2 mrd €`) jelennek meg, „X× erősebb" indikátor nincs.
- [x] **Részleges/üres adat kezelése:**
  - Egyik csapat `marketValueEur` null → annál a sornál `—` jelenik meg az érték helyén, üres bar-track
  - `marketValueEur` mindkettőnél null → a teljes "Csapatértékek" blokk rejtve (`v-if`), a Polymarket bar továbbra is látható
- [x] **Csapatnév linkek (ha a transfermarktId megvan):**
  - A csapatnév link: `https://www.transfermarkt.com/<slug>/startseite/verein/{transfermarktId}` (`target="_blank"`, `rel="noopener noreferrer"`). A `<slug>` a csapatnévből generálódik (lowercase, ékezetek eltávolítva, nem alfanumerikus karakterek `-`-re cserélve).
  - Ha a transfermarktId null, a csapatnév sima szöveg (nem link)
- [x] **Forrás-attribution:** a blokk alján apró szürke szöveg (`text-xs text-gray-500`): `Forrás: Transfermarkt` (sima szöveg, link nélkül).
- [x] **Accessibility:**
  - Minden bar `role="img"` + `aria-label="<csapatnév> piaci értéke <formázott érték>"` (pl. `"Magyarország piaci értéke 850 M €"`)
  - A csapatnév/rövidítés látszik a sor elején — nem csak szín különbözteti meg a két oldalt
- [x] **Mobil (<375px):**
  - Csapatnév helyén 3-betűs rövidítés (`shortCode` mező, pl. `HUN`, `GER`)
  - `text-xs` font-size
  - Layout egy soros marad (zászló + rövidítés + bar + érték)
- [x] **Skeleton:** nincs külön loading state, az adatforrás egyből eljön a match detail fetch-sel
- [x] Vitest teszt a komponensre:
  - Mindkét érték megvan → mindkét sor + bar arányosan — exakt sávszélesség ellenőrzés (70/30 → `width: 70%` / `width: 30%`)
  - Egyik null → `—` az adott sornál, üres bar
  - Mindkettő null → blokk rejtve (`v-if`)
  - Transfermarkt linkek generálása (URL helyes-e, slug + id formátum, ékezet stripping)
  - Reveal-gate: szülő nézet tesztben (MatchDetailView) ellenőrizve, a market-values blokk egy reveal-gate alatt van a Polymarket bar-ral

### Formázás

- [x] Új utility függvény: `formatMarketValueEur(eur: number, locale: 'hu' | 'en'): string` — `packages/frontend/src/utils/formatMarketValue.ts`
  - **HU (kisbetű, sportsajtó konvenció):**
    - `1_200_000_000` → `"1,2 mrd €"`
    - `850_000_000` → `"850 M €"`
    - `45_000_000` → `"45 M €"`
    - `0 < x < 1_000_000` → `"<1 M €"`
    - `0` → `"0 €"`
  - **EN (Transfermarkt konvenció):**
    - `1_200_000_000` → `"€1.2B"`
    - `850_000_000` → `"€850M"`
    - `0` → `"€0"`
  - Szóköz a szám és az egység között HU-ban, közvetlen összevonás EN-ben
- [x] Unit teszt: határértékek (0, <1M, 1M, 999M, 1mrd, 999mrd), HU + EN lokalizációval

## UX-Tervezés

### Vizuális minta

Ugyanaz a design pattern, mint az `MatchOddsBar.vue`:

1. **Szegély és háttér:** `bg-gray-50 border border-gray-200 rounded-lg p-4`
2. **Fejléc:** `text-xs font-medium text-gray-500 tracking-wide uppercase` – "Csapatértékek" | jobbra link "Transfermarkt ↗"
3. **Sor szerkezet (2× sor):**
   ```
   [Zászló 16×12]  [Név/Rövidítés]  [████████░] [érték]
   [Zászló 16×12]  [Név/Rövidítés]  [███░░░░░░] [érték]
   ```
4. **Bar-szín:**
   - Hazai: `bg-blue-500` (primary szín, Polymarket konzisztencia)
   - Vendég: `bg-emerald-500` (secondary szín, Polymarket konzisztencia)
   - Track háttér: `bg-gray-100`
5. **Chip (ha >1.5×):** ` Badge komponens vagy inline `<span>`: `"2,4× erősebb"` – vörös/narancssárga háttér, fehér szöveg, `text-xs` font

### Reveal-gate integráció

- A teljes insights + market values blokk egy közös `<div>` alatt az `:class` reveal logikával
- Középre pozicionált "Mutasd az insightokat" gomb mindkét blokkot fedi el `blur-sm select-none pointer-events-none transition-[filter] duration-300` class-szal
- Gomb helyezése: a kombinált blokk középpontja, nem csak az odds bar fölött
- Az `useInsightsRevealStore` ugyanaz a flag, nincs külön state

### Mobil (<375px)

- Csapatnév helyén 3-betűs `code` mező (`HUN`, `GER`, stb.)
- Font-size: `text-xs`
- Padding/gap finomítás, hogy a sor még szűk képernyőn sem törik meg

### Részleges/üres adat

- **Egyik csapat null:** érték helyén `—`, üres bar-track (halvány szín), chip nincs
- **Mindkettő null:** a teljes blokk nem jelenik meg (`v-if="showMarketValues"`), nem marad üresen hagyott terület

### Accessibility

- **Vizuális információ szöveg formában:**
  - Zászló előtt csapatnév vagy rövidítés
  - Chip szövege (`"2,4× erősebb"`) olvasható, nem csak szín
- **ARIA:**
  - Bar: `role="img"` + `aria-label="<csapatnév> piaci értéke <formázott érték>"` – pl. `"Magyarország piaci értéke 850 M €"`
  - Link: `rel="noopener noreferrer"` a Transfermarkt URL-eken

## Technikai megjegyzések

- **Backend:** A `GET /api/matches/:matchId` lekérdezésben a `teams` tábla JOIN már megvan (`home_team_id`, `away_team_id`), csak az `marketValueEur` és `transfermarktId` mezőket kell az SELECT-hez hozzáadni. Nincs külön endpoint szükséges.
- **Frontend komponens:** Egy új `MarketValuesBar.vue` (vagy hasonló) mini-komponens, amely Props-ként kapja:
  - `homeTeam: { name: string; code: string; marketValueEur: number | null; transfermarktId: number | null }`
  - `awayTeam: { name: string; code: string; marketValueEur: number | null; transfermarktId: number | null }`
  - `isRevealed: boolean` (reveal-gate flag az `useInsightsRevealStore`-ból)
- **Formázás:** Utility függvény `formatMarketValueEur` a `packages/frontend/src/utils/` alatt
- **DB:** Nincs migráció szükséges — a `teams.squad_market_value` és `teams.transfermarkt_id` már megvan
- **Sync:** A Transfermarkt szinkronizáció (`transfermarkt.service.ts`) már naponta frissíti az értékeket — nincs módosítás

## Nem tartalmazza

- Játékos szintű piaci értékek (ki a legértékesebb tag a kezdő 11-ben) — külön story
- Történeti grafikon vagy érték-trend (értékváltozás az időben)
- Manuális admin override a megjelenített értékre
- Push értesítés értékváltozáskor
- A Match Pulse szekcióban való megjelenés (azt az US-1307 fedi le, ez attól független)
- Logaritmikus skála — lineáris arány, az extrém eltérést a numerikus chip kommunikálja

## Komplexitás

**S** — Backend: 1 endpoint kiterjesztés (SELECT 2 mező hozzáadása + 4-ágú return logika) + unit teszt. Frontend: 1 mini-komponens (2 sor + chip + reveal-gate) + 1 formázás util + szülő view módosítása (Polymarket wrapper kiterjesztése, gomb pozicionálása). A `squad_market_value` és Transfermarkt sync már kész.
