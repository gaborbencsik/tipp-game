# UX-030: Pontozási szabályzat modal – mobil nézet finomhangolás

## Vezérelv

**Mobil-first olvashatóság.** A szabályzatnak telefonon ugyanolyan kényelmesen olvashatónak kell lennie, mint asztali nézetben — a táblázatos meccs-pontozás szekció jelenleg **nem fér ki** kis képernyőn, a "Példa" oszlop tartalma elcsúszik vagy levágódik.

## Leírás

Mint **mobil felhasználó** szeretném, hogy **a Pontozási szabályzat modalban a meccs-pontozás szekció ne vágódjon le** és **kényelmesen olvashassam a példákat**, hogy **megértsem a pontszámítás logikáját anélkül, hogy oldalra kellene scrolloznom vagy desktopon kelljen újranéznem**.

## Jelenlegi helyzet

A `ScoringExplainerModal.vue` (`packages/frontend/src/components/ScoringExplainerModal.vue`) három szekcióból áll:

1. **Meccs-pontozás** — 3-oszlopos `<table>` (`Eset` / `Pontok` / `Példa`)
2. **Torna-pontozás** — kártyás (responsive)
3. **Statisztika-pontozás** — kártyás (responsive)

A 2. és 3. szekció már most **mobil-barát kártya layout**-ot használ (lásd sorok `170-188` és `194-208`), de az 1. szekció **`<table>`-t** alkalmaz, ami szűk viewportban (~360-414 px) megtörik:

### Konkrét tünetek (csatolt screenshot, ~390 px viewport)

- **"Példa" oszlop tartalma levágódik:**
  - "Eredmény: 3-1" → **"Eredmé"** (második sor a viewporton kívülre kerül)
  - "összes pontszám ×2" → **"összes po"**
  - "hazai 11-esekkel" cramped, függőlegesen széttöredezik
- **"Hosszabbítás/tizenegyes kimenetel bónusz (egyenes kiesésben)"** szöveg jelenleg túl hosszú a `Eset` oszlopban → 4 sorra törik
- A táblázat overflow-x **nem scrollozható**, a tartalom egyszerűen levágódik a `rounded-xl` konténerből
- A `w-20` fix szélesség a `Pontok` oszlopon redundáns helyet foglal mobilon

## Hatókör (scope)

### Benne van

1. **Meccs-pontozás szekció átalakítása mobilon kártyás layoutra** (a torna/statisztika szekciók mintájára), **desktopon marad a táblázat** (≥640 px / `sm` breakpoint)
2. Az ugyanazon Vue komponensben (`ScoringExplainerModal.vue`) megoldjuk – nem készítünk új komponenst
3. Tailwind responsive utility-kkel (`sm:hidden` / `hidden sm:table` minta)
4. **i18n stringek nem változnak** – a meglévő kulcsok használatával
5. Vitest unit teszt frissítése: a kártyás layout elemek (`data-testid="match-rule-card"`) elérhetők mobil viewport simulation alatt
6. **Vizuális regression**: a desktop nézet **pixelszinten ne változzon** (CSS scope check)

### Nincs benne (külön story / nem kell)

- A torna és statisztika szekciók átalakítása (már jók)
- Új scoring szabályok hozzáadása
- A modal nyitás/zárás logikája
- Fordítások módosítása

## Elfogadási kritériumok

### 1. Layout viselkedés breakpointok szerint

| Viewport | Meccs-pontozás megjelenítés |
|---|---|
| `< 640px` (mobil) | **Kártyás layout** – minden szabály egy `rounded-lg border` kártyában: bal oldalon szabály neve + példa, jobb oldalon a pontok pill |
| `≥ 640px` (`sm`+) | **Jelenlegi 3-oszlopos `<table>`** változatlan |

A váltás Tailwind `sm:` breakpoint-ekkel (`sm:hidden` és `hidden sm:block`) – nincs JavaScript viewport-detection.

### 2. Mobil kártya struktúra (mintát ad a meglévő torna/stats kártya)

```
┌──────────────────────────────────────────┐
│  Helyes kimenetel (1-X-2)         [ 1 ]  │
│  Te: 2-0, Eredmény: 3-1                  │
└──────────────────────────────────────────┘
```

- Felső sor: `flex items-start justify-between gap-3`, **bal**: szabály neve (`text-sm font-medium`), **jobb**: pontok pill (változatlan stíluskép)
- Alsó sor (a kártya szélességében): példa szöveg `text-xs text-gray-500 mt-1.5`
- Multiplier szabály (`favoriteTeamMultiplier`) esetén a "(aktív: ...)" zárójeles megjegyzés a szabály neve alá kerül, **a példa fölé** (`text-xs italic text-gray-500 mt-0.5`)

### 3. CSS finomhangolás (a kártyás verzióban)

- `border border-gray-200` + `rounded-lg` (mint a torna szekció kártyák)
- `bg-white` + `hover:bg-gray-50` átmenet (konzisztencia a többi szekcióval)
- `p-3` padding
- `mb-2 last:mb-0` egymás közti gap
- Pill (pontok) változatlan: `inline-flex min-w-[2rem] items-center justify-center rounded-full px-2.5 py-0.5 text-sm font-bold tabular-nums` + `matchPillClass(rule)`

### 4. Desktop táblázat változatlan

- A jelenlegi `<table>` block kerül `<div class="hidden sm:block">` wrappelésbe (vagy `sm:` változatban)
- Pixelszinten ugyanúgy nézzen ki ≥640 px-en, mint most
- Vizuálisan ellenőrizve Chrome DevTools-ban: `390px` (iPhone 14 Pro), `414px` (iPhone Plus), `640px` (`sm` breakpoint), `768px` (`md`), `1024px` (`lg`)

### 5. A "matchScoringNote" (záró italic szöveg)

A "A bónuszok stackelhetők — egy meccsen több is járhat." szöveg a szekció **alján** marad – nem változik.

### 6. Tesztek

- **Unit (Vitest):** `ScoringExplainerModal.spec.ts` frissítése
  - Új teszt: mobil viewport simulation (`window.matchMedia` mock vagy CSS class assertion) – minden `MATCH_RULES` szabály megjelenik **mind a két layoutban** (mobil kártya + desktop table), Tailwind `hidden`/`sm:hidden` osztályokon keresztül
  - Meglévő tesztek zöldek maradnak (3 oszlop, pontok pill, multiplier kék vs amber stb.)
- **E2E (Playwright):** ha van hozzá pwa/scoring spec, **mobil viewport** (`page.setViewportSize({ width: 390, height: 844 })`) alatt a "Pontos eredmény bónusz" + "Eredmény: 2-1" példa **látható** és nem levágott
- **Manuális regression:**
  - Chrome DevTools mobile emulation (iPhone 14 Pro, iPhone SE)
  - Valódi iOS Safari + Android Chrome ellenőrzés
  - Desktop nézet pixelszintű azonosság (előtte-utána screenshot összevetés)

## Kockázatok és mitigáció

| # | Kockázat | Hatás | Mitigáció |
|---|---|---|---|
| 1 | A meglévő tesztek a táblázat-struktúrát assertelik, és mobil JSDOM-ban a `hidden` osztály dummy-ként benne marad | Hamis pozitív / negatív teszt | A teszteket explicit `sm:` breakpoint-tal írjuk: a query a CSS visibility helyett a Tailwind class jelenlétét nézi |
| 2 | A mobil kártya és desktop tábla **két helyre** írja le ugyanazt a `MATCH_RULES` listát | Duplikált markup, drift kockázat | **Ne** duplikáljuk a logikát: a `MATCH_RULES` constans **egy helyen** van, és **két `v-for`** rendereli (mobil és desktop) – a CSS `hidden`/`sm:hidden` a megjelenést kontrollálja |
| 3 | Tailwind v4 + `@import "tailwindcss"` setup esetén néha a `sm:` prefix nem indexelődik, ha új class-okat használunk | Mobil verzió mindig látszik / sosem látszik | A használt utility-k mind már léteznek a kódbázisban (grep a meglévő `sm:` használatra), nincs új class-purge konfig változás szükséges |
| 4 | A modal `max-h-[90vh]` + `overflow-y-auto` mobilon esetleg furcsán viselkedik, ha a kártyák magasabbak mint a tábla | Modal scroll oda-vissza | A kártyák magassága ~ azonos a táblázat soraival (3 sornyi szöveg max), nem gyorsabban skálázódik – ha mégis, manuális teszt során finomhangolható |

## Implementáció lépései

1. **Markup duplikálás** (~15 perc) – a `<table>` blokk wrappelése `<div class="hidden sm:block">`-be, alá új `<div class="sm:hidden">` mobil kártya layouttal, ugyanaz a `v-for="rule in MATCH_RULES"`
2. **Mobil kártya stílus** (~15 perc) – torna szekció CSS osztályainak adoptálása
3. **Manuális vizuális ellenőrzés** (~15 perc) – DevTools mobile emulation, 4 viewport
4. **Unit teszt frissítés** (~20 perc)
5. **E2E vagy manuális regression** (~15 perc)

**Becsült teljes munka:** ~1.5 óra.
