# US-944: Csoportmeghívó kontextus megőrzése és automatikus csoportba lépés bejelentkezés után

## Leírás

Mint **bejelentkezés nélküli felhasználó**, szeretnék egy csoportmeghívó linkre (`/app/join/:code`) kattintani, bejelentkezni vagy regisztrálni, és a bejelentkezés után **automatikusan belépni az adott csoportba**, hogy **ne vesszenek el a meghívó kontextusa és az invite flow ne szakadjon meg**.

## Jelenlegi helyzet

- `/app/join/:code`-ra kattintás unauthenticated felhasználóként redirect-ál `/login?redirect=/app/join/:code`-ra
- Email/password loginnál a redirect működik (`LoginView.vue:111-114`)
- **DE** Google OAuth callback (`AuthCallbackView.vue:24`) nem kezeli a redirect paramétert → fix `home`-ra megy
- Regisztráció (email vagy Google sign-up) után a redirect paraméter szintén elveszik
- Hasil: a user bejelentkezik, de a `/app/join/:code` flow nem folytatódik, elveszik az invite mágikus link

## Elfogadási kritériumok

### Frontend – Meghívó kód tárolása bejelentkezéskor

- [ ] Ha felhasználó **unauthenticated** `/app/join/:code`-ra érkezik, a router guard már redirect-ál `/login?redirect=/app/join/:code`-ra
- [ ] `LoginView.vue`: bejelentkezéskor (`loginWithEmail()`, `registerWithEmail()` után) **a `redirect` query paramétert is feldolgozzuk** — ha `/app/join/...` kezdetű, elmentsük (pl. localStorage-ban `pendingInviteCode` kulcs alatt)
- [ ] **Technikai döntés**: melyik módszer legyen?
  - **Opció A (localStorage):** Az auth store-ban tárolunk egy `pendingInviteCode: string | null` state-et, amely bejelentkezéskor ki lesz olvasva a localStorage-ból
  - **Opció B (router state):** Az `AuthCallbackView` fázisában az OAuth state paraméterbe csomagoljuk az invite kódot
  - **Ajánlás**: Opció A (localStorage), mert egyszerűbb, nem igényel OAuth state módosítást, és a Supabase config nem igényel metaadatok paraméterezése

### Frontend – Bejelentkezés után automatikus csoportba lépés

- [ ] `AuthCallbackView.vue:24` után (Google OAuth callback kezeléskor) és `LoginView.vue:114` után (email login/signup után) ellenőrizze az auth store a `pendingInviteCode`-ot
- [ ] Ha meglévő `pendingInviteCode`, a `groupsStore.joinGroup({ inviteCode: code })` automatikusan meghívódik
- [ ] Sikeres csoportbeléptetés után:
  - Törlődik a `pendingInviteCode` localStorage-ból
  - Redirect a csoport oldalra (`/app/groups/{groupId}`) — `router.push()` vagy `router.replace()`
- [ ] Sikertelen csoportbeléptetés (pl. lejárt kód, már tag):
  - `pendingInviteCode` törlődik
  - Toast hibaüzenet jelenik meg
  - Fallback: redirect a csoportok listára (`/app/groups`)
- [ ] Email/Google login + regisztráció esetén **azonos viselkedés** — a 4 belépési útvonal egységesen kezeli az invite kódot:
  1. Email bejelentkezés (`loginWithEmail()` után)
  2. Email regisztráció (`registerWithEmail()` után)
  3. Google OAuth login (létező felhasználó, `AuthCallbackView` handleSession után)
  4. Google OAuth sign-up (új felhasználó első bejelentkezés, `AuthCallbackView` handleSession után)

### Frontend – UX szélső esetek

- [ ] Ha az invite kód **már lejárt vagy érvénytelen**: hibaüzenet toast-ban, redirect `/app/groups`
- [ ] Ha a felhasználó **már tag az adott csoportban**: jelenleg a backend `POST /api/groups/join` 409-et ad (conflict) — ezt a frontend toast-ban megjeleníti, majd redirect `/app/groups/{groupId}` (mivel ismert a groupId az error response-ból vagy a csoportok listájából)
- [ ] Ha az invite kód **szintaxist nem felel meg** (nem 6-8 karakteres, stb.): a backend `POST /api/groups/join` 400-at ad — frontend toast hibaüzenet, redirect `/app/groups`
- [ ] Ha az **invite csoportja már törölve van** (soft delete): backend 404-et ad — toast hibaüzenet, redirect `/app/groups`
- [ ] **Többszöri bejelentkezés ugyanabban a tabban**: localStorage-ban csak egyetlen `pendingInviteCode` lehet — az első lépés után törlődik, így újabb bejelentkezéskor (ha történne) nincs invite kontextus, normál home redirect
- [ ] **Google OAuth hibaállapot** (user megtagadja, cancel): az `AuthCallbackView` kezelnie kell a hibaállapotot — törli a `pendingInviteCode`-ot, redirect `/login`

### Backend – Meglévő logika

- [ ] `POST /api/groups/join` endpoint (`packages/backend/src/routes/groups.routes.ts:39-50`) **változatlan marad** — az invite kódot validálja, csoportba lép, group visszaadja az ID-t
- [ ] Validációs hibák: 400 (érvénytelen kód), 404 (nem létezik), 409 (már tag), 410 (csoport/invite törölt)
- [ ] Backend NEM kell tudnia a localStorage-ról, a frontend ezt kezeli

### Tesztek

- [ ] **Unit (Frontend)**: localStorage-ba írás/olvasás `pendingInviteCode`-nál, törlés után
- [ ] **Integration (Frontend)**: email login → `pendingInviteCode` meglétével → `groupsStore.joinGroup()` meghívása → redirect
- [ ] **E2E (Smoke)**: unauthenticated user → `/join/VALID_CODE` kattintás → redirect login → email/Google login → automatikus csoportba lépés → `/app/groups/{id}` végül

## Szélső esetek

| Eset | Viselkedés |
|------|-----------|
| Lejárt invite kód | Toast "Meghívó kód lejárt", redirect `/app/groups` |
| Már tag a csoportban | Toast "Már csoporttag vagy", redirect `/app/groups/{id}` |
| Csoport vagy invite törölt | Toast "Csoportmeghívó nem elérhető", redirect `/app/groups` |
| Szintax hiba az invite kódban | Toast "Érvénytelen meghívó kód", redirect `/app/groups` |
| OAuth hibaállapot (megtagadás) | Toast "Bejelentkezés sikertelen", redirect `/login` |
| Második tab nyitása, azonos invite kód | Az első tab után localStorage törlődik → második tab → normál home redirect |
| localStorage disabled (privacy mode) | Fallback: localStorage nem elérhető → nincs `pendingInviteCode` → normál home redirect (graceful) |

## Technikai megjegyzések

### Frontend – Megvalósítási terv

**1. Auth store módosítás** (`packages/frontend/src/stores/auth.store.ts`):
```typescript
// Új state
const pendingInviteCode = ref<string | null>(null)

// Getter
function getPendingInviteCode(): string | null {
  return pendingInviteCode.value
}

// localStorage helper-ek
function savePendingInviteCode(code: string): void {
  try {
    localStorage.setItem('pendingInviteCode', code)
    pendingInviteCode.value = code
  } catch {
    // localStorage disabled
  }
}

function loadPendingInviteCode(): string | null {
  try {
    return localStorage.getItem('pendingInviteCode')
  } catch {
    return null
  }
}

function clearPendingInviteCode(): void {
  try {
    localStorage.removeItem('pendingInviteCode')
    pendingInviteCode.value = null
  } catch {
    // localStorage disabled
  }
}
```

**2. LoginView.vue módosítás** (111–114. sor meglétét kiterjesztjük):
```typescript
// loginWithEmail() vagy registerWithEmail() után
const redirect = route.query.redirect as string | undefined
if (redirect?.startsWith('/app/join/')) {
  // Extract invite code: /app/join/:code
  const code = redirect.split('/').pop()
  if (code) {
    authStore.savePendingInviteCode(code)
  }
}
```

**3. AuthCallbackView.vue módosítás** (handleSession után):
```typescript
// A handleSession meghívása után:
await authStore.handleSession(session)
const pendingCode = authStore.getPendingInviteCode()
if (pendingCode) {
  try {
    const group = await groupsStore.joinGroup({ inviteCode: pendingCode })
    authStore.clearPendingInviteCode()
    await router.push({ name: 'group-detail', params: { id: group.id } })
  } catch (err) {
    authStore.clearPendingInviteCode()
    toast.error('Csoportba lépés sikertelen: ' + err.message)
    await router.push({ name: 'groups' })
  }
} else {
  await router.push({ name: 'home' })
}
```

**4. Groups store módosítás** (`packages/frontend/src/stores/groups.store.ts`):
- `joinGroup()` error handling: már kezeli a `groups.service` exception-eit, frontend ezt csak toast-ban mutatja meg

### Backend – Nem szükséges módosítás

- `POST /api/groups/join` már ad vissza `groupId`-t, amelyet a frontend felhasznál
- A backend URL-ben szereplő invite kódot validálja, szintaxis/érvényességet ellenőrzi

### localStorage stratégia

- `pendingInviteCode` localStorage key: egyszerű string, pl. `"ABC123"`
- Törléskor: `localStorage.removeItem('pendingInviteCode')`
- Graceful fallback: ha localStorage disabled, nincs `pendingInviteCode` → normál home redirect
- Nincs TTL — az invite kód TTL-jét maga az invite endpoint validálja
