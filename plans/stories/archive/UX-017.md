# UX-017: Liga megjelenítése a csoportokon

## Leírás

Mint **felhasználó**, szeretnék **látni, hogy egy csoport melyik liga(ák) köré szerveződik**, hogy **el tudjam különíteni a csoportokat, és biztosan a megfelelőhöz csatlakozzam**.

## Elfogadási kritériumok

### GroupsView — csoport kártyák

- [x] Ha a csoport pontosan egy ligához tartozik: a kártya cím (`group.name`) alatt megjelenik a `league.name` szöveg — stílus: `text-xs text-gray-500`
- [x] Ha a csoport több ligához tartozik: a csoport neve alatt egymás mellett pill badge-ek jelennek meg, mindegyik a liga `shortName` értékét mutatja — stílus: `bg-gray-100 text-gray-600 text-xs rounded px-1.5 py-0.5`; az összes badge egyetlen sorban fér el (nem tördelődik)
- [x] Ha a csoportnak nincs egy ligája sem (`leagues` üres tömb): a cím alatt semmi extra nem jelenik meg, a kártya layout nem változik
- [x] A liga szöveg/badge-ek csak információ jellegűek, nem kattinthatók

### GroupDetailView — fejléc

- [x] A csoport nevét megjelenítő `<h1>` alatt, a tab bar felett megjelenik a liga jelzés — ugyanaz a logika: 1 liga = `league.name` szöveg, több liga = `shortName` pill badge-ek
- [x] Az elrendezés mobil nézetben is rendezett marad (375 px szélesség alatt sem törik szét a badge-ek a layout-ot)
- [x] Több liga esetén ha az összes badge nem fér ki egyszerre, a sor vízszintesen görgethetővé válik (`overflow-x: auto`) ahelyett hogy sortörés lenne

### Adatkezelés

- [x] A komponensek a már meglévő `group.leagues` tömböt olvassák — nincs új API hívás
- [x] Ha `group.leagues` undefined vagy null, a komponens nem dob hibát (fallback: üres tömb)

## UX specifikáció

### Egy liga — szöveg elrendezés (GroupsView)

```
┌─────────────────────────────────────┐
│  Barátok Liga                       │
│  FIFA World Cup 2026          [...]  │
└─────────────────────────────────────┘
```

### Több liga — pill badge elrendezés (GroupsView)

```
┌─────────────────────────────────────┐
│  Kombinált Liga                     │
│  [WC26]  [NB I]               [...]  │
└─────────────────────────────────────┘
```

### GroupDetailView fejléc

```
  Barátok Liga
  FIFA World Cup 2026          ← plain text (1 liga esetén)
  ─────────────────
  [ Tablescore | Meccsek | ... ]
```

```
  Kombinált Liga
  [WC26]  [NB I]               ← pill badge-ek (több liga)
  ─────────────────
  [ Tablescore | Meccsek | ... ]
```

## Technikai megjegyzések

- Érintett fájlok: `packages/frontend/src/views/GroupsView.vue`, `packages/frontend/src/views/GroupDetailView.vue`
- Backend módosítás nem szükséges — `group.leagues: [{ id, name, shortName }]` már elérhető
- A liga jelzés logikáját érdemes `useLeagueDisplay(leagues)` composable-be kiemelni (visszaadja: `mode: 'none' | 'single' | 'multi'`, `label: string`, `badges: string[]`), hogy mindkét view ugyanazt a kódot használja
- A composable-hez tartozzon unit teszt: üres tömb, egy elem, több elem esetén
- A `flag-icons` library opcionálisan használható egy kis ikon hozzáadásához a liga neve elé — nincs követelmény rá, nem akadályozza az AC-kat
- A pill badge HTML ne duplikálódjon — emeld ki egy `LeagueBadge.vue` komponensbe, ha több helyen jelenik meg

## Nem tartalmazza

- Sidebar módosítás
- Liga részletes oldalra navigálás a badge-ről
- Backend API változás
- Ligák szűrése vagy rendezése a csoportlistán

## Komplexitás

S (csak frontend, az adat már elérhető)
