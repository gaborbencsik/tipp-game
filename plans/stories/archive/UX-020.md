# UX-020: Admin sync felület konzisztens redesign + meccs-sync szekció előkészítése

## Leírás

Mint **adminisztrátor**, szeretném, hogy az **AdminSyncView.vue** szinkronizációs felület egységes vizuális és funkcionális struktúrát kövessen minden szinkron-típushoz (meccsek, Polymarket odds, játékosok, Transfermarkt értékek), hogy könnyebben navigálható, konzisztens legyen, és felkészítve a jövőbeli ligakonfigurációk hozzáadására (pl. US-1404 Pre-VB Edzőmeccsek).

## Jelenlegi helyzet

- **AdminSyncView.vue** jelenleg 6 szekciót tartalmaz:
  1. "Sync mód" — top-level select (valójában a manuális szinkronhoz tartozik)
  2. "Polymarket odds szinkronizáció" — toggle + trigger, **de nincs "Utolsó futás" sor**
  3. "Játékos szinkronizáció" — toggle + utolsó futás + trigger (✅ helyes minta)
  4. "Transfermarkt piaci értékek" — toggle + utolsó futás + trigger (✅ helyes minta)
  5. "Sync status" — külön gray meta-kártya (Utolsó sync, API hívások, Folyamatban...)
  6. "Manuális szinkronizáció" — gomb + meccs-szinkron eredmények

- **Inkonzisztenciák (a `screenshots/screenshot1.png` alapján is megerősítve):**
  - Polymarketnél hiányzik az "Utolsó futás" metaadat, amit a többi szekcióban mutatunk
  - A "Sync mód" select logikailag a "Manuális szinkronizáció"-hoz tartozik, de fent áll globálisnak tűnve
  - A "Sync status" meta szekció (szürke kártya) a "Manuális szinkronizáció" szekciót kellene hogy tartalmazd, nem külön
  - A "Manuális szinkronizáció" megnevezés túl általános — pontosabban **"Mérkőzés szinkronizáció"** (futball API-ből teams + fixtures + results)
  - Az API hívásszám és "Folyamatban" indikátor csak a football API-hoz tartozik, de nem világos ez a felületen

- **További UX-szempontú inkonzisztenciák a screenshot alapján:**
  - **Page title félrevezető**: "Futball API szinkronizáció" — viszont a Polymarket és a Transfermarkt **nem** futball API
  - **Sync mód select értelmezhetetlen kontextus nélkül**: csak `Off / Final only / Adaptive / Full live` opciók látszanak, magyarázat nélkül — admin nem tudja melyiket válassza. Tooltip vagy help text kell minden módra
  - **Toggle "aktív" felirat eltérő tartalmú**:
    - Polymarket: csak `Kikapcsolva` (toggle ON állapotban: csak `Aktív` frekvencia nélkül)
    - Játékos / Transfermarkt: `Aktív (napi 1×)` — frekvencia szerepel
    - Konzisztencia: minden toggle mutassa a frekvenciát ON állapotban (`Aktív (5 percenként)`, `Aktív (napi 1×)`)
  - **Dátum formátum eltérés**:
    - Játékos / Transfermarkt szekció: abszolút (`2026.05.17 19:50:03`)
    - "Sync status" doboz: relatív (`1 napja`)
    - Konzisztencia: relatív szöveg + abszolút tooltip minden szekcióban
  - **Szekciócímek eltérő szóhasználat**:
    - `Polymarket odds szinkronizáció`, `Játékos szinkronizáció` → tartalmazza a "szinkronizáció" szót
    - `Transfermarkt piaci értékek` → nem tartalmazza
    - Konzisztencia: minden cím `<X> szinkronizáció` mintát kövesse
  - **Üres állapot ("Még nem futott") elnyelődik**: a többi szekcióban `v-if` mögött rejtett az "Utolsó futás" sor, ha még sosem futott — ehelyett mindenhol jelenjen meg `Még nem futott` placeholder, hogy az admin lássa az állapotot és a felület ne ugráljon
  - **Leírás (`<p>`) elhelyezése eltérő**: minden szekcióban a kártya alján egy szürke szöveg adja meg a frekvenciát ("5 percenként…", "Lekéri a kereteket…") — ezt a Mérkőzés szekcióban is kell biztosítani konzisztensen

## Elfogadási kritériumok

### Frontend — Újrarányozás és szekció-reorganizáció

1. **Szekciósorszerkezet — új sorrend:**
   - **Mérkőzés szinkronizáció** (Sync mód select + API hívások + Folyamatban + eredmények) — **1. pozíció**
   - **Polymarket odds szinkronizáció** — **2. pozíció** (új: utolsó futás sor)
   - **Játékos szinkronizáció** — **3. pozíció** (nincs változás)
   - **Transfermarkt piaci értékek** — **4. pozíció** (nincs változás)

2. **Mérkőzés szinkronizáció szekció — egységes kártya**
   - `<h2>` Mérkőzés szinkronizáció
   - **Szekció 1: Szinkron mód kiválasztása**
     - `<label>` Szinkron mód
     - A jelenlegi `<select>` (Off / Final only / Adaptive / Full live) ide kerül
     - Mentés/Mentve indikátor megmarad
   - **Szekció 2: Konfigurált ligák** (read-only chip lista)
     - Cím: "Konfigurált ligák:"
     - Chip-ek megjelenítése (pl. "VB", "NB I", "Pre-VB Edzőmeccsek")
     - Az alkalmazás ezen ligákról szinkronizálja az adatokat (nem szerkeszthető ebből a felületről)
     - **Backend támogatottság szükséges:** új read-only endpoint vagy meglévő settings endpoint bővítése — ne adja vissza az API kulcsokat, csak a konfigurált ligák listáját (név + external ID)
   - **Szekció 3: API kvóta és futási státusz**
     - "API hívások: X / 100" (piros, ha > 80%)
     - "Folyamatban..." indikátor (spinning ikon + szöveg), ha aktív
   - **Szekció 4: Trigger és eredmény**
     - "Szinkronizálás indítása" gomb (letiltva ha `mode === 'off'`)
     - Eredmény/hiba szöveg
   - **Szekció 5: Eredménykártyák** (sikeres/részleges bontás: csapatok + meccsek + eredmények)
     - A jelenlegi `syncResults` kártyák megmaradnak

3. **Polymarket odds szinkronizáció — kiegészítés**
   - `<h2>` Polymarket odds szinkronizáció
   - Toggle: Aktív / Kikapcsolva (már van)
   - **Utolsó futás:** `{{ lastPolymarketSyncAt.toLocaleString(getDateLocale()) }}` (NEW — mint a Játékos és Transfermarkt szekciónál)
     - Ha `lastPolymarketSyncAt` null → "Még nem futott"
   - Manuális trigger gomb: "Odds lekérése most"
   - Eredmény/hiba szöveg
   - Leírás: "5 percenként lekéri a Polymarket odds-okat az összes VB meccshez."

4. **Játékos szinkronizáció — NINCS VÁLTOZÁS** (már helyes minta)
   - Toggle + Utolsó futás + trigger + leírás

5. **Transfermarkt piaci értékek — NINCS VÁLTOZÁS** (már helyes minta)
   - Toggle + Utolsó futás + trigger + leírás

6. **"Sync status" szekció eltávolítása**
   - Tartalma (Utolsó sync, API hívások, Folyamatban) bekerül a "Mérkőzés szinkronizáció" szekcióba
   - A szürke `bg-gray-50` kártya törlődik

7. **Mobil reszponzivitás**
   - A szekciók stack-eljék egymást vertical breakpoint alatt
   - Chip-ek a konfigurált ligáknál responsive flex-en
   - Gomb szöveg rövid marad (nem török sorokra)

8. **Page title általánosítás**
   - A jelenlegi `<h1>Futball API szinkronizáció</h1>` átnevezve `<h1>Adatszinkronizáció</h1>`-ra (vagy hasonló semleges címre), mert a Polymarket és a Transfermarkt nem futball API
   - A `<title>` meta is frissítendő ha külön van állítva

9. **Egységes "sync card" minta minden szekcióhoz**
   - Cím (`<h2>`) – mintázat: `<X> szinkronizáció` (Transfermarkt esetén is)
   - Toggle: `Aktív (<frekvencia>)` ON-ban (pl. `Aktív (5 percenként)`, `Aktív (napi 1×)`), `Kikapcsolva` OFF-ban
   - Utolsó futás sor: relatív szöveg (`X perce`, `Y órája`, `Z napja`) + abszolút tooltip a `<title>` attribútumon keresztül
   - Üres állapot: `Még nem futott` placeholder szöveg, ha `last*SyncAt` null (ne legyen rejtve)
   - Manuális trigger gomb (egyezően stílussal: `bg-blue-600 px-3 py-1.5`)
   - Eredmény / hiba szöveg
   - Rövid leírás (`<p class="text-xs text-gray-400">`) a kártya alján — frekvencia + adatforrás

10. **Sync mód select kontextus és help text**
    - Minden opció mellé rövid leírás (vagy tooltip / `<select>` alatti dinamikus help text):
      - `Off`: nem szinkronizál automatikusan, csak manuálisan
      - `Final only`: csak a befejezett meccsek eredményét frissíti
      - `Adaptive`: élő meccs alatt sűrűbben fut, egyébként ritkábban
      - `Full live`: minden meccsállapotot követ percenként
    - A magyarázatok forrása: a backend `SyncMode` típus dokumentációja (ha hiányzik, ki kell alakítani)

### Backend — API kiterjesztés

1. **Polymarket "Utolsó futás" adat**
   - Ha a `sync_state` táblában nincs `lastPolymarketSyncAt` mező: **migration szükséges** (`ALTER TABLE sync_state ADD COLUMN last_polymarket_sync_at TIMESTAMPTZ;`)
   - A `runPolymarket` (admin endpoint) végén: `markPolymarketSyncFinished()` hívás kell, amely frissíti az `lastPolymarketSyncAt`-ot
   - Az admin settings GET endpoint visszaadja az `lastPolymarketSyncAt` értéket

2. **Konfigurált ligák metaadat endpoint**
   - Új read-only endpoint vagy meglévő `GET /api/admin/sync/settings` kiterjesztése:
     - Visszaadja: `{ configuredLeagues: Array<{ name: string, externalId: number }> }`
     - Pl.: `[{ name: "VB", externalId: 1 }, { name: "NB I", externalId: 848 }, { name: "Pre-VB Edzőmeccsek", externalId: 10 }]`
     - Az adatok az env változóból jönnek (`FOOTBALL_INTERNAL_WC_LEAGUE_ID`, `FOOTBALL_INTERNAL_NBI_LEAGUE_ID`, és esetlegesen jövőbeli `FOOTBALL_INTERNAL_PRE_VB_LEAGUE_ID`)
     - **Fontos:** API kulcs tilos kiküldésre frontend felé

3. **Utolsó futás — relatív megjelenítés (frontend logika)**
   - Frontend az `lastPolymarketSyncAt` alapján számítja: "X perce", "Y órája", "Z napja" (már van `lastSyncRelative` computed property)

## Technikai megjegyzések

### Adatbázis — migration (ha szükséges)

```sql
-- Ha nincs last_polymarket_sync_at:
ALTER TABLE sync_state ADD COLUMN last_polymarket_sync_at TIMESTAMPTZ DEFAULT NULL;
```

### Backend — sync-state.service.ts bővítés

```typescript
// Ha még nincs:
async markPolymarketSyncFinished(): Promise<void> {
  await this.db.update(syncState).set({
    lastPolymarketSyncAt: new Date(),
  });
}
```

### Frontend — getConfiguredLeagues() composable vagy direct API hívás

```typescript
async function loadSettings() {
  const token = await getToken()
  const data = await api.admin.sync.getSettings(token)
  // data.configuredLeagues → ref-be
  configuredLeagues.value = data.configuredLeagues ?? []
}
```

### Szekciórendezés CSS

A szekciók már az adott sorrendben jelennek meg a template-ben (`<section id="match-sync">`, `<section id="polymarket-sync">`, stb.).

## Kizárások

- **Nem tartalmazza:**
  - Új szinkron-típus hozzáadása (az US-1404-hez való előkészítés, nem beépítés)
  - Liga runtime hozzáadása UI-ról (env-konfig, backend feladat)
  - A `runSync` backend logika módosítása (csak az UI-t érdekli)
  - Cron ütemezés vagy frekvencia módosítása
  - Polymarket API integrációja magadási valtozas (a `lastPolymarketSyncAt` mező hozzáadása és aktualizálása csak)
  - Audit log vagy történeti adatok tárolása a sync futásokról

## Komplexitás

M — frontend layout reorganizáció (rövid munka) + 1-2 backend endpoint/migration (ha Polymarket `lastPolymarketSyncAt` hiányzik és új `configuredLeagues` endpoint kell)
