# UX-034: Ranglista oszlopok újrarendezése – Arány, bontások, Torna pont

> Epic: E5 – Csoportok és ranglista

## Háttér

A jelenleg megjelenő ranglista oszlopok (`LeaderboardView.vue`):
- `#`, `Játékos`, `Tippek`, `Találat`, `Pont`

Az új tervezet az üzleti érték alapján átrendez:
1. **Tippek oszlop eltávolítása** — redundáns az Arány oszloppal.
2. **Találat oszlop eltávolítása** — az Arány (%) lefedi.
3. **Pont előtti bontások:** Arány (eltalált/leadott %), ⚽ Meccs (match-pontok), 👟 Gólszerző (scorer bonus pontok), 🏆 Torna (special/tournament pontok).
4. **Pont oszlop marad a végén** — total, bold, kiemelve.

**Adatmodell:** Minden adat már létezik a DB-ben; az aggregáció a meglévő service query-kben történik. Nincs migration szükséges.

## Story

Mint **felhasználó**, szeretném **vizuálisan szétválasztott pont-kategóriákat látni a ranglista Pont előtt (Arány %, Meccs, Gólszerző, Torna)**, hogy **könnyebben megértsem ki hogyan szerzett pontokat, és ne kelljen a megtekintést két klikk alatt végigmegynem.**

## Jelenlegi helyzet

- `LeaderboardView.vue` táblája 5 oszlopos: `#`, `Játékos`, `Tippek`, `Találat`, `Pont`.
- Backend `LeaderboardEntry` interface: `totalPoints`, `specialPredictionPoints`, de `matchPoints`, `scorerBonusPoints`, `successRate` hiányzik.
- Globális leaderboard query hardcoded `specialPredictionPoints: 0` — az `is_global=true` típusokra nem számol pontot.

## Elfogadási kritériumok

### Backend

1. **LeaderboardEntry interface kiterjesztése**
   - Új mezők: `matchPoints: number`, `scorerBonusPoints: number`, `successRate: number | null`.
   - A `specialPredictionPoints` név marad (nem átnevezés — más helyeken is használt), a frontend label "Torna pont".
   - `successRate` backend által számított: integer 0–100 vagy `null` ha `predictionCount = 0`.

2. **Globális leaderboard query módosítása**
   - Meglévő SELECT bővítve: `SUM(predictions.scorer_bonus_points)` AS `scorerBonusPoints`.
   - Plusz SELECT kifejezés: `CASE WHEN count(predictions.id) > 0 THEN round(count(CASE WHEN predictions.points_global > 0 THEN 1 END)::numeric * 100 / count(predictions.id), 0)::int ELSE 0 END` AS `successRate`.
   - Plusz query az `is_global=true` torna pontokra: `SELECT user_id, SUM(points) AS tournamentPoints FROM special_predictions JOIN special_prediction_types ON special_predictions.special_prediction_type_id = special_prediction_types.id WHERE special_prediction_types.is_global = true AND special_predictions.group_id IS NULL GROUP BY user_id`.
   - Az eredményt merge-elni a felhasználók listájába (LEFT JOIN szemantika).

3. **Csoport leaderboard query módosítása (mindkét ág)**
   - `useGroupPoints=true` és `useGroupPoints=false` ágak: ugyanaz a `scorerBonusPoints` SUM és `successRate` CASE kifejezés; `matches.id IS NOT NULL` őrökkel az inner join megőrzéséhez.

4. **Unit tesztek: leaderboard.service.test.ts és group-leaderboard.service.test.ts**
   - `matchPoints` helyes kiszámítása: `totalPoints - scorerBonusPoints`.
   - `scorerBonusPoints` helyessége.
   - `successRate` = 0, 100, közbenső érték, null (0 prediction) eset.
   - Csoport fav-team×2 esetre: `matchPoints + scorerBonusPoints !== totalPoints` (szándékos, komment dokumentálja).

### Frontend

5. **LeaderboardView.vue táblázat szerkezete**
   - Desktop (≥768px): `#`, `Játékos`, **`Arány`** (NEW), **`⚽ Meccs`** (NEW), **`👟 Gólszerző`** (NEW), **`🏆 Torna`** (NEW), `Pont`.
   - Nincs több `Tippek` (`leaderboard.tips` i18n) oszlop.
   - Nincs több `Találat` (`leaderboard.correct` i18n) oszlop.

6. **Arány oszlop megjelenítés**
   - Formula: `predictionCount > 0 ? round(correctCount / predictionCount * 100) + '%' : '—'` (em-dash).
   - Jobbra igazítva, `tabular-nums` osztály.
   - Data-tooltip: `"{correctCount} / {predictionCount}"` (ha `predictionCount > 0`).

7. **Bontás oszlopok (Meccs, Gólszerző, Torna) — tompa megjelenítés**
   - Cellatartalom jobbra igazított, `tabular-nums`.
   - Cella: `text-gray-500` szín.
   - Fejléc (header): `text-gray-400`, emoji opcionális (⚽/👟/🏆).
   - Mindhárom oszlop akkor is jelenik meg, ha 0 az érték.

8. **Pont oszlop marad**
   - `font-bold text-blue-700`, mint eddig.

9. **Mobile nézetben (<768px) — kompakt 2-soros megjelenítés**
   - Desktop bontás-oszlopok `hidden md:table-cell`.
   - A `Játékos` cella után új `<div class="md:hidden mt-0.5 ml-8 text-[0.65rem] text-gray-500 tabular-nums">⚽ {matchPoints} · 👟 {scorerBonusPoints} · 🏆 {tournamentPoints}</div>`.
   - Mobilon csak `#`, `Játékos` (+ bontás mini-row), `Arány`, `Pont` látszik scroll nélkül.

10. **Kondicionalitás: Torna oszlop elrejtése**
    - Computed flag: `showTournamentColumn` = `entries.value.some(e => e.specialPredictionPoints > 0)`.
    - Ha `false`: a `🏆 Torna` oszlop Desktop `<th>` és `<td>` `hidden`, mobil bontás mini-row `🏆` része szintén `hidden`.

11. **Vitest komponens teszt: LeaderboardView.test.ts**
    - Új oszlopok rendelésének tesztelése: fejléc szövegek (`leaderboard.successRate`, `leaderboard.matchPoints`, `leaderboard.scorerBonusPoints`, `leaderboard.tournamentPoints`).
    - Arány cella megjelenítés: `67%`, `—` (0 prediction), tooltip tartalom.
    - Régi oszlopok (`leaderboard.tips`, `leaderboard.correct`) assertek **eltávolítva** (ezek nem renderelődnek).
    - Mobilon bontás mini-row láthatósága, ikon/pont-érték helyessége.
    - Torna oszlop kondicionalitása: minden felhasználó `specialPredictionPoints=0` → oszlop nem látszik; legalább egy `>0` → oszlop látszik.

12. **i18n — lokalizáció (hu.json, en.json)**
    - Új kulcsok:
      - `leaderboard.successRate`: HU: "Arány", EN: "Success Rate" (vagy "Ratio").
      - `leaderboard.matchPoints`: HU: "Meccs", EN: "Match".
      - `leaderboard.scorerBonusPoints`: HU: "Gólszerző", EN: "Scorer".
      - `leaderboard.tournamentPoints`: HU: "Torna", EN: "Tournament".
    - Régi kulcsok (`leaderboard.tips`, `leaderboard.correct`) — opcionális: eltávolítás vagy elhagyás (ha nincs máshol használva, távolítsd el).

13. **ETag stabilitás**
    - Az új `matchPoints`, `scorerBonusPoints`, `successRate` mezők determinisztikusak.
    - Sorrendezés nem változik.
    - A `/api/leaderboard` response ETag-je a new schema miatt **frissülhet** — ügyelj a fetch policy-re, hogy ne omitálódjon a validation.

14. **Edge case-ek**
    - `predictionCount = 0` → `Arány` = `—`, `matchPoints = 0`, `scorerBonusPoints = 0`, `successRate = null`.
    - `successRate = null` → frontend nem mutat semmit (az `Arány` cellában `—` az érték már).
    - Globális nézet, senki nem tippelt torna tipn → `showTournamentColumn = false`.
    - Csoport nézet `useGroupPoints=false` (pontok globálisan összeadódnak) → `specialPredictionPoints` stb. ugyanúgy működik.

## Technikai megjegyzések

### Backend query frissítés

**Globális leaderboard:**

```sql
-- Original SELECT...
SELECT user_id, SUM(predictions.points_global) AS totalPoints, ...
-- Add:
       SUM(predictions.scorer_bonus_points) AS scorerBonusPoints,
       CASE WHEN count(predictions.id) > 0 
            THEN round(count(CASE WHEN predictions.points_global > 0 THEN 1 END)::numeric * 100 / count(predictions.id), 0)::int 
            ELSE 0 END AS successRate
FROM predictions
WHERE ...
GROUP BY user_id
```

**Torna pontok (plusz query):**

```sql
SELECT user_id, SUM(special_predictions.points) AS specialPredictionPoints
FROM special_predictions
JOIN special_prediction_types ON special_predictions.special_prediction_type_id = special_prediction_types.id
WHERE special_prediction_types.is_global = true 
  AND special_predictions.group_id IS NULL
  AND special_predictions.deleted_at IS NULL
GROUP BY user_id
```

Merge a kétségek közé (LEFT JOIN a user_id-n, default 0 ha null).

**Csoport leaderboard (mindkét ág):** Szintén `SUM(predictions.scorer_bonus_points)` és `successRate` CASE.

## Implementáció (végleges állapot)

A megvalósítás iteratívan finomult a vizsgálat során. A leszállított oszlopstruktúra:

**Desktop (≥768px) — 9 oszlop, ha nincs torna pont; 10 ha van:**
`#` | `Játékos` | `Sikeres tippek` (8/12) | `%` | `⚽ Meccs pont` | `%` | `👟 Gólszerző pont` | `%` | `[🏆 Torna]` | `Pont`

**Mobil (<768px) — 4 oszlop:**
`#` | `Játékos` | `Sikeres tippek` | `Pont`

A többi oszlop `hidden md:table-cell`. A `Játékos` cella alatt nincs mini-row a mobil verzión (a részletes bontás csak desktopon).

### Új backend mezők (`LeaderboardEntry`)

- `matchPoints: number` — `pointsGlobal − scorerBonusPoints` (tiszta eredmény-tipp pont).
- `scorerBonusPoints: number` — gólszerző bónusz összeg.
- `successRate: number | null` — összes pontot ért tipp aránya.
- `matchSuccessRate: number | null` — meccs-pontot szerző tippek aránya (eredmény alapján, gólszerző nélkül).
- `scorerSuccessRate: number | null` — gólszerzőt eltaláló tippek aránya.
- `specialPredictionPoints` — globális torna pontok (is_global=true, group_id IS NULL) is bekerülnek (előtte hardcoded 0 volt).

### Tooltipek

- `Sikeres tippek` fejléc: `tt-down` variáns (új CSS osztály) — lefelé pozícionál, nem vágódik le a tábla `overflow-hidden` szélén.
- `Sikeres tippek` cella: `tt` (felfelé), sor közepén nincs gond a vágással.

### i18n kulcsok

- `leaderboard.scoredFraction` — "Sikeres tippek" / "Successful tips"
- `leaderboard.scoredFractionTooltip` — "Pontot ért tippek / összes leadott tipp" / "Tips that scored points / total tips submitted"
- `leaderboard.successRate` — "%" (egységes mind a 3 százalék oszlopra)
- `leaderboard.matchPoints` — "Meccs pont" / "Match pts"
- `leaderboard.scorerBonusPoints` — "Gólszerző pont" / "Scorer pts"
- `leaderboard.tournamentPoints` — "Torna" / "Tourn."
- A régi `leaderboard.tips` és `leaderboard.correct` kulcsok eltávolítva.
