# US-004: Production deploy pipeline

> Epic: E0 – Fejlesztői infrastruktúra

**Story:**
Mint **fejlesztő**, szeretnék **egy automatizált pipeline-t, amely a `main` branch minden változásakor kideployolja az alkalmazást**, hogy **a legfrissebb verzió mindig elérhető legyen éles környezetben**.

**Elfogadási kritériumok:**
- [x] Frontend deployolva Vercelre; minden `main` push után automatikusan buildel és kiad
- [x] Backend deployolva Render-re; production `Dockerfile` (multi-stage build, nem dev server)
- [x] `render.yaml` Blueprint definiálja a backend service-t és env var referenciákat (IaC)
- [x] Supabase projekt konfigurálva: prod DB, Google OAuth callback URL beállítva
- [x] Minden szükséges env var beállítva Vercelen és Render-en (nem `.env` fájlból)
- [x] `https://` végpont elérhető: frontend URL-ről az API hívások működnek (CORS)
- [x] `npm run db:migrate` fut Render deploy hook-ként (pre-deploy command)
- [x] GitHub Actions CI: typecheck + test zöld marad minden merge előtt
- [x] Rollback: Render dashboard-ról bármely korábbi deploy visszaállítható

**Komplexitás:** M
**Prioritás:** Must Have

**Státusz:** ✅ Kész
