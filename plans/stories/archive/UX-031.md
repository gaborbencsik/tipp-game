# UX-031: Match Detail – ország megjelenítése a város mellett

> Epic: E5 – Match Detail UX

## Háttér

A `venues` tábla `country` oszlopa már szinkronizálva van (API-Football fixture / seed), de a frontend csak a `city` mezőt jeleníti meg. A backend `mapRowToMatch` (`packages/backend/src/services/matches.service.ts:62`) sem adja vissza. A Match Detail oldalon (`packages/frontend/src/views/MatchDetailView.vue:62`) a `{{ match.venue.name }}, {{ match.venue.city }}` formátum él.

Forrás: a sync-ek behoznak `venues.country` adatot, de jelenleg sehol nem használjuk a felületen — egyszerű low-hanging fruit.

## Story

Mint **felhasználó a Match Detail oldalon**, szeretném **a stadion neve és városa után az országot is látni**, hogy **gyors földrajzi kontextusom legyen, melyik országban játszanak (különösen vegyes nemzeti / klubliga / friendly meccseknél, ahol nem nyilvánvaló).**

## Elfogadási kritériumok

### Backend

- [ ] A `packages/backend/src/services/matches.service.ts` `mapRowToMatch` függvénye (és minden olyan hely, ami `venue` mezőt ad vissza a `GET /api/matches` és `GET /api/matches/:id` válaszában) kiegészül a `country` mezővel:
  ```ts
  venue: row.venues
    ? { name: ..., city: ..., country: row.venues.country, imageUrl: ... }
    : null
  ```
- [ ] **Üres string normalizálás:** ha a `venues.country` üres string (a meglévő seed/sync miatt egyes venue-knél így van), a backend `null`-ra konvertálja, hogy a frontend egyértelműen szűrhessen rá.
- [ ] **Backend unit teszt:** új mező mappingje + üres string → `null` normalizáció + venue nélküli match esete.

### Frontend — típusok és megjelenítés

- [ ] A `packages/frontend/src/types/index.ts` `MatchVenue` interface-e kiegészül a `readonly country: string | null` mezővel.
- [ ] A `packages/frontend/src/views/MatchDetailView.vue` venue-blokkja frissül:
  - Ha `country` van: `{{ venue.name }}, {{ venue.city }}, {{ venue.country }}`.
  - Ha nincs (`null`): `{{ venue.name }}, {{ venue.city }}` (a meglévő viselkedés).
- [ ] A `packages/frontend/src/components/matches/VenueBanner.vue` komponensben: ha ott is megjelenik szöveges venue-info, ugyanaz a logika; ha nem, érintetlen.
- [ ] **`data-testid="venue-location"`** a venue szöveg konténerén (assertálható E2E-ben).

### Tesztelés

- [ ] Frontend komponens teszt (Vitest) a `MatchDetailView.vue`-ra: country-vel és anélkül is helyesen renderel.
- [ ] Backend unit teszt a service-re (lásd fent).
- [ ] **E2E nem kötelező** — XS scope, a komponens teszt elég.

## Mit ne csináljunk

- Ne adj zászló-ikont az országhoz — egyszerű szöveg-konkatenáció.
- Ne hozzunk létre fordítás-kulcsot — egy vessző nem indokolja.
- Ne nyúlj a `venues.capacity` mezőhöz (külön téma).

Nincs.
