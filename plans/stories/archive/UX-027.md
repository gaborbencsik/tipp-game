# UX-027: Mérkőzések oldal szűrő állapot localStorage mentése

## Probléma

A `MatchesView` komponensen a "minden meccs" / "következő meccsek" toggle (vagy szűrő) állapota nem marad meg az oldal újratöltése után. A felhasználó minden alkalommal ismét módosítania kell a beállítást, bár a választása személyes preferencia és nyilvánvalóan perzisztens kellene.

## Story

Mint **mérkőzéseket böngésző felhasználó**, szeretném **az általam beállított szűrő állapotot (minden meccs vagy csak következők)**, hogy **az oldal újratöltése után is visszatöltődjön az utolsó választott beállítás**.

## Elfogadási kritériumok

1. **localStorage kulcs név**
   - localStorage kulcs: `matches-show-all-filter` (vagy `match-filter-show-all`)
   - Érték: boolean (`true` = minden meccs, `false` = csak következő)

2. **Alapértelmezett viselkedés**
   - Oldal első megnyitásakor (nincs localStorage érték): **default = `true`** (minden meccs szűrő bekapcsolva)
   - Desktopon és mobilon egyaránt

3. **Szűrő állapot mentése**
   - Toggle / checkbox state módosításakor: localStorage-ba írás **azonnal** (szinkron, `localStorage.setItem()`)
   - Memória (Pinia store) és localStorage közötti szinkron: store inicializálásakor localStorage értékéből olvasás, toggle módosításakor frissítés

4. **Oldal újratöltés**
   - Page refresh után (`F5`, browser back, `location.reload()`, másik URL-ről vissza linkkel): localStorage értéke betöltésre kerül
   - A szűrő a mentett állapotból helyreáll (happy path)

5. **Nincsen cross-tab szinkronizáció szükséges**
   - Ha másik tabon írják a localStorage-t, **nem kell** frissíteni az aktuális tabot (single-tab scope)
   - Ha szükséges, az `storage` event figyelése opcionális out-of-scope

6. **SSR/Hydration biztonság**
   - Vite SPA (nincs SSR), ezért localStorage olvasás biztonságos
   - localStorage hozzáférés csak komponens mount után vagy effect-ben történik (nincs ReferenceError serverside renderingnél — itt nem értelmes, de best practice)

## Technikai megjegyzések

### Fájlok érintve

- `packages/frontend/src/views/MatchesView.vue` — toggle/filter state management módosítás
- `packages/frontend/src/stores/matches.store.ts` — Pinia store szűrő state inicializálása localStorage-ből, setter action

### Implementációs vázlat

**Pinia store (`matches.store.ts`):**

```typescript
export const useMatchesStore = defineStore('matches', () => {
  // Szűrő state inicializálása localStorage-ből
  const showAll = ref<boolean>(() => {
    const stored = localStorage.getItem('matches-show-all-filter');
    return stored !== null ? JSON.parse(stored) : true; // default: true
  }());

  // Setter action
  const setShowAllFilter = (value: boolean) => {
    showAll.value = value;
    localStorage.setItem('matches-show-all-filter', JSON.stringify(value));
  };

  return { showAll, setShowAllFilter };
});
```

**MatchesView template:**

```vue
<label class="flex items-center gap-2">
  <input
    type="checkbox"
    :checked="matchesStore.showAll"
    @change="matchesStore.setShowAllFilter($event.target.checked)"
  />
  {{ $t('matches.showAll') }}
</label>
```

### localStorage key naming

- **Javasolt:** `matches-show-all-filter` (domain-specifikus, olvasható, egyértelmű)
- Alternatív: `match-filter-show-all` (egyformán jó)

### Megjegyzés: Nincs versioning szükséges

localStorage érték: egyetlen boolean. Nincs verzió byte, nincs JSON schema validálás szükséges — parse hibán fallback az alapértelmezett `true`.
