# US-807: Admin használati statisztikák (dashboard)

> Epic: E8 – Admin panel

## Leírás

Mint **platform admin**, szeretnék **egy áttekintő statisztikai dashboardot**, hogy lássam a platform aktivitását, azonosítsam az inaktív felhasználókat, és tudjam, ki nem tippelt még a következő meccsre.

## Összesített mutatók (4 summary card)

| Metrika | Leírás | Admin akció |
|---------|--------|-------------|
| Regisztrált felhasználók | `users` tábla nem-törölt sorok | Növekedés tracking |
| Aktív felhasználók (7 nap) | Legalább 1 tipp az elmúlt 7 napban | Engagement drop észlelés |
| Összes leadott tipp | `predictions` tábla sor száma | Volumen tracking |
| Kitöltési arány (fill rate) | `predictions / (users × meccsek) × 100` | Ha alacsony → emlékeztető küldés |

Másodlagos infó (subtitle, nem kártya): csoportok száma + átlagos méret.

## Felhasználónkénti táblázat

| Oszlop | Forrás |
|--------|--------|
| Avatar + név | `users` |
| Összes tipp | `count(predictions)` |
| Kitöltési % | `user predictions / matchCount × 100` |
| Szerzett pont | `sum(predictions.pointsGlobal)` |
| Csoportok száma | `count(group_members)` |
| Utolsó aktivitás | `max(predictions.createdAt)` |
| Tiltott? | `users.isBanned` badge |

- Default rendezés: utolsó aktivitás (desc) — inaktívak azonnal látszanak
- 0 tippes userek vizuálisan kiemelve (piros pont vagy háttér)
- Keresés név alapján
- Rendezhető: tipp db / pont / kitöltés / utolsó aktivitás

## Meccs-szintű statisztikák (táblázat)

| Oszlop | Leírás |
|--------|--------|
| Meccs | csapatok + dátum |
| Tippelt / összes user | hányan tippeltek |
| Kitöltési % | részvétel aránya |
| Eredmény | ha már van |

- Default rendezés: dátum (desc)
- Rendezhető: kitöltési %, dátum

**Eltávolítva:** átlag hazai/vendég tipp (nem actionable az admin számára).

- Fill rate < 50% → sárga háttér a kártyán

## Elfogadási kritériumok

### Backend

- [ ] `GET /api/admin/stats` — `authMiddleware + adminMiddleware`
  - Response: `{ summary: { userCount, activeUsers7d, predictionCount, fillRate, groupCount, avgGroupSize, zeroTipUsers }, users: [...] }`
- [ ] `GET /api/admin/stats/matches` — meccs-szintű részvételi statisztikák
  - Response: `{ matches: [{ matchId, homeTeam, awayTeam, date, tippedCount, totalUsers, fillPercent, result }] }`
- [ ] Külön `stats.service.ts` — hatékony aggregáció (nem N+1)
- [ ] Unit tesztek az aggregációs service-hez

### Frontend

- [ ] `/admin/stats` route — `requiresAuth + requiresAdmin`
- [ ] Admin menüben "Statisztikák" link
- [ ] 4 summary card renderelve (nagy szám + label + delta ha elérhető)
- [ ] Fill rate kártya sárga háttér ha < 50%
- [ ] Felhasználói táblázat: keresés + rendezés (tipp/pont/fill/utolsó aktivitás)
- [ ] 0 tippes userek vizuálisan kiemelve
- [ ] Meccs táblázat: rendezés (dátum/fill%)
- [ ] Minden adat valós idejű — oldal betöltésekor friss query
- [ ] Frontend tesztek az `AdminStatsView`-hoz

## Technikai megjegyzések

- 2 endpoint (nem 3) — a user lista a summary-val együtt jön, 200 usernél nincs szükség paginációra
- `fillRate = predictionCount / (userCount × matchCount) × 100`, ahol `matchCount` = nem-törölt meccsek
- `zeroTipUsers`: `users WHERE id NOT IN (SELECT DISTINCT user_id FROM predictions)`
- `activeUsers7d`: `SELECT COUNT(DISTINCT user_id) FROM predictions WHERE created_at > now() - interval '7 days'`

## Nem tartalmazza

- Aktivitási trend chart (sparkline) — nincs chart library, 1 hónapos tornán overkill
- Átlag hazai/vendég gól tipp oszlop — nem actionable
- Export funkció (CSV)
- Push notification küldés az admin dashboardról
- Per-csoport statisztikák (az a csoport detail oldalra való)

## Komplexitás

M — backend aggregáció (2 endpoint) + frontend 1 nézet (cards + 2 táblázat)
