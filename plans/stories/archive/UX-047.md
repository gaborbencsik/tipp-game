# UX-047: Bracket döntő/bronz – „A te tipped" sor a kiértékelt nézetben

> Epic: E5 – Torna tippek UX (kapcsolódó: UX-045 bracket kiértékelt nézet)
> Mockup: `plans/design/UX-045-final-bronze-actual-winner.mockup.html`

## Háttér

A `FinalsAndBronzeCard` kiértékelt nézetében a döntő chip-párja félrevezető. A chipek felirata az
**eredetet** írja le ("M1 GYŐZTES", "M2 GYŐZTES" = az SF1/SF2 győztesei = a két döntős), a szín viszont
az **eredményt** (zöld = bajnok, piros = nem bajnok). Emiatt a piros chip (pl. POR) úgy néz ki, mintha
a csapat "be sem jutott volna a döntőbe", pedig valójában csak annyit jelent: nem az a csapat nyerte a tornát.

A user visszajelzése: nem derül ki egyértelműen, **melyik csapatra tippelt bajnoknak**, és eltalálta-e.

## Story

Mint **felhasználó**, szeretném **a kiértékelt döntőnél egyértelműen látni, melyik csapatot tippeltem
bajnoknak és eltaláltam-e**, hogy **ne keverjem össze a chipek eredet-feliratát a tényleges eredménnyel.**

## Megoldás (mockup alapján)

A chipek **változatlanok** (M1/M2 győztes feliratok, zöld/piros a tényleges eredmény szerint). A döntő
chip-párja alá bekerül egy új, **teljes szélességű, középre igazított „tipp sor"**, ami a user által
bajnoknak tippelt csapatot mutatja, és a döntő eredménye szerint színeződik.

### A tipp sor 3 state-je

A sor state-jét **a döntő tényleges győztese** (`championId`, amit az admin állít be) vezérli. A tipp sor
**csak kiértékelt módban** (`evaluation != null`) jelenik meg — a pre-eval picker UI változatlan.

1. **Döntősök kiértékelve, bajnok még nincs** (`evaluation` megvan, de `championId` még `null`): mindkét
   chip **zöld** (mindkét csapat bejutott a döntőbe), a tipp sor **függőben** — semleges (szürke) stílus,
   `⏳` ikon + a user által tippelt csapat, nincs zöld/piros ítélet.
2. **Kiértékelt – talált** (`userChampionPick === championId`): a chipek közül a bajnok zöld, a másik piros;
   a tipp sor **zöld**, `✓` + a csapat (zászló + shortCode), nincs áthúzás.
3. **Kiértékelt – nem talált** (`userChampionPick !== championId`): a chipek ugyanígy (bajnok zöld, másik
   piros); a tipp sor **piros**, `✗` + a csapat **áthúzva** (`line-through decoration-red-400`).

### Scope-kizárás

- A **bronz** meccshez ebben a módosításban **nem** kell tipp sor.
- A **tippelő (pre-eval) UI változatlan** — a mockupban pirossal körözött chip-pár (state 1) nem módosul.

## Elfogadási kritériumok

### Megjelenítés

- [ ] Kiértékelt döntő chip-pár alatt megjelenik a tipp sor (`data-testid="bracket-final-user-pick"`).
- [ ] A tipp sor középre igazított, teljes szélességű, a chip-pár és (ha van) az összegzés között.
- [ ] **Függőben** state (döntősök megvannak, bajnok még nincs): semleges (szürke) border+bg, `⏳` ikon,
      csapat zászló + shortCode, nincs áthúzás, nincs zöld/piros.
- [ ] **Talált** state: emerald border+bg, `✓` ikon, csapat zászló + shortCode, nincs áthúzás.
- [ ] **Nem talált** state: red border+bg, `✗` ikon, csapat zászló + shortCode, `line-through`.
- [ ] Kiértékeletlen módban (`evaluation == null`) a tipp sor **nem renderelődik** (pre-eval UI érintetlen).
- [ ] A bronz meccsnél tipp sor **nincs**.
- [ ] Nincs „A te tipped" felirat-szöveg a soron — csak ikon + csapat.

### Adat / logika

- [ ] A user bajnok-tippje = a döntő meccsre mentett `winners[finalMatchId]` (a `state.winners`-ből,
      amit a `BracketProgressionPicker` amúgy is ismer).
- [ ] A tényleges bajnok = `correctAnswerParsed.champion` (`championId`), ugyanaz, amit a chipek színezéséhez
      használunk (`finalMatchAdvancingIds`).
- [ ] **Függőben:** ha `evaluation != null` de `championId == null` → a sor semleges (⏳), nincs ítélet.
- [ ] **Talált** = `championId != null && userChampionPick === championId`.
- [ ] **Nem talált** = `championId != null && userChampionPick !== championId`.
- [ ] Ha a user nem tippelt bajnokot (`winners[finalMatchId]` hiányzik), a tipp sor **nem jelenik meg**
      (nincs mit megjeleníteni/összevetni).
- [ ] A `championId` és a user-tipp propként érkezik a `FinalsAndBronzeCard`-ba, onnan a döntő
      `BracketMatchCard`-jába (vagy a `FinalsAndBronzeCard` maga rendereli a sort a döntő alatt — implementációs döntés).

### i18n

- [ ] Nincs új felirat-kulcs szükséges a soron (ikon + csapat rövid kód). Ha kell aria-label, új kulcs:
      `bracketProgression.yourChampionPick` (HU: "A bajnok-tipped: {team}", EN: "Your champion pick: {team}").

### Tesztelés

- [ ] Vitest komponens teszt a `FinalsAndBronzeCard.vue`-ra (4 eset):
  1. Nincs `evaluation` → tipp sor nem renderelődik.
  2. `evaluation` megvan, `championId == null` → sor látszik, semleges (⏳), nincs `✓`/`✗`, nincs `line-through`.
  3. `championId === userPick` → `[data-testid="bracket-final-user-pick"]` látszik, emerald, `✓`, nincs `line-through`.
  4. `championId !== userPick` → sor látszik, red, `✗`, `line-through`.
- [ ] E2E nem kötelező — S scope, komponens teszt elég.

## Mit ne csináljunk

- Ne módosítsuk a chipek jelentését/színét (M1/M2 győztes feliratok maradnak).
- Ne adjunk tipp sort a bronz meccshez.
- Ne nyúljunk a pre-eval (tippelő) UI-hoz — a kék „kiválasztott" chip marad a tipp jelzése.
- Ne írjunk hardcoded csapat-adatot — `teamMap`-ből jön a zászló + shortCode.

## Függőség

UX-045 (bracket kiértékelt nézet) — a `championId` és a kiértékelt mód (`evaluation`) már megvan a
`FinalsAndBronzeCard`-ban, erre épül.
