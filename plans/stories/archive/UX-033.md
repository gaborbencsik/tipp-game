# UX-033: Pontozási badge duplikáció eltávolítása

## Leírás

Mint **felhasználó**, szeretném, hogy a tipp melletti pontzavardelem világosan különítse el az **eredménytippből származó pontot** és a **gólszerzős bonuszpontot**,
hogy ne legyen félreértés az összpontszámról.

## Jelenlegi helyzet

A Match Detail oldalon (és a Matches listában, Dashboard-on) a felhasználó tippje mellett **duplikált pontozási információ** jelenik meg:

```
Tippem: 2 – 0   [+3 pont]
⚽ Raúl Alonso Jiménez Rodríguez   [+1 pont]
```

A probléma: a 3 pont már magában foglalja az 1 pont gólszerzős bonuszt (2 pontos eredmény tipp + 1 pont scorer bonus = 3 pont össz). A `+1 pont` a gólszerző név mellett azonban **külön kiírva** azt sugallja, hogy összesen **4 pont** (3 + 1), pedig a valódi összeg 3.

**Backend logika** (`scoring.service.ts` sor 71–75):
```ts
const resultPoints = calculatePoints(prediction, result, config)  // pl. 2
const scorerBonusPoints = calculateScorerBonus(scorer)             // pl. 1
const subtotal = resultPoints + scorerBonusPoints                  // 2 + 1 = 3
const pointsGlobal = applyFavoriteTeamMultiplier(subtotal, favCtx) // × 2 ha kedvenc
return { pointsGlobal, scorerBonusPoints }
```

A frontend-en megjelenítjük: `pointsGlobal` (3) és `scorerBonusPoints` (1) egyszerre, ami félrevezető.

## Elfogadási kritériumok

1. **Eredmény- és scorer-pont szétbontása a backendben**
   - A prediction response tartalmazza az új `pointsResult` mezőt: az eredménytippből származó pont (favorite multiplier-rel már alkalmazva, scorer bonus nélkül)
   - `pointsResult = pointsGlobal - (scorerBonusPoints × favoriteMultiplier)`
   - Edge case: ha `scorerBonusPoints === 0` (nem nyert), akkor `pointsResult = pointsGlobal`
   - Edge case: ha favorite team és scorer bonus mindkettő aktív, az multiplier csak az eredmény-pontra vonatkozik vagy mind a kettőre? **Backend dönt: a spec szerint az egész `subtotal` szorozódik, ezért `pointsResult` szintén multiplier-rel lett kalkulálva**

2. **Frontend: Match Detail oldal (MatchDetailView.vue)**
   - Tipp badge: csak `pointsResult` jelenjen meg
   - Gólszerző badge: csak `scorerBonusPoints` jelenjen meg (ha > 0)
   - Pl.: `🔒 Tippem: 2 – 0   [+2 pont]` majd új sorban `⚽ Raúl Alonso   [+1 pont]`
   - A leaderboard mutatja a `pointsGlobal`-t (nem változik)

3. **Frontend: Matches lista (MatchesView.vue)**
   - Ugyanaz: `+pointsResult` a tipp-résznél, `+scorerBonusPoints` a gólszerző-résznél
   - Tesztek: 210–215. sor

4. **Frontend: Dashboard PredictionListItem.vue**
   - Fő pont: csak `pointsResult` badge
   - Gólszerző sor: csak `scorerBonusPoints` (ha > 0)
   - Pl.: `2–0` majd `+2` badge, alatta `⚽ Jiménez` majd `+1` badge

5. **Edge case-ek tesztelése**
   - **Pontos eredmény (2–0, 2–1, stb.)**
     - `resultPoints = 2 + 1 (exact) = 3` → `pointsResult = 3` ha nincs scorer, `pointsResult = 3` ha van scorer de nem nyert (a scorer bonus 0)
   - **Csak kimenetel (1–0 helyett 1–1 találat, de győzelem igen)**
     - `resultPoints = 1` (csak correctOutcome) → `pointsResult = 1` ha nincs scorer
   - **Gólszerző nyert, kedvenc csapat (×2 multiplier)**
     - `subtotal = 2 + 1 = 3` → `pointsGlobal = 3 × 2 = 6`
     - `pointsResult = 6 - (1 × 2) = 4` vagy `pointsResult = (2 × 2) = 4`? → Backend: az eredmény-pont szintén ×2, így `pointsResult = 4`
   - **Gólszerző nem nyert, kedvenc csapat (×2 multiplier)**
     - `subtotal = 2 + 0 = 2` → `pointsGlobal = 2 × 2 = 4`
     - `pointsResult = 4 - 0 = 4` ✓
   - **0 pont mindent**
     - `resultPoints = 0` (nem találat) → `pointsResult = 0`, `pointsGlobal = 0`, `scorerBonusPoints = 0` → semmi nem jelenik meg

6. **Backend response validáció**
   - API GET `/predictions` (vagy `/matches/:id/myprediction` stb.) visszaadja: `{ pointsGlobal, pointsResult, scorerBonusPoints }`
   - Drizzle query teszt: az SQL helyes-e?

7. **Visual/UX validáció**
   - Nincs téves összeg-sugallat
   - Gólszerző badge csak akkor jelenik meg, ha `scorerBonusPoints > 0` és `scorerPickPlayerId` nem null
   - Szöveg egyértelmű (pl. "Tippem", "Gólszerző", "Pont" label-ek helyesen fordítottak)

## Technikai megjegyzések

### Backend: `packages/backend/src/services/predictions.service.ts`

Jelenleg az API response-ban:
```ts
interface PredictionResponse {
  pointsGlobal: number | null
  scorerBonusPoints: 0 | 1
  // ...
}
```

Módosítás:
```ts
interface PredictionResponse {
  pointsGlobal: number | null
  pointsResult: number | null        // ← új mező: eredmény-pont (multiplier már alkalmazva, scorer bonus nélkül)
  scorerBonusPoints: 0 | 1
  // ...
}
```

Kalkuláció a service-ben (nem DB mező, derived):
```ts
// Ha pointsGlobal van
const pointsResult = pointsGlobal - (scorerBonusPoints > 0 ? scorerBonusPointsApplied : 0)
```

**Probléma:** a `scorerBonusPoints` DB-ben **multiplier nélkül** van tárolva (0/1), de ha favorite team van, az már a `pointsGlobal`-ba van ×2-ezve. Ezért a backend-nek tudnia kell, hogy a scorer bonus a multiplier alatt vagy felett volt-e szorozva.

**Megoldás:** A `calculatePointsWithScorer` függvényt vizsgáld meg — azt már `multiplied=true` formában adja vissza? Vagy a storage előtt külön kell kezelni? 

**Legjobban:** a `predictions` tábla új mezője: `scorerBonusPointsApplied: 0 | 1` — csak a scorer bonus a multiplier-rel szorzódva. De ez **migráció** = komplexitás.

**Alternatíva (MVP):** a frontend szimplán számol: `pointsResult = pointsGlobal - scorerBonusPoints` ha favorite multiplier nincs, de **ezt csak akkor** tudjuk, ha a frontend ismeri a multiplier logikát (ami probléma). → **Backend solution jobb.**

### Tesztek

**Backend teszt** (`scoring.service.test.ts`):
- `calculatePointsWithScorer` már létezik, az `ScoredPrediction` interface-t terjeszd ki `pointsResult` mezővel
- Unit teszt: `pointsResult = pointsGlobal - (scorerBonus × multiplier)` ✓

**Frontend komponens teszt**:
- `MatchDetailView`: a scorer badge csak akkor jelenik meg, ha `scorerBonusPoints > 0` ✓
- `MatchesView` / `PredictionListItem`: ugyanaz ✓
- Snapshot teszt az új badge struktúrához ✓
