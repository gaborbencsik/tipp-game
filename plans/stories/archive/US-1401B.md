## Leírás

Mint **rendszergazda**, szeretném, hogy **a pontszámítási szabályok az első tipp leadása után automatikusan zárolódjanak** (freeze), de szükség esetén **admin override-dal módosíthatók legyenek**, és ilyenkor **újraszámolás** biztosítsa a konzisztenciát.

Mint **rendszergazda**, szeretném, hogy **a sync ne írja felül a már kiszámolt pontokat**, hanem csak az újonnan lezárt meccsekre számoljon — kivéve, ha az eredmény változott.

## Jelenlegi helyzet (problémák)

1. **Nincs freeze:** A scoring config bármikor módosítható, nincs lock mechanizmus
2. **Sync mindig felülír:** Minden sync futás az összes finished meccsre újraszámolja a pontokat — ha közben változott a config, csendben inkonzisztens állapot jön létre
3. **Nincs célzott recalc:** Ha szabályt módosítunk, nincs "számold újra az összeset az új szabállyal" funkció

## Elfogadási kritériumok

### Rész 1: Config freeze

- [ ] A `scoring_configs` tábla kap egy `frozen_at TIMESTAMPTZ` mezőt
- [ ] A globális config automatikusan freeze-elődik, amikor az adott config-hoz tartozó első prediction beérkezik
- [ ] A csoport-szintű config override automatikusan freeze-elődik, amikor a csoportban az első prediction beérkezik
- [ ] Freeze állapotban a `PUT /api/admin/scoring-config` és `PUT /api/groups/:groupId/scoring-config` 409-et ad: `"Scoring config is frozen"`
- [ ] A frontend disable-olja a szerkesztő mezőket, felettük egy banner: "A konfiguráció zárolva van ([dátum] óta). Az első tipp leadása után a szabályok védve vannak."
- [ ] A bannerben jobb oldalon egy "Felülír" gomb (csak site admin és group admin számára)

### Rész 2: Admin override

- [ ] A "Felülír" gombra kattintva egyetlen modal jelenik meg:
  - Warning szöveg (1 mondat)
  - Reason dropdown (kötelező): "Hibás beállítás" / "Versenyrendező kérés" / "Technikai javítás" / "Egyéb"
  - Opcionális szabad szöveges megjegyzés
  - Checkbox: "Pontok újraszámítása az összes mérkőzésre" (alapértelmezetten bekapcsolva)
  - Ha checked → megjelenik: "Ez N mérkőzést és M tippet érint."
  - Gombok: "Mégsem" (secondary) | "Felülírás" (destructive red)
- [ ] Site admin: globális config-ot és bármely csoport config-ját override-olhatja
- [ ] Group admin: csak saját csoportjának config-ját override-olhatja
- [ ] Ha `recalculate: true` → az override után automatikusan lefut az újraszámolás
- [ ] Ha `recalculate: false` → csak az ezután rögzített eredmények használják az új config-ot

### Rész 3: Sync módosítás — skip already calculated

- [ ] A `match_results` tábla kap egy `points_calculated_at TIMESTAMPTZ` mezőt
- [ ] A `syncResults` csak akkor hív `calculateAndSavePoints` + `calculateAndSaveGroupPoints`, ha:
  - Nincs még `points_calculated_at` (új eredmény), VAGY
  - Az eredmény megváltozott (score correction)
- [ ] Sikeres pontszámolás után a sync beállítja a `points_calculated_at = now()`-t
- [ ] Ha `resultAlreadyExists && !scoreChanged` → skip (nem nyúl a pontokhoz)

### Rész 4: Teljes újraszámolás (admin)

- [ ] Új endpoint: `POST /api/admin/scoring/recalculate-all`
- [ ] Opcionális `groupId` query paraméter — ha megadva, csak az adott csoport pontjait számolja újra
- [ ] Site admin jogosultság szükséges
- [ ] Response: 202 Accepted (async futtatás)
- [ ] A `syncInProgress` flag-et ellenőrzi indulás előtt → ha sync fut, 409
- [ ] Eredmény lekérdezhető: `GET /api/admin/scoring/recalculate-status` → `{ status: 'running' | 'idle', lastResult?: { matches, predictions, duration_ms } }`
- [ ] Az admin UI-on: override modalon belül checkbox VAGY külön "Újraszámolás" gomb a scoring config oldalon
- [ ] Párhuzamos futás megakadályozva (`syncInProgress` guard)

## Technikai megjegyzések

### Freeze trigger

Az `upsertResults`-ban (nem a scoring service-ben): első result insert-nél ellenőrzi és beállítja a `frozen_at`-ot.

**Alternatív (ajánlott):** A prediction service-ben, az első prediction mentésekor — ez korábban zár és védi azokat akik már tippeltek.

### Sync skip logika

```typescript
for (const fixture of finishedFixtures) {
  const match = await findMatchByExternalId(fixture.fixture.id)
  if (!match) continue

  const existingResult = await getMatchResult(match.id)
  const scoreChanged = existingResult &&
    (existingResult.homeGoals !== homeGoals || existingResult.awayGoals !== awayGoals)

  // Upsert match result (mindig)
  await upsertMatchResult(match.id, homeGoals, awayGoals, outcomeAfterDraw)

  // Pont csak ha: új eredmény VAGY eredmény változott
  if (!existingResult || scoreChanged) {
    await calculateAndSavePoints(match.id, result)
    await calculateAndSaveGroupPoints(match.id, result)
    await db.update(matchResults)
      .set({ pointsCalculatedAt: new Date() })
      .where(eq(matchResults.matchId, match.id))
  }
}
```

### Recalculate-all

```typescript
async function recalculateAll(groupId?: string): Promise<RecalcResult> {
  // Guard: syncInProgress check
  const allResults = await db.select().from(matchResults)
  let matchCount = 0, predCount = 0

  for (const result of allResults) {
    await calculateAndSavePoints(result.matchId, result)
    if (groupId) {
      await calculateAndSaveGroupPointsForGroup(result.matchId, result, groupId)
    } else {
      await calculateAndSaveGroupPoints(result.matchId, result)
    }
    await db.update(matchResults)
      .set({ pointsCalculatedAt: new Date() })
      .where(eq(matchResults.matchId, result.matchId))
    matchCount++
  }
  return { matchesRecalculated: matchCount, predictionsUpdated: predCount, duration_ms }
}
```

### Concurrency guard

```typescript
// Recalculate-all elején
const syncState = await getSyncState()
if (syncState.syncInProgress) {
  ctx.status = 409
  ctx.body = { error: 'Sync is currently running. Try again later.' }
  return
}
```

### Migration

1. `ALTER TABLE scoring_configs ADD COLUMN frozen_at TIMESTAMPTZ` — nullable, default null
2. `ALTER TABLE match_results ADD COLUMN points_calculated_at TIMESTAMPTZ` — nullable, default null
3. Nincs backfill — a következő sync run természetesen kitölti

## User-facing átláthatóság

- A ranglistán / csoport beállításoknál egy összecsukható "Hogyan számítjuk a pontokat?" szekció mutatja az aktuális config értékeket
- Ha újraszámolás történt, rövid inline jelzés: "Pontok frissítve: [dátum]"

## Nem tartalmazza

- Scoring config verziózás / history tábla
- Visszaállítás korábbi config-ra (undo)
- Per-meccs scoring override (pl. "dupla pont a döntőre") — külön story
- Audit log bejegyzések
