# UX-038: Torna tippek – Csoport végeredmény helyes válasz olvasható megjelenítése

> Epic: E5 – Torna tippek – admin funkciók és pontozási szabályok

## Háttér

A kiértékelt `all_groups_standing` ("Csoport végeredmény") tornatipp helyes válasza jelenleg nyers JSON-ként jelenik meg a felhasználói felületen — `{"groups":{"A":["uuid","uuid",…]},"best3rds":["uuid",…]}` szerű blob. A backend `correctAnswerLabel` resolver (`packages/backend/src/services/tournament-tips.service.ts` ~194–217 sor) csak a `team_select / player_select / text / dropdown / multi_team_weighted` típusokra állít elő olvasható labelt, a többi típusra `null`-t küld. Ekkor a frontend `TournamentTipsView.vue:84` a `formatRawCorrectAnswer()` fallbackre esik, ami csak `[…]` JSON-tömböket kezel, objektumot nem — így a teljes JSON nyersen kikerül a kártyára.

A `bracket_progression` típusra ugyanez a probléma elvileg fennáll, de a meglévő screenshot a `all_groups_standing` típust mutatja.

## Story

Mint **felhasználó**, szeretném **a kiértékelt Csoport végeredmény torna-tipp helyes válaszát olvasható szövegként látni** (csapatnevek csoportonként + továbbjutó 3. helyezettek listája), hogy **ne nyers JSON-t kapjak a kártya alján.**

## Megoldási irány

- A backend `tournament-tips.service.ts` `correctAnswerLabel` resolverét bővítsük új ágakkal:
  - **`all_groups_standing`**: a `parseAllGroupsStandingAnswer()` segítségével dekódoljuk a JSON-t, majd csoportonként soronkénti formátumban adjunk vissza string-et, csapat-ID → név fordítással. Best 3rd-ek listája külön sorba.
  - **`bracket_progression`**: a `parseBracketProgressionAnswer()` parse után meccsenként "slotA vs slotB → győztes" formátum (ha tudunk slot-címkét adni; egyébként csak `győztes` lista).
- A team ID-k összegyűjtése a meglévő `teamIds` aggregálóhoz adódik hozzá (ld. `tournament-tips.service.ts:84-131` blokk).
- Frontend `TournamentTipsView.vue` változtatás minimális: ha a `correctAnswerLabel` többsoros (`\n`-eket tartalmaz), `whitespace-pre-line` osztállyal jelenítjük meg, hogy a struktúra megmaradjon.

## Elfogadási kritériumok

### Backend (TDD – Red → Green → Refactor)

1. **`tournament-tips.service.ts` `correctAnswerLabel` ág `all_groups_standing` típusra:**
   - Bemenet: a `correctAnswer` egy valid `{ groups: {...}, best3rds: [...] }` JSON.
   - Kimenet: többsoros olvasható string. Csoportonként: `A: csapat1, csapat2, csapat3, csapat4`. Záró sor (ha vannak best 3rd-ek): `Továbbjutó 3. helyezettek: X, Y, Z`.
   - Hiányzó pozíció (`null`) → `–` jellel jelenik meg azon a helyen.
   - Ismeretlen team UUID → `?` jellel jelenik meg azon a helyen (ne dobjon hibát).
   - Üres / parse-hibás `correctAnswer` → `correctAnswerLabel` marad `null` (visszaesés a meglévő frontend fallbackre).

2. **`team_ids` aggregáló:** a parse-olt `correctAnswer` csapat-UUID-jei is bekerülnek a `teamIds`-be (a name lookup-hoz).

3. **`bracket_progression` ág** (best effort, szigorú slot-címke nélkül is):
   - Bemenet: valid `{ winners: { matchId: teamUuid, ... } }`.
   - Kimenet: meccsenkénti sor, pl. `Match <id>: <csapatnév>` (vagy ha tudunk slot-címkét, `slotA vs slotB → győztes`).
   - Ismeretlen team UUID → `?`.

4. **Backend unit tesztek** (`packages/backend/tests/tournament-tips.service.test.ts`):
   - "UX-038: builds a human-readable correctAnswerLabel for all_groups_standing"
   - "UX-038: includes a 'Továbbjutó 3. helyezettek' line for non-empty best3rds"
   - "UX-038: omits the best3rds line when best3rds is empty"
   - "UX-038: renders '?' for unknown team UUIDs in correctAnswer"
   - "UX-038: returns null label when correctAnswer JSON cannot be parsed"
   - "UX-038: builds a label for bracket_progression winners"

### Frontend

1. **`TournamentTipsView.vue`:**
   - A "Helyes válasz" sor whitespace-pre-line megjelenítést kap, így a backend `\n` elválasztók sortörést eredményeznek.
   - A `formatRawCorrectAnswer()` változatlan (BC – továbbra is a single-element JSON tömb fallbackét végzi).

2. **Komponens teszt** (`TournamentTipsView.test.ts`-be új teszt vagy meglévőhöz hozzáfűzve):
   - Bemenet: tipp a `correctAnswerLabel: 'A: X, Y, Z\nB: …'` mezővel.
   - Elvárás: a render eredmény tartalmazza mindkét sort és nincs benne nyers `{` karakter.

### Mit ne csináljunk

- Ne adjunk hozzá új DB oszlopot vagy mezőt; a `correctAnswer` továbbra is a meglévő `TEXT` mező.
- Ne nyúljunk a `evaluateSpecialPrediction()` pontozási logikájába.
- Ne építsünk dedikált frontend renderer komponenst (pl. read-only `GroupStandingsPicker`) — szöveges megjelenítés elég.
- Ne nyúljunk az admin admin scoring evaluation oldalához (`AdminTournamentEvaluationView.vue`).
