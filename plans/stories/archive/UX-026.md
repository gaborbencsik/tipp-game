# UX-026: Admin oldalak responsive layout

## Probléma — UX expert review (2026-05-25)

Kilenc admin route él (`/admin/stats`, `/admin/matches`, `/admin/teams`, `/admin/players`, `/admin/users`, `/admin/scoring`, `/admin/waitlist`, `/admin/global-types`, `/admin/sync`). Mindegyik view külön-külön **bemásolja ugyanazt a 9 elemű router-link sort** a template tetejére — vagyis a navigáció kilenc helyen párhuzamosan duplikált. A duplikációból több hiba is fakad:

### 1. Nav layout — egyedül a `/admin/sync` reszponzív

- `AdminSyncView.vue:6` használja a helyes `flex flex-wrap gap-2 mb-6` osztályt → mobilon szépen több sorra tördelődik.
- A többi nyolc view (`AdminStatsView`, `AdminMatchesView`, `AdminTeamsView`, `AdminPlayersView`, `AdminUsersView`, `AdminScoringView`, `AdminWaitlistView`, `AdminGlobalTypesView`) `flex gap-2 mb-6` osztályt használ, **`flex-wrap` nélkül** → mobilon (375px) a 9 tab egyetlen sorba próbál állni, vízszintesen overflow-ol vagy levágódik (a `AppLayout` belső padding miatt akár láthatatlan tabok is lehetnek).
- A `AdminMatchesView.vue:15` `flex gap-2` plusz a header `+ Új mérkőzés` gombja külön sorban → vegyes mintázat.

### 2. Címke-inkonzisztencia és typo-k

A 9-szer kopipasztolt nav között már most is **eltérnek a címkék** (futó hiba):

| View                  | "Mérkőzések" link  | "Felhasználók" link |
| --------------------- | ------------------ | ------------------- |
| AdminSyncView         | Mérkőzések         | Felhasználók        |
| AdminStatsView        | Mérkőzések         | Felhasználók        |
| AdminMatchesView      | Mérkőzések         | Felhasználók        |
| AdminTeamsView        | Mérkőzések         | Felhasználók        |
| AdminPlayersView      | Mérkőzések         | Felhasználók        |
| AdminUsersView        | Mérkőzések         | Felhasználók        |
| AdminScoringView      | Mérkőzések         | Felhasználók        |
| **AdminWaitlistView** | **Meccsek**        | **Felhasznalok** ← ékezet hiba |
| **AdminGlobalTypesView** | **Meccsek**     | Felhasználók        |

Egy közös komponens megszünteti ezeket az eltéréseket egyszer s mindenkorra.

### 3. Mobil főtartalom — UX expert javaslatok (nem kell minden 100%)

A user kérése szerint a **fő layout fix a navigáció**, a többire a UX expert ad javaslatot — nem szükséges minden admin oldalt mobilbarátra átírni, viszont a kritikusabb torzulásokat érdemes egy gyors pass-szal kezelni:

- **AdminUsersView** (`AdminUsersView.vue:26-91`) — 6 oszlopos `<table>` `overflow-x-auto`-val. Mobilon vízszintes scroll működik; külön mobil layout **nem szükséges**, de a header `+` toggle / role select érintőre apró → `min-h-[36px]`-os tap target ajánlott.
- **AdminMatchesView** (`AdminMatchesView.vue:1-13`) — header `+ Új mérkőzés` gomb mellett a cím; mobilon csúszhat. Javaslat: `flex-col md:flex-row` mintázat (mint UX-025-ben), gomb alá tördelődik. A meccslista táblázata mobilon érinthetetlen (sok oszlop) — **scope-on kívül**, csak overflow-x-auto wrapper biztosítása.
- **AdminScoringView** (`AdminScoringView.vue`) — kategóriánkénti pontérték űrlap, `<input type="number">` mezőkkel. Mobilon a label + input pár sorba törhet — `grid` átalakítás `grid-cols-1 sm:grid-cols-2` segíthet.
- **AdminSyncView** (`AdminSyncView.vue`) — már több section card-ot használ; a section header gombok (Run / Save / Toggle) néha egy sorba zsúfolva — javaslat `flex-wrap gap-2`.
- **AdminWaitlistView**, **AdminPlayersView**, **AdminTeamsView**, **AdminGlobalTypesView** — táblázat-alapúak. Mobilon `overflow-x-auto` elég, **kifejezetten nem kell** card-os mobil layoutot bevezetni.
- **AdminStatsView** — több kis dashboard kártya. Ezek többnyire már `grid-cols-1 md:grid-cols-2/3`-osak — gyors ellenőrzés, nem prioritás.

> **Egyértelmű scope határ:** a story célja, hogy admin felületre **érdemi belépni** mobilról (nav használható, header nem törött, modálok elérhetők). A bonyolult adminisztratív műveletek (meccs szerkesztés, nagy táblázatok) továbbra is elsődlegesen desktop felület — nem várjuk el a teljes mobil paritást.

---

## Story

Mint **adminisztrátor**, szeretnék **mobilon is el tudni navigálni az admin felületek között és látni a fontos információkat**, hogy **terepen vagy útközben gyors beavatkozást tehessek (pl. meccs eredmény rögzítés, user ban, sync trigger) anélkül, hogy desktopot kelljen kipakolni**.

## Elfogadási kritériumok

### A — Egységes `AdminNav` komponens (kötelező)

- [ ] Új komponens: `packages/frontend/src/components/admin/AdminNav.vue`, ami a 9 admin route linket tartalmazza, **egy helyen**
- [ ] A komponens layout megegyezik a jelenlegi `AdminSyncView` mintájával: `flex flex-wrap gap-2 mb-6`
- [ ] Mind a 9 link egységes címkét használ (forrás: jelenlegi `AdminSyncView`):
  - `Statisztikák`, `Mérkőzések`, `Csapatok`, `Játékosok`, `Felhasználók`, `Pontrendszer`, `Waitlist`, `Speciális tippek`, `Szinkron`
- [ ] A "Meccsek" és "Felhasznalok" elírások **megszűnnek** (a `AdminWaitlistView` és `AdminGlobalTypesView` régi inline nav-jai cserélve)
- [ ] Az aktív tab `exact-active-class="!bg-blue-600 !text-white"` styling marad, button stílus marad (`bg-gray-200 text-gray-700 hover:bg-gray-300`)
- [ ] Mind a 9 admin view (`Admin*View.vue`) beágyazza az `<AdminNav />` komponenst — a 9-szer duplikált inline router-link blokk **eltűnik**
- [ ] A komponens **nem fogad propsot** — a router maga jelöli ki az aktív tab-ot az `exact-active-class`-szal
- [ ] Mobilon (375px) a nav max **2-3 sorra tördelődik**, nem fut túl horizontálisan, nem clippel

### B — Header layout fix (mobilon olvasható oldalcím + akciógomb)

- [ ] `AdminMatchesView` header (`<h1>` + `+ Új mérkőzés` gomb) mobilon `flex-col md:flex-row`-ra átalakítva (UX-025 mintára), gomb alá tördel
- [ ] `AdminSyncView` belső section header-ek (Polymarket / Player sync / Transfermarkt / Raw stats / Run all) gombjai `flex-wrap gap-2` használnak, hogy mobilon ne csússzanak ki
- [ ] Az `<h1 class="text-2xl">` cím minden admin oldalon olvasható marad mobilon (nem törik 3 sorra)

### C — UX expert javaslatok (rögzítés, nem implementálás kötelező)

A főtartalom javaslatai (nem kötelező implementálni, csak dokumentálva legyen a follow-up-okhoz). **Ami megvalósul ebben a story-ban:**

- [ ] Minden admin view, ami `<table>`-t használ, rendelkezik `overflow-x-auto` wrapper-rel (gyors ellenőrzés; a legtöbb már így van)
- [ ] Az érintőképernyős cél (tap target) **min. 36px magas** a nav linkeknél (`py-1.5` vagy `py-2` mobilon → desktopon `md:py-1`)

**Ami nem ebben a story-ban valósul meg, de UX javaslatként rögzítve a Technikai jegyzetekben** (lásd lent):

- AdminUsersView mobil card layout (a 6-oszlopos tábla helyett)
- AdminMatchesView meccs lista mobil card layout
- AdminScoringView grid átrendezés
- Modálok (`ScoringOverrideModal`) mobil viselkedése — külön audit kell

### D — Tesztelés

- [ ] Új komponens teszt: `AdminNav.test.ts` — renderel 9 router-link-et, mindegyik a megfelelő `to` célra mutat, az osztálylista tartalmazza a `flex-wrap`-et
- [ ] Egyik admin view teszt frissítve (pl. `AdminUsersView.test.ts`): a `<AdminNav />` komponens jelen van a renderelt DOM-ban (lásd: a 9 `/admin/...` link mind megtalálható)
- [ ] Vizuális ellenőrzés viewport-okon: 320px, 375px, 768px, 1024px+
- [ ] Manuális regresszió: minden admin route megnyitva mobilon (375px), a nav használható (kattintható, nem clippelt)

### Implementációs vázlat

**`AdminNav.vue`:**
```vue
<template>
  <nav class="flex flex-wrap gap-2 mb-6" aria-label="Admin navigáció">
    <router-link
      v-for="link in links"
      :key="link.to"
      :to="link.to"
      class="px-3 py-1.5 md:py-1 text-sm rounded bg-gray-200 text-gray-700 hover:bg-gray-300 transition-colors"
      exact-active-class="!bg-blue-600 !text-white"
    >
      {{ link.label }}
    </router-link>
  </nav>
</template>

<script setup lang="ts">
const links: ReadonlyArray<{ readonly to: string; readonly label: string }> = [
  { to: '/admin/stats', label: 'Statisztikák' },
  { to: '/admin/matches', label: 'Mérkőzések' },
  { to: '/admin/teams', label: 'Csapatok' },
  { to: '/admin/players', label: 'Játékosok' },
  { to: '/admin/users', label: 'Felhasználók' },
  { to: '/admin/scoring', label: 'Pontrendszer' },

  { to: '/admin/global-types', label: 'Speciális tippek' },
  { to: '/admin/sync', label: 'Szinkron' },

</script>
```
