# PWA-001: PWA alapok – telepíthető app + offline shell

## Vezérelv

**Minimum invazív PWA bevezetés.** A cél, hogy az alkalmazás Lighthouse szerint PWA-nak minősüljön, mobilon telepíthető legyen a kezdőképernyőre, és offline alap shell betöltődjön. **Semmi sem törhet el** a meglévő működésből (auth flow, SSE, API hívások, Supabase callback). A push notification service worker-t a [[PUSH-001]] saját maga építi – ez a story azzal **nem ütközhet**.

## Leírás

Mint **felhasználó** szeretném, hogy **a Tipp game-et telepíthessem a telefonom kezdőképernyőjére natív app-szerű ikonnal és élménnyel**, hogy **gyorsabban elérjem és offline állapotban se egy fehér képernyőt lássak**.

## Jelenlegi helyzet

- Az `index.html` egy minimális Vue boilerplate; nincs `manifest.webmanifest`, sem PWA ikon, sem `theme-color`, sem `apple-touch-icon`.
- Service worker nincs; offline használat nem támogatott.
- Lighthouse PWA audit jelenleg bukna.
- A `vite-plugin-pwa` még nincs telepítve.

## Hatókör (scope)

### Benne van

1. **Web App Manifest** + ikonok + iOS-specifikus meta-k (Fázis 1)
2. **Vite PWA plugin** integrálása `injectManifest` módban (custom SW-vel) – offline app shell precache
3. **Minimális service worker** ami **csak statikus asset-eket** cache-el; az API-t és az SSE-t **érintetlenül hagyja**
4. **Offline fallback page** (`/offline.html`) – ha a navigation fetch elbukik, ezt mutatja
5. **Lighthouse PWA pass** Chrome desktop + mobile audit-on
6. **PUSH-001 előkészítés:** a SW fájl olyan szerkezetű, hogy a PUSH-001 csak push handler-eket adjon hozzá hozzá, ne kelljen újraírni

### Nincs benne (külön story)

- Push notification logika ([[PUSH-001]])
- Background sync, periodic sync
- Speciális route-cache (pl. leaderboard offline)
- Asset versioning a meglévő `logos/` és `venues/` képekre (azok már statikus path-en élnek, nem kell)

## Elfogadási kritériumok

### 1. Manifest és ikonok

- `packages/frontend/public/manifest.webmanifest` létrejön a következő tartalommal:
  - `name: "VB Tippjáték"`
  - `short_name: "Tipp game"` (max 12 karakter ✓)
  - `description: "Nyári labdarúgó VB tippjáték"`
  - `lang: "hu"`, `dir: "ltr"`
  - `start_url: "/"`, `scope: "/"`, `display: "standalone"`, `orientation: "portrait"`
  - `theme_color: "#3b82f6"` (Tailwind `blue-500` – világoskék, illeszkedik a domináns `blue-600`-hoz)
  - `background_color: "#ffffff"` (splash háttér)
  - `icons[]`: minimum **5 méret** – 192×192, 512×512 (mindkettő `purpose: "any"`), 192×192 és 512×512 `purpose: "maskable"`, plus 180×180 apple-touch
  - `categories: ["sports", "games"]`
- Ikonok forrásai a `packages/frontend/public/icons/` mappába kerülnek (PNG, lossless)
- **Ikon design:** egyszerű "TG" monogram vagy stilizált focilabda + "TG" világoskék (#3b82f6) háttéren, fehér foreground; maskable verzióban legalább **20% safe zone** padding minden oldalon
- A forrás ikon (1024×1024) `packages/frontend/public/icons/icon-source.png` néven kerül a repo-ba (verziózott), hogy újragenerálható legyen

### 2. index.html metaadatok

- `<link rel="manifest" href="/manifest.webmanifest">`
- `<meta name="theme-color" content="#3b82f6">`
- `<link rel="apple-touch-icon" sizes="180x180" href="/icons/apple-touch-icon-180.png">`
- `<meta name="apple-mobile-web-app-capable" content="yes">`
- `<meta name="apple-mobile-web-app-status-bar-style" content="default">`
- `<meta name="apple-mobile-web-app-title" content="Tipp game">`
- A `lang="hu"` attribútum a `<html>` tag-en már jó, marad

### 3. vite-plugin-pwa konfiguráció

- `npm i -D vite-plugin-pwa` a `packages/frontend`-be
- `vite.config.ts` kiegészítése:
  - `VitePWA({ strategies: 'injectManifest', srcDir: 'src/sw', filename: 'sw.ts', registerType: 'autoUpdate', injectRegister: false, devOptions: { enabled: false } })`
  - **`injectRegister: false`** – a regisztrációt mi kontrolláljuk explicit a `main.ts`-ben (lásd 4. pont)
  - **`devOptions.enabled: false`** – fejlesztés alatt SW kikapcsolva, hogy a Vite HMR működjön
- A manifest **NEM** a plugin által generált, hanem a kézi `public/manifest.webmanifest` (egyszerűbb, statikusan szerkeszthető)

### 4. Service Worker (`packages/frontend/src/sw/sw.ts`)

- Workbox `precacheAndRoute(self.__WB_MANIFEST)` az app shell precache-hez (build-időben generált asset-lista)
- `setDefaultHandler(new NetworkOnly())` minden nem-ismert kérésre
- **Navigation fallback:** `registerRoute(new NavigationRoute(handler, { denylist: [/^\/api\//, /^\/auth\//, /\/sse/, /\/events/] }))` – csak az SPA HTML route-okra vonatkozik, az API-t és SSE-t nem érinti
- **Offline fallback:** ha a navigation handler elbukik (offline + nincs cache), a `/offline.html` precache-elt válasszal felel
- `self.skipWaiting()` az `install` event-ben
- `self.clients.claim()` az `activate` event-ben
- **Üres `push` és `notificationclick` handler stub** kommenttel: `// PUSH-001 ide illeszti be a webpush logikát` – így a PUSH-001 csak két függvény tartalmát írja meg

### 5. Service Worker regisztráció

- `packages/frontend/src/main.ts` végén:
  ```ts
  if ('serviceWorker' in navigator && import.meta.env.PROD) {
    navigator.serviceWorker.register('/sw.js').catch((err) => {
      console.error('SW registration failed', err)
    })
  }
  ```
- **Csak PROD build-ben aktív** (`import.meta.env.PROD`) – fejlesztésben semmi nem változik

### 6. Offline fallback oldal

- `packages/frontend/public/offline.html` – statikus, minimális, ugyanaz a brand színe és `Inter` betűtípus mint az app, üzenet: "Nincs internetkapcsolat – próbáld újra később"
- A SW precache tartalmazza, hogy offline-on is elérhető legyen

### 7. Vercel hosting kompatibilitás

- A `vercel.json`-t nem kell módosítani – a manifest, sw.js, és offline.html statikus fájlokként szolgálódnak ki
- **Cache-Control:** a `sw.js`-re `Cache-Control: no-cache` szükséges, hogy a böngésző mindig friss SW-t töltsön (verziófrissítés). Ha a `vercel.json` jelenleg minden statikus fájlra hosszú cache-t ad, **headers szabály hozzáadandó** a `sw.js`-re és `manifest.webmanifest`-re.

### 8. Lighthouse / böngésző validáció

- Chrome DevTools → Application → Manifest: minden mező zöld, ikonok renderelődnek
- Chrome DevTools → Application → Service Workers: a SW aktív (csak prod buildben tesztelhető)
- Lighthouse PWA audit (mobile, prod build): **"Installable"** és **"PWA Optimized"** kategóriák zöldek
- Mobil Chrome-on (Android) "Hozzáadás a kezdőképernyőhöz" prompt megjelenik
- iOS Safari: a "Add to Home Screen"-nel telepítve a megfelelő ikon és app név látszik

### 9. Tesztek

- **E2E (Playwright):** új test `e2e/specs/pwa.spec.ts`:
  - Manifest endpoint (`/manifest.webmanifest`) elérhető és valid JSON
  - Az `index.html` tartalmazza a `<link rel="manifest">` és `theme-color` meta-t
  - Az `apple-touch-icon` link létezik és betölthető (200 status)
  - Az `/offline.html` elérhető
- **Service worker E2E:** prod build-en futtatva check, hogy a `/sw.js` 200-as választ ad és tartalmazza a `precache` szót (Workbox marker)
- Manuális regression checklist:
  - Bejelentkezés Google OAuth-tal működik (auth callback nem cache-elt)
  - Realtime SSE (leaderboard / match updates) működik
  - API hívások (tipp leadás, leaderboard fetch) friss adatot adnak, nem cache-eltet

## Kockázatok és mitigáció (TÖRÉSI VESZÉLY!)

| # | Kockázat | Hatás | Mitigáció |
|---|---|---|---|
| 1 | **API válaszok cache-elése** | User elavult tippet/leaderboardot lát | A SW `denylist`-jén a `/api/*` szerepel; csak app shell precache van, runtime API cache **nincs** |
| 2 | **SSE kapcsolat megszakad** | Realtime frissítések nem működnek | `denylist`-en `/events`, `/sse` is; manuális regression test PR előtt |
| 3 | **Auth callback cache-elés** | Login redirect loop | `denylist`-en `/auth/*` szerepel |
| 4 | **Régi SW "ragad" deploy után** | User a régi UI-t látja | `registerType: 'autoUpdate'` + `skipWaiting` + `clientsClaim` + `Cache-Control: no-cache` a `sw.js`-en |
| 5 | **Dev mode törés** | Vite HMR nem működik / "ragadt" build | `devOptions.enabled: false` – SW csak prod-ban regisztrál |
| 6 | **PUSH-001 SW ütközés** | Két SW regisztráció, push nem működik | A SW fájl `injectManifest` módban van, push handler stub-ja már most ott; PUSH-001 csak handler tartalmat ír bele, **nem új SW-t** |
| 7 | **iOS PWA kvóta + manifest** | iOS több mezőt nem támogat | Apple-specific meta-k duplikálva az `index.html`-ben (manifest mellett); manuálisan tesztelve iPhone Safari-n |
| 8 | **Vercel deploy gyorsítótár** | Régi `sw.js` szolgálódik ki | `vercel.json` headers szabály a `sw.js` és `manifest.webmanifest` no-cache-elésére |
