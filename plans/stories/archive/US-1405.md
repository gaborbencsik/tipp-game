# US-1405: Tippjeim oldal redesign - Személyes statisztikai dashboard

**Prioritás:** Should Have  
**Függőség:** --  
**Becsült méret:** L (frontend-heavy, minimal backend)

---

## Kontextus

A jelenlegi "Tippjeim" oldal (~`MyTipsView.vue`) ~95%-ban redundáns a "Meccsek" oldallal: mindkettő kronologikus meccslista tippelési lehetőséggel. A felhasználó számára nincs egyedi érték.

Az átalakítás célja: személyes statisztikai dashboard, ami megválaszolja a "Hogy állok?" kérdést.

---

## User Story

**Mint** bejelentkezett felhasználó,  
**szeretném** látni a tippjeim összesített statisztikáit (pontok, pontosság, sorozatok) és szűrhető listáját,  
**hogy** átfogó képet kapjak a teljesítményemről anélkül, hogy az összes meccset végig kellene görgetnem.

---

## Elfogadási kritériumok

### AC-1: KPI kártyák megjelenítése
- [ ] Az oldal tetején 2x2 grid-ben (mobile) / 4 egymás mellett (desktop) KPI kártyák:
  - **Leadott tippek:** `{submitted} / {total_available}` (ahol total = finished + live + scheduled meccsek száma)
  - **Összpontszám:** globális pontszám összeg (sum of pointsGlobal where not null)
  - **Pontosság:** `{correct_count} / {evaluated_count}` százalékban (correct = pointsGlobal > 0, evaluated = pointsGlobal is not null)
  - **Legjobb sorozat:** leghosszabb egymás utáni helyes tipp (pointsGlobal > 0, meccs scheduledAt sorrend alapján)
- [ ] A kártyák vizuálisan elkülönülnek (háttér, ikon, érték kiemelés)
- [ ] Ha nincs kiértékelt tipp: "Nincs még kiértékelt tipp" placeholder

### AC-2: Aktuális sorozat (current streak)
- [ ] A KPI szekcióban megjelenik az aktuális sorozat (hány egymás utáni helyes/helytelen tipp a legutóbbitól visszafelé)
- [ ] Zöld szín ha pozitív sorozat (helyes tippek), piros ha negatív (0 pontos tippek)
- [ ] Sorozat számításnál csak kiértékelt (finished) meccsek számítanak, scheduledAt sorrendben

### AC-3: Szűrhető tipp-lista
- [ ] A KPI kártyák alatt szűrősáv:
  - `Mind` (default) - összes leadott tipp, kiértékeltek előre
  - `Helyes` - pointsGlobal > 0
  - `Pontos` - pointsGlobal === exactScore (max pont a scoring configból)
  - `0 pont` - pointsGlobal === 0
  - `Kihagyott` - olyan finished meccsek ahol nincs prediction
- [ ] Szűrők segmented control / chip-ek formában, scrollable mobilon
- [ ] A lista elemei kompakt kártyák: csapatok, eredmény, tipp, kapott pont, dátum
- [ ] A szűrt lista eredményszám megjelenik a szűrő mellett badge-ként

### AC-4: Tipp lista kártya design
- [ ] Minden kártya tartalmazza:
  - Csapat nevek (TeamBadge komponenssel)
  - Végeredmény (ha van)
  - Saját tipp
  - Kapott pont (színkódolt: zöld = max, kék = részleges, szürke = 0, piros = kihagyott)
  - Meccs dátuma (kompakt formátum)
- [ ] A kártyák rendezése: legutóbbi kiértékelt meccs elől (desc by scheduledAt)
- [ ] Kattintásra navigáció a meccs részletekhez (`/app/matches/:id`)

### AC-5: Pont-eloszlás vizualizáció (MVP: egyszerű)
- [ ] A KPI kártyák és a lista között egy kompakt pont-eloszlás sáv:
  - Horizontális stacked bar: pontos találat | helyes győztes+diff | helyes győztes | helyes döntetlen | 0 pont
  - Színkódolt szegmensek tooltip-pel (hover/tap: "Pontos: 5 db")
  - Csak CSS/Tailwind, nincs chart library dependency
- [ ] Ha nincs kiértékelt tipp: elrejtve

### AC-6: Tippelés továbbra is elérhető
- [ ] A "Meccsek" oldalon továbbra is lehet tippelni (az ottani inline edit marad)
- [ ] A Tippjeim oldalon a scheduled meccsekre vonatkozó "Tippelj!" link/gomb átirányít a Meccsek oldalra (vagy a meccs detail-re)
- [ ] A Tippjeim oldalról ELTŰNIK az inline tipp-szerkesztés (ez most a Meccsek oldal felelőssége)

### AC-7: Responsive design
- [ ] Mobile (< 768px): KPI 2x2 grid, szűrők horizontálisan scrollable, lista full-width
- [ ] Desktop (>= 768px): KPI 4 oszlop, szűrők egy sorban, lista max-width korláttal
- [ ] Skeleton loading state a KPI kártyákhoz és listához

### AC-8: Empty states
- [ ] Ha egyetlen tipp sincs: motiváló üzenet + CTA gomb a Meccsek oldalra
- [ ] Ha a szűrő üres eredményt ad: "Nincs ilyen tipp" + szűrő reset lehetőség

## Technikai terv

### Frontend (packages/frontend)

**Új / módosított fájlok:**
- `src/views/MyTipsView.vue` - teljes újraírás (dashboard layout)
- `src/components/dashboard/KpiCard.vue` - újrafelhasználható KPI kártya
- `src/components/dashboard/PointDistributionBar.vue` - stacked bar vizualizáció
- `src/components/dashboard/PredictionListItem.vue` - kompakt tipp kártya
- `src/composables/useMyStats.ts` - statisztika számítás composable

**Data flow (nincs új backend endpoint az MVP-ben):**
```typescript
// useMyStats.ts - computed from existing store data
interface MyStats {
  submittedCount: number
  totalAvailable: number
  totalPoints: number
  evaluatedCount: number
  correctCount: number
  accuracyPercent: number
  bestStreak: number
  currentStreak: number  // positive = correct streak, negative = incorrect
  distribution: {
    exact: number
    winnerAndDiff: number
    winner: number
    draw: number
    incorrect: number
    missed: number
  }
}
```

**Streak calculation logic:**
```typescript
// Sort evaluated predictions by match.scheduledAt ASC
// Walk through: if pointsGlobal > 0 → increment current streak
// Track longest streak encountered
// Current streak: walk backwards from most recent evaluated
```

**Scoring config szükséges a frontend-en:**
- Az `exactScore` érték kell ahhoz, hogy a "Pontos" szűrő működjön
- Lehetőségek:
  - (A) Hardcode 3 (global default) - egyszerű, de nem flexibilis
  - (B) Új lightweight endpoint: `GET /api/scoring-config/default` → `{ exactScore: 3, ... }`
  - (C) A meglévő predictions válaszban benne van a `pointsGlobal` - a max érték kiszámítható az adatból
- **Javaslat:** (C) - `Math.max(...evaluatedPredictions.map(p => p.pointsGlobal))` a "Pontos" filterhez, nincs új endpoint

**Filter types:**
```typescript
type PredictionFilter = 'all' | 'correct' | 'exact' | 'zero' | 'missed'
```

**Matches szükséges a "Kihagyott" szűrőhöz:**
- A matchesStore-ból: finished meccsek ahol nincs prediction
- Ez már elérhető client-side (matchesStore.matches + predictionsStore.predictions)

### Backend (packages/backend)

**MVP-ben NEM kell új endpoint.** Minden adat elérhető:
- `GET /api/users/:userId/predictions` - az összes prediction pointsGlobal-lal
- `GET /api/matches` - az összes meccs (status, scheduledAt)

**Post-MVP lehetőség (US-1405B):**
- `GET /api/users/:userId/stats` - szerver-oldali aggregáció (ha a client-side számítás lassú lesz 100+ meccsnél)
- Trend data endpoint (napi/heti pontok timeline-hoz)

## Tesztelési terv

### Unit tesztek (Vitest)
- `useMyStats.ts` composable: streak számítás, distribution, accuracy edge cases
  - 0 prediction → all zeros
  - all correct → 100% accuracy, bestStreak = evaluatedCount
  - mixed results → correct streak calc
  - missed matches detection
- `PointDistributionBar.vue`: correct segment widths, tooltip content
- Filter logic: each filter returns correct subset

### E2E (Playwright)
- Dashboard betöltés: KPI kártyák megjelennek értékekkel
- Szűrő kattintás: lista frissül
- Empty state megjelenik ha nincs tipp
- Mobile responsive: 2x2 grid check
