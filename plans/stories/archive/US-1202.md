# US-1202: Futball API szinkronizációs service

> Epic: E12 – Automatikus adatszinkron

**Story:**
Mint **fejlesztő**, szeretnék **a kiválasztott futball API-ból meccs- és csapatadatokat lekérdezni és a saját adatbázisba szinkronizálni**, hogy **az admin ne kelljen manuálisan vigye fel az összes mérkőzést és eredményt**.

**Elfogadási kritériumok:**
- [x] `packages/backend/src/services/football-api.service.ts` – typed HTTP wrapper a külső API fölé; minden közvetlen API hívás itt van, domain logika nélkül
- [x] `packages/backend/src/services/sync.service.ts` – szinkronizációs logika: fixtures upsert, results upsert, teams upsert; csak a saját DB-t írja
- [x] Fixtures upsert: mérkőzések betöltése a `matches` táblába (`ON CONFLICT DO UPDATE`)
- [x] Results upsert: befejezett meccsek eredménye → `match_results` tábla + `matches.status = 'finished'`
- [x] Teams upsert: csapatok (név, shortCode, flagUrl, group) → `teams` tábla
- [x] Minden szinkron futás írja az `audit_logs` táblát (`action: 'create'`, `entityType: 'sync_run'`)
- [x] Hibakezelés: API timeout, rate limit (429) → exponential backoff (max 3 retry); részleges siker logolva, nem dob el mindent
- [x] Admin sync pause: `PUT /api/admin/sync/settings` endpoint — sync mód runtime váltása (off/final_only/adaptive/full_live)
- [x] `GET /api/admin/sync/settings` — aktuális sync mód
- [x] Admin manual trigger: `POST /api/admin/sync/run` — azonnal elindít egy sync futást, response-ban a futás eredménye (hány rekord szinkronizálva)
- [x] Unit tesztek: `sync.service.test.ts`, `football-api.service.test.ts`, `sync-config.test.ts` – 35 teszt összesen
- [x] `.env.example` frissítve: `FOOTBALL_API_KEY`, `FOOTBALL_API_BASE_URL`, league ID-k, `FOOTBALL_SYNC_MODE`
- [x] Admin UI: `/admin/sync` oldal — sync mód választó + manuális trigger gomb + eredmény megjelenítés
- [x] Migráció 0017: `external_id` INTEGER UNIQUE oszlopok (teams, matches, players) + `match_results.recorded_by` nullable

**Technikai megjegyzések:**
- `football-api.service.ts` csak typed HTTP wrapper – domain logika tilos benne
- `sync.service.ts` nem hívható közvetlenül HTTP-n – csak a cron job (US-1203) és az admin trigger hívja
- Native `fetch` (Node 24) – nincs axios dependency
- Lefedett ligák: FIFA World Cup 2026 (league_id=1), Magyar NB I (league_id=271)
- Sync mód in-memory (env-ből inicializálva) — nem DB-ben tárolva
- A `FOOTBALL_INTERNAL_WC_LEAGUE_ID` és `FOOTBALL_INTERNAL_NBI_LEAGUE_ID` env-ek szükségesek a belső league UUID-k összerendeléséhez
