# US-1306: Match Pulse – insight felfedése felhasználónként és admin reveal statisztika

## Leírás

Mint **bejelentkezett felhasználó**, szeretnék **egy "Elemzés felfedése" gombbal tudatosan felfedni az elmosódott Match Pulse szekciót**, hogy **az elemzés megtekintése az én aktív döntésem legyen, és következő látogatásra is megmaradjon**.

## Elfogadási kritériumok

### Adatbázis

- [ ] Létezik egy `insight_reveals` tábla az alábbi sémával:
  - `id` UUID PK
  - `user_id` UUID NOT NULL (FK → `users.id`)
  - `match_id` UUID NOT NULL (FK → `matches.id`)
  - `revealed_at` TIMESTAMPTZ NOT NULL DEFAULT NOW()
  - UNIQUE constraint: `(user_id, match_id)`
- [ ] RLS engedélyezve a táblán:
  - INSERT: csak a saját `user_id`-jére írhat a hitelesített felhasználó
  - SELECT: mindenki csak a saját sorait láthatja; admin role minden sort lát
- [ ] A tábla Drizzle sémában és migrációban is szerepel

### Backend — reveal endpoint

- [ ] Új endpoint: `POST /api/matches/:matchId/insights/reveal` — autentikáció kötelező
- [ ] Az endpoint idempotens: ha már létezik sor `(user_id, match_id)`-re, `200 OK`-ot ad vissza, nem hibát
- [ ] Sikeres vagy meglévő reveal esetén response: `{ revealed: true }`
- [ ] Ha a `matchId` nem létezik, `404 Not Found` választ ad
- [ ] Az endpoint rate-limited: max 10 reveal / perc / felhasználó

### Backend — insight lekérés módosítása

- [ ] A meglévő `GET /api/matches/:matchId/insight` endpoint kiegészül egy `revealed: boolean` mezővel a válaszban
- [ ] A `revealed` értéke `true`, ha az `insight_reveals` táblában létezik sor az adott `(userId, matchId)` párhoz, egyébként `false`
- [ ] A lekérdezés egy JOIN-nal, plusz DB-hívás nélkül oldandó meg

### Frontend — blur + felfedés flow

- [ ] A `?showInsights=true` query string logika teljes mértékben eltávolítva a kódbázisból
- [ ] A Match Pulse szekció alapértelmezetten elmosódva (CSS `blur`) jelenik meg, ha `revealed === false`
- [ ] Az elmosódott szekció felett megjelenik egy "Elemzés felfedése" feliratú gomb
- [ ] A gomb megnyomásakor: a frontend meghívja a `POST .../insights/reveal` endpointot, sikeres válasz után a blur animáltan eltűnik (CSS transition), a gomb eltűnik
- [ ] Ha az API hívás sikertelen (5xx, hálózati hiba), a gomb aktív marad, rövid hibaüzenet jelenik meg a szekció alatt ("Nem sikerült felfedni, próbáld újra"), automatikusan eltűnik 4 másodperc után
- [ ] Ha `revealed === true` (az oldalbetöltéskor a backend már `revealed: true`-t ad vissza), a szekció azonnal blur nélkül, gomb nélkül jelenik meg — nincs animáció
- [ ] A felfedett állapot a Pinia insights store-ban frissítve van (`insights[matchId].revealed = true`), az oldalfrissítés a DB-ből tölti vissza

### Admin statisztika

- [ ] Az admin statisztika oldalon (`AdminDashboardView`) megjelenik egy "Match Pulse felfedések" szekció
- [ ] A szekció táblázatban mutatja: meccs neve (hazai – vendég), meccs dátuma, felfedések száma (hány egyedi user fedett fel)
- [ ] A lista meccs dátum szerint csökkenően rendezett, max 20 meccs jelenik meg egyszerre
- [ ] Az adatot kiszolgáló endpoint: `GET /api/admin/insights/reveal-stats` — admin middleware mögött
- [ ] Az endpoint response sémája: `{ matches: Array<{ matchId, homeTeam, awayTeam, matchDate, revealCount: number }> }`

### Tesztek

- [ ] Unit teszt a reveal service-re: idempotens insert, nem létező `matchId` 404, sikeres eset
- [ ] Unit teszt a `GET .../insight` endpointra: `revealed: true` ha létezik sor, `revealed: false` ha nem
- [ ] Vitest teszt a frontend `useInsight` composable-re: `revealed: false` → gomb megjelenik; reveal API hívás után `revealed: true` → blur eltűnik; API hiba eset
- [ ] Vitest teszt: `revealed: true` esetén nincs gomb és nincs blur az inicializáláskor

## Technikai megjegyzések

- A `POST .../reveal` endpoint `ON CONFLICT DO NOTHING` upsert-tel dolgozzon Drizzle-ben
- A blur + gomb overlay pozícionálása: `relative` wrapper a szekción, `absolute` overlay a blur felett — a szekció tartalma ne ugorjon el felfedéskor
- CSS transition: `filter: blur(8px)` → `filter: blur(0)` 300ms ease-in-out
- Az admin stat query aggregáljon: `SELECT match_id, COUNT(DISTINCT user_id) FROM insight_reveals GROUP BY match_id`
- Ne tárold a `revealed` állapotot localStorage-ban — kizárólag DB az igazságforrás

## Nem tartalmazza

- Push értesítés reveal eseménykor
- Reveal visszavonása (egyirányú akció)
- Insight megosztás gomb
- Per-user reveal analitika az adminnak (csak összesített darabszám)
