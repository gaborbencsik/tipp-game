# US-1304: Match Pulse – insight megjelenítése a meccs kártyán

## Leírás

Mint **bejelentkezett felhasználó**, szeretnék **minden mérkőzésnél egy rám determinisztikusan kiosztott statisztikai insight-ot látni**, hogy **meccs előtt adatalapú háttérinformációt kapjak, és természetes csoportbeszélgetések indulhassanak el**.

## Elfogadási kritériumok

### Backend — API endpoint

- [ ] Új endpoint: `GET /api/matches/:matchId/insight` — autentikáció kötelező
- [ ] Az endpoint a következő logikával adja vissza a megfelelő insight-ot:
  - Lekéri az összes insight-ot a `match_insights` táblából a `matchId`-hoz
  - Ha 0 insight létezik: `204 No Content` választ ad (insight még nem generált)
  - Ha legalább 1 insight létezik: `insightIndex = fnv1a(userId + matchId) % availableCount` alapján kiválaszt egyet
  - Visszaadja: `{ id, type, title, body, dataPoints, generatedAt }`
- [ ] A hash függvény (`fnv1a`) tiszta, mellékhatás-mentes utility függvény, unit tesztelve
- [ ] Ha `availableCount < 5` (generálás folyamatban vagy sikertelen volt), a hash a ténylegesen elérhető darabszámmal számol — nem hibát dob
- [ ] Unit tesztek:
  - Azonos `userId + matchId` mindig ugyanazt az insight indexet adja
  - Különböző `userId` értékek különböző indexeket adnak ugyanarra a `matchId`-re (legalább 2 különböző userId-vel ellenőrzött)
  - 0 insight esetén `204` visszatér

### Frontend — megjelenítés

- [ ] A meccs részletes nézetén (match detail / meccs kártya kibontott állapotában) megjelenik egy "Match Pulse" szekció
- [ ] A szekció csak akkor jelenik meg, ha a backend `200`-at ad vissza (204 esetén a szekció rejtett, nincs hibaüzenet)
- [ ] A megjelenített tartalom: kategória badge (pl. "Védekezés"), cím (`title`), törzs (`body`)
- [ ] A `dataPoints` mező NEM jelenik meg a felhasználónak — csak belső adat
- [ ] A szekció betöltési állapota skeleton loaderrel jelzett (nem üres fehér terület)
- [ ] Ha a fetch sikertelen (hálózati hiba, 5xx), a szekció csendesen elrejtve marad — nincs hibaüzenet a felhasználónak
- [ ] Az API hívás a match detail megnyitásakor egyszer fut le, az eredmény a Pinia store-ban cachelve van a session idejére (`insights` store, kulcs: `matchId`)
- [ ] Vitest teszt a `useInsight(matchId)` composable-re: betöltési állapot, sikeres adat, 204 eset, hiba eset

## UX megjegyzések

- A kategória badge színe a `type` értéke alapján meghatározott (pl. `defense` = kék, `attack` = piros, `form` = zöld) — Tailwind utility osztályokkal, nem inline style-lal
- A szekció a meccs kártyán a tipp beviteli mező FELETT helyezkedik el
- Mobilon a `body` szöveg csonkítva jelenik meg 3 sorban, "Tovább" gombbal kibontható (ne legyen truncate JS nélkül)
- A "Match Pulse" felirat felett NEM szerepel magyarázat, hogy mások más insight-ot látnak — ez szándékos

## Nem tartalmazza

- Admin nézet az összes 5 insight megtekintéséhez
- Insight megosztás gomb
- In-tournament statisztikák (US-1305)
- Push értesítés az insight megjelenésekor
