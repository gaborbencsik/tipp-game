# US-1406: Transfermarkt keret-összérték szinkronizáció

## Leírás

Mint **adminisztrátor**, szeretném, hogy a platform automatikusan szinkronizálja a válogatott keretek összpiaci értékét a Transfermarkt API-ból, hogy a csapatokhoz naprakész, valós piaci értékadat álljon rendelkezésre a Match Pulse KPI-ok és a jövőbeli insight generálás számára.

## Elfogadási kritériumok

### Adatbázis

- [x] A `teams` tábla tartalmaz `transfermarkt_id` (text) és `squad_market_value` (bigint, EUR cent) oszlopot
- [x] A `sync_state` tábla tartalmaz `transfermarkt_sync_enabled` (boolean) és `last_transfermarkt_sync_at` (timestamptz) oszlopot
- [x] A migráció beseedelezi a `transfermarkt_id` értéket mind a 73 VB-válogatott csapathoz

### Backend — service

- [x] `parseMarketValue(raw: string): number` tiszta függvény konvertálja az API string értékét (pl. `"450000000"`) egész számmá
- [x] `syncTransfermarktValues()` lekéri az összes aktív csapat Transfermarkt adatát, és upsert-eli a `squad_market_value` mezőt
- [x] A service a `data.squadDetails.currentMarketValue.value` mezőt olvassa az API válaszból (nem top-level `squadMarketValue`)
- [x] Ha egy csapathoz nincs `transfermarkt_id`, a szinkron azt a csapatot kihagyja (nem dob hibát)
- [x] Ha az API-hívás 404-et vagy hálózati hibát ad, a service logol és folytatja a többi csapattal

### Backend — route-ok

- [x] `POST /api/admin/sync/transfermarkt-run` – admin manuálisan triggereli a szinkront (admin JWT szükséges)
- [x] `POST /api/internal/sync/transfermarkt-tick` – belső cron endpoint, 24 órás cooldown-nal fut
- [x] A cron tick csak akkor fut le, ha `transfermarkt_sync_enabled = true` a `sync_state`-ben
- [x] A cooldown lejárta után `last_transfermarkt_sync_at` frissül a szinkron végén

### Backend — sync state

- [x] `setTransfermarktSyncEnabled(enabled: boolean)` váltja az engedélyezési flag-et
- [x] `markTransfermarktSyncFinished()` frissíti a `last_transfermarkt_sync_at` timestampot

### Frontend — admin UI

- [x] Az admin sync oldalon megjelenik a Transfermarkt szekció:
  - Toggle a szinkronizáció engedélyezéséhez/tiltásához
  - "Szinkronizálás most" gomb manuális triggerhez
  - Utolsó szinkron időpontja olvasható formátumban
- [x] A toggle és a gomb disabled állapotba kerül, ha a kérés folyamatban van
- [x] Sikeres szinkron után az utolsó szinkron időpontja frissül az oldalon

### Tesztek

- [x] Legalább 15 unit teszt lefedi a `parseMarketValue` és `syncTransfermarktValues` logikát
- [x] `parseMarketValue` teszteli: normál szám, null/undefined input, nem szám string, negatív szám, nulla
- [x] `syncTransfermarktValues` mock-olt HTTP-vel teszteli: sikeres szinkron, kihagyott csapat (nincs transfermarkt_id), részleges API hiba (egyik csapat 404)

## Technikai megjegyzések

- **API endpoint:** `GET https://tmapi-alpha.transfermarkt.technology/club/{transfermarkt_id}`
- **Válasz struktúra:** `data.squadDetails.currentMarketValue.value` — **nem** top-level `squadMarketValue`; ez az API egy belső alpha változata, a struktúra változhat
- **Érték formátuma:** string-be csomagolt egész szám (EUR), pl. `"450000000"` — a `parseMarketValue` végzi az átalakítást
- **Cooldown logika:** a `last_transfermarkt_sync_at` alapján: ha a legutóbbi szinkron kevesebb mint 24 órája volt, a tick kihagyja a futást
- **Migrációs fájlok:** `0027_player_stats_and_sync.sql` (sync_state oszlopok), `0028_transfermarkt_market_value.sql` (teams oszlopok + seed)

## Nem tartalmazza

- Transfermarkt-on alapuló pontszámítás vagy scoring logika változtatás
- Automatikus Transfermarkt ID keresés / matching (az ID-k kézzel kerültek beseedelezésre)
- Piaci érték megjelenítése a felhasználói felületen (csak háttéradat a Match Pulse-hoz)
- Játékos-szintű piaci értékek szinkronizálása (csak keret-összérték)
