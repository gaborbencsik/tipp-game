# UX-022: Torna tippek kitöltöttsége az admin statisztikákban

> Epic: E6 – Admin & üzemeltetés

**Háttér:**
Az admin statisztika oldal (`/admin/stats`) jelenleg csak a meccs-tippek kitöltöttségét mutatja:

- **Summary kártya** – `fillRate` (összes meccstipp / (felhasználók × meccsek))
- **Felhasználó tábla** – `tipCount`, `fillPercent` (csak meccs-tippek)
- **Meccs tábla** – meccsenkénti `tippedCount` / `totalUsers`

A **torna tippek** (speciális tippek; `special_predictions` + `special_prediction_types` táblák, `is_global = true` típusok pl. „Torna gólkirálya”, csoport-specifikus típusok) jelenleg sehol nem jelennek meg az admin statisztikákban, így az adminnak nincs rálátása arra, hányan töltötték ki ezeket. A backend forrás: `packages/backend/src/services/admin-stats.service.ts`, a frontend: `packages/frontend/src/views/AdminStatsView.vue`.

**Story:**
Mint **admin**, szeretném **látni a torna tippek (speciális tippek) kitöltöttségét is az admin statisztikák oldalon**, hogy követhessem, hány felhasználó tippelt globális speciális típusokra.

## Elfogadási kritériumok

### Summary szekció – kétsoros elrendezés és átnevezés

- [ ] A summary kártyák **két csoportba** rendeződnek, vékony szövegcímkével elválasztva: **„Meccs-tippek”** és **„Torna tippek”**.
- [ ] Az 1. sor (meccs) változatlanul: regisztrált felhasználók, aktív (7 nap), összes meccs-tipp, **„Meccs-kitöltés”** (átnevezve a jelenlegi „Kitöltési arány”-ról).
- [ ] A 2. sor (torna) új kártyái: **globális speciális típusok darabszáma**, **összes leadott torna-tipp**, **„Torna-kitöltés” %** (kitöltött `special_predictions` / (aktív, nem-törölt felhasználók × globális, aktív speciális típusok)).
- [ ] Mindkét kitöltés-kártyához **tooltip / `?` ikon** ami kiírja a képletet, hogy ne keveredjen össze a két mutató.
- [ ] A „Torna-kitöltés” kártya is használja a **konzisztens 50%-os sárga (`bg-yellow-50`) figyelmeztető küszöböt**, ahogy a meccs-kártya.
- [ ] A torna-kártyák megkülönböztető háttérszínt (pl. `bg-indigo-50`) vagy bal oldali színes szegélyt kapnak a két csoport vizuális szétválasztására.

### Felhasználó-tábla – csoportosított header

- [ ] A meglévő „Tippek” és „Kitöltés” oszlopok **csoport-headerré bővülnek** (`<th colspan="2">`), két alá-fejléccel:
  - „Tippek” → **Meccs** | **Torna** (`tipCount` | `tournamentTipCount`)
  - „Kitöltés” → **Meccs %** | **Torna %** (`fillPercent` | `tournamentFillPercent`)
- [ ] A torna-tipp formátum: `kitöltött / összes globális típus` (pl. `7/10`), külön oszlopban a százalék.
- [ ] **Mobilon** a torna-oszlopok `hidden md:table-cell` osztállyal rejtve maradnak; a 7→9 oszlop nem fér ki kényelmesen 768 px alatt.
- [ ] Az új oszlopok **sortolhatók** (`tournamentTipCount`, `tournamentFillPercent`).
- [ ] A piros sor-háttér (`bg-red-50`) szabálya **frissül**: csak akkor pirosít, ha a felhasználó **SE meccs-tippet, SE torna-tippet** nem adott le. Ha bármelyikre tippelt, semleges háttér.

### Új tábla: „Torna tippek (globális speciális típusok)”

- [ ] Önálló H2 címmel, a meccs-tábla **alatt** elhelyezve, vízszintes elválasztóval.
- [ ] Halvány alcím: „Csak globális (`is_global = true`) típusok. Csoport-specifikus típusok statisztikája külön nézetben.”
- [ ] Oszlopok:
  1. **Típus neve**
  2. **Leadási határidő** (dátum)
  3. **Hátralévő idő** – emberbarát formátum (pl. „3 nap”, „2 óra”, „lejárt”)
  4. **Státusz badge** – `Aktív` (zöld), `Lejárt` (szürke), `Lezárt` (kék – ha `correctAnswer` ki van töltve)
  5. **Kitöltöttek / összes felhasználó** (pl. `42/68`)
  6. **Kitöltési arány %**
- [ ] **Sor-szintű színkódolás** kitöltési arány alapján (csak aktív típusoknál):
  - `< 30%` → `bg-red-50`
  - `30–60%` → `bg-yellow-50`
  - `> 60%` → semleges
  - Lejárt / lezárt típusok mindig semleges szürke (`bg-gray-50`).
- [ ] **Sortolható** határidő, kitöltési arány és státusz szerint.

### Edge case-ek és üres állapotok

- [ ] **Nincs egyetlen aktív globális speciális típus sem** → a torna-szekció explicit empty state-et mutat („Nincs aktív globális speciális típus”), nem tűnik el némán; a summary kártya „—”-t mutat tooltippel.
- [ ] **Lejárt deadline** → a kitöltési arány a deadline pillanatában **véglegesen lefagyasztva** marad, NEM esik vissza 0%-ra (különben félrevezető lenne). A státusz badge jelzi, hogy lezárult.
- [ ] **Lejárt típusok beleszámítanak a felhasználói nevezőbe** (`kitöltött / összes globális típus`), különben a százalék mesterségesen javulna idővel.
- [ ] Inaktív (`is_active = false`) és soft-deleted típusok mindkét számításból kizárva.
- [ ] Csoport-specifikus (`is_global = false`) típusok az MVP-ben **nem szerepelnek** — külön story.

### Backend és típusok

- [ ] Backend: `getAdminStats` és `getAdminStatsMatches` (vagy új `getAdminStatsSpecial`) bővül a torna-mutatókkal.
- [ ] Lekérdezés: `special_predictions` JOIN `special_prediction_types` `WHERE is_global = true AND is_active = true`, csoportosítva típusonként és felhasználónként.
- [ ] Új / frissített típusok mindkét rétegben: `AdminStatsSummary`, `AdminStatsUser`, új `AdminStatsSpecialType` (`packages/backend/src/types/index.ts` és `packages/frontend/src/types/index.ts`).
- [ ] A meglévő `fillRate` (meccs-tippek) értéke **változatlan**.

### Tesztelés és infrastruktúra

- [ ] **Unit teszt** a service-re: új mezők számítása, üres állapotok, törölt felhasználók kizárása, lejárt deadline kezelés (fagyasztott arány), inaktív típusok kizárása.
- [ ] **Komponens teszt** az `AdminStatsView.vue`-ra: új summary kártyák, csoportosított felhasználó-tábla header, új speciális típusok tábla render és sortolás, empty state.
- [ ] **`data-testid` attribútumok** minden új elemen, a meglévő stílusban: `tournament-fill-rate-card`, `tournament-types-table`, `tournament-type-row`, `user-tournament-tip-count`, `user-tournament-fill-percent`, `summary-tournament-row`.
- [ ] **Saját loading skeleton** a torna-tábla blokkjához, ne egyetlen globális spinner alatt töltsön mindent.

### MVP-ből kizárva (post-MVP, külön story)

- Drill-down: kattintható típus-sor / felhasználó-cella, ami modálban mutatja a leadott válaszokat.
- „Top válasz” oszlop és válaszok megoszlása (mini bar-chart) a típus-táblában.
- Csoport-specifikus speciális típusok admin nézete (külön epic, csoport-admin scope).
- Kombinált „összes kitöltés” mutató felhasználónként (meccs + torna súlyozva).

A sorok az MVP-ben **ne legyenek** kattinthatónak látszó affordance-szal (`cursor-pointer`, hover-szín), hogy ne legyen confused affordance.
