# US-957: Per-liga szinkronizáció ki/bekapcsolása az admin szinkron oldalról

> Epic: E-Admin – Adatszinkronizáció

## Háttér

Az admin szinkron oldal (`AdminSyncView.vue`) jelenleg a globális szinkron módot (off/final_only/adaptive/full_live) és több szinkronizációs source-t kezel (polymarket, játékosok, transfermarkt, stb.). A konfiguráció **per-liga szinkronizálhatóságáról** a backend már DB-szinten tudni szeretne: mely ligák szinkronizálódjanak az ütemezett futásban.

**Meglévő DB állapot (US-949, US-950 kész):**
- `leagues` tábla: `syncEnabled` (boolean, default false), `externalId` (int, unique), `season` (smallint), `status` ('active'|'archived'), `syncFrom`/`syncTo` (timestamptz), `fixtureAllowlist` (int[])
- `PUT /api/admin/leagues/:id` — `syncEnabled` módosítható
- `sync-runner.ts` hard rule: csak `syncEnabled=true AND status='active'` liga szinkronizálódik

**Jelenlegi UI hiánya:**
- `AdminSyncView.vue`: "Konfigurált ligák" szekció **csak READ-ONLY chip-ek** (37–44. sor) — nincs per-liga toggle
- Nincs feedback arról, hogy egy liga szinkronizálható-e vagy sem

**Viszony más story-khoz:**
- **US-951 (LEZÁRT)**: admin liga CRUD UI (create/edit oldal), ahol `syncEnabled`, `externalId`, `season`, stb. mezők BEÁLLÍTHATÓK — ez egy **másik felület** (AdminLeaguesView.vue vagy hasonló)
- **US-956 (nyitott)**: per-liga **on-demand** sync trigger (kézi futtatás egy liga adott típusáról: meccs+csapat+játékos) — ez a **kézi futtatásról** szól, nem az ütemezett futás engedélyezéséről
- **US-957 (ez a story)**: az **admin szinkron oldalon** a gyors per-liga `syncEnabled` toggle, amely azt vezérli, hogy az ütemezett futás mely ligákat szinkronizálja

## Story

Mint **admin**, szeretném **a szinkron oldalról közvetlenül ki/bekapcsolni az egyes ligák ütemezett szinkronizálását**, hogy **gyorsan kezeljem az aktív és nem-aktív szinkronizációs ligákat anélkül, hogy az admin liga-szerkesztő oldalt fel kellene nyitni**.

## Elfogadási kritériumok

### Backend

- [ ] **Meglévő endpoint felhasználása**
  - A backend `PUT /api/admin/leagues/:id` már támogatja a `syncEnabled` mezőt és validálást → **nincs új backend endpoint szükséges**
  - A validáció meglévő: `syncEnabled=true` esetén `externalId` kötelező (422 különben); `season` null kizáró (404 vagy warning, a spec szerint)
  - Az ütemezett sync-runner meglévő szabálya: csak `syncEnabled=true AND status='active'` liga fut

### Frontend — AdminSyncView.vue UI

- [ ] **Per-liga toggle a "Konfigurált ligák" szekció kártyáin**
  - Minden konfigurált ligánál (a `configuredLeagues` tömb elemeinél) egy **switch/toggle** jelenik meg
  - Alaphelyzet: a toggle **a liga aktuális `syncEnabled` állapotát** tükrözi
  - Toggle click → **optimista update** (azonnali UI frissítés) + `PUT /api/admin/leagues/:id` hívás `{ syncEnabled: boolean }` payload-dal
  - Sikeres válasz: UI marad az új állapotban
  - Hiba: UI **visszaáll az előző állapotra**, error message: "Hiba a szinkronizáció ki/bekapcsolásakor: [error detail]" (data-testid: `league-sync-error-{leagueId}`)

- [ ] **Állapot-jelzések a toggle mellett**
  - **"Szinkronizálható"** – a liga `syncEnabled=true` és aktív (toggle bekapcsolható, zöld/aktív stílus)
  - **"Hiányzó externalId"** – nem létezik `externalId` a ligához → toggle **LETILTVA** (disabled), szürke, tooltip: "Ez a liga nem szinkronizálható: hiányzik az externalId"
  - **"Hiányzó season"** – `season` null → **WARNING ikon** a toggle mellett, tooltip: "A season nincs beállítva; a liga az ütemezett futásból kimarad, amíg be nem állítod az admin liga-szerkesztőben"
  - **"Archivált"** – `status='archived'` → toggle **LETILTVA**, szürke, tooltip: "Archivált ligák nem szinkronizálhatók"

- [ ] **Loading state**
  - Toggle click során: **spinner/loading ikon** a toggle mellett, toggle átmenetileg disabled
  - Sikeres válasz után: spinner eltűnik
  - Hiba után: spinner eltűnik, error message jelenik meg 3 másodpercig

- [ ] **Data-testid-k**
  - `configured-leagues-section` – a szekció container-je
  - `league-sync-toggle-{leagueId}` – a toggle input
  - `league-sync-status-{leagueId}` – az állapot-jelzés (szöveg + ikon)
  - `league-sync-loading-{leagueId}` – a loading spinner
  - `league-sync-error-{leagueId}` – az error message

### Frontend — API wrapper

- [ ] `packages/frontend/src/api/index.ts` — `api.admin.leagues`
  - Meglévő: `update(token: string, id: string, input: Partial<LeagueInput>)` — ez már van, csak a frontend komponens hívja meg
  - **NEM szükséges új függvény**; a meglévő `update` metódus használható `{ syncEnabled: true/false }` payloaddal

### Frontend — data flow

- [ ] `AdminSyncView.vue` komponens-logika
  - `loadSettings()` (meglévő) → `configuredLeagues` betöltése: név, externalId, season
  - **NEW:** `toggleLeagueSyncEnabled(league)` — az optimista update + API hívás
  - A `configuredLeagues` tömb kiterjesztése: minden ligához `syncEnabled`, `status` mező (HA szükséges; alternatíva: a teljes `League` objektum a backend-ről)
  - Az API már visszaadja ezt az `updateSettings`-ben? **Nem** — az csak a globális sync mode-ot adja vissza (lásd api/index.ts:558). Alternatíva: **külön GET `/api/admin/leagues?includeArchived=true`** hívás a részletes liga adatokért a toggle működéséhez

- [ ] **Adatfrissítés stratégia**
  - Az `loadSettings()` már hozza a `configuredLeagues`-t
  - Szükséges-e további adat (syncEnabled, status per liga)? **Igen**, mert az aktuális `configuredLeagues` csak `{ name, externalId, season }` (lásd AdminSyncView.vue:389)
  - **Opció A:** Az `updateSettings` response kiterjesztése, hogy visszaadja a frissített ligákat → backend módosítás (scope?)
  - **Opció B:** Toggle után egy külön `GET /api/admin/leagues?includeArchived=true` hívás a részletes adatokért (kliensoldali, egyszerűbb)
  - **Opció C:** A toggle sikeres PUT után a local `configuredLeagues`-ben frissíteni az adott liga `syncEnabled` mezőjét (optimista; a GET-et csak szükség szerinti refresh-nél)

  **Választott megoldás (Opció C + fallback):**
  - Toggle → optimista UI update + PUT hívás
  - PUT sikeres → local `configuredLeagues`-ben update a `syncEnabled` flag-et
  - PUT hibás → UI rollback, error message
  - (Opcionális) beépített 5 másodperces auto-refresh a liga teljes adatainak igazolásához (hiányzó mező esetén)

### Data flow a komponensben

```ts
// 1. Beim Laden
async loadSettings() {
  const data = await api.admin.sync.getSettings(token)
  configuredLeagues.value = data.configuredLeagues
  // configuredLeagues: { name, externalId, season }
  // syncEnabled, status: kell-e GET /admin/leagues?
}

// 2. Toggle kattintás
async toggleLeagueSyncEnabled(league) {
  const oldValue = league.syncEnabled
  league.syncEnabled = !league.syncEnabled  // optimista
  try {
    await api.admin.leagues.update(token, league.id, { syncEnabled: league.syncEnabled })
    // Sikeres
  } catch (err) {
    league.syncEnabled = oldValue  // rollback
    // error message show 3 sec
  }
}

// 3. UI render
v-for="league in configuredLeagues"
  :disabled="!league.externalId || league.status === 'archived'"
  :value="league.syncEnabled"
  @change="toggleLeagueSyncEnabled(league)"
```

### Backend validáció (már existe)

```ts
// leagues.service.ts validateSyncFields()
if (input.syncEnabled === true && !input.externalId) {
  throw new AppError(422, 'externalId is required to enable sync')
}
```

### Kérdések, amelyeket el kell dönteni a kódolás előtt

1. **Teljes liga data a `configuredLeagues`-ben?**
   - Az `updateSettings` response csak `{ name, externalId, season }`-t ad vissza. Kell-e `syncEnabled`, `status` is?
   - **Ajánlás:** `loadSettings()` után (vagy toggle után) végezz egy `GET /api/admin/leagues?includeArchived=true` hívást a teljes adatokért, és merge-eld a `configuredLeagues`-gel.

2. **Perfor­mance:** Minden toggle kattintás után GET? Vagy csak lokális update?
   - **Ajánlás:** Lokális update az PUT sikerre, majd opcionális 5 másodperces delay után auto-GET a megerősítéshez (ha az adatokat valaki manuálisan módosította közben).
