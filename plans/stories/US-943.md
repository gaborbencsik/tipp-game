# US-943: `GroupDetailView`: `?tab=special` query param kezelés mount-on

> Forrás: special-tippek-banner-stories.md

### Cím

`GroupDetailView`: `?tab=special` query param kezelés mount-on

### User Story

Mint csoporttag, szeretném, hogy ha a bannerre kattintok és a rendszer a `/app/groups/{id}?tab=special` URL-re navigál, a csoport oldal automatikusan a "Speciális tippek" tabra nyíljon meg, hogy ne kelljen manuálisan rákattintanom a tabbra.

### Elfogadási kritériumok

1. A `GroupDetailView.vue` `<script setup>` blokkjában az `onMounted` hook lefutásakor a `useRoute().query.tab` értékét ellenőrzi.
2. Ha `route.query.tab === 'special'`, a `switchToSpecialTab()` függvényt hívja meg (a tab beállítását és a speciális tippek betöltését egyszerre elvégzi).
3. Ha `route.query.tab` más értékű vagy hiányzik, az `activeTab` értéke változatlan marad (`'leaderboard'` az alapértelmezett).
4. A `switchToSpecialTab()` funkcionalitása nem változik: beállítja `activeTab.value = 'special'`, és ha nincs még betöltve (`!groupsStore.specialPredictionsMap[groupId]`), meghívja a `fetchSpecialPredictions(groupId)` store actiont.
5. A `useRoute` importálása a `vue-router`-ből — ha még nem importált, add hozzá.
6. Az URL-ben maradó `?tab=special` query param nem okoz UI problémát (az `activeTab` ref-re nem hat vissza a query param változása navigáció nélkül — nincs `watch` a route-ra).
7. Unit teszt (`GroupDetailView.spec.ts` bővítése): ha a route `query.tab === 'special'` értékkel van mockúlva, a `switchToSpecialTab` meghívódik mount-kor; ha a `query.tab` hiányzik, nem hívódik meg.

### Edge case-ek

| Helyzet | Elvárt viselkedés |
|---------|-------------------|
| `?tab=leaderboard` query param | Nincs hatás — a leaderboard tab az alapértelmezett, nem kell explicit kezelés |
| `?tab=settings` query param | Nincs hatás — a tab query csak a `special` értéket kezeli |
| `?tab=special` query param, de a user nem tagja a csoportnak | Az `assertGroupMember` a backend oldalon 403-at ad, a `fetchSpecialPredictions` hibával tér vissza — a meglévő `specialPredictionsError` state kezeli |
| A user direktben tippeli be az URL-t (`/app/groups/xxx?tab=special`) | Működik — az `onMounted` kezeli |
| Navigáció a bannertől: `router.push({ path: ..., query: { tab: 'special' } })` | A GroupDetailView frissen mount-olódik (ha nem volt megnyitva), a query param olvasódik |

### Technikai megjegyzések

- Érintett fájl: `packages/frontend/src/views/GroupDetailView.vue`
- Az `onMounted` hook végén (miután az alap adatok betöltöttek) add hozzá:
  ```ts
  if (route.query.tab === 'special') {
    await switchToSpecialTab()
  }
  ```
- A `route` a `useRoute()` composable-ból jön: `const route = useRoute()`.
- A `useRoute` és `useRouter` valószínűleg már importált a fájlban (ellenőrizd); ha nem, a `vue-router`-ből importálandó.
- A `switchToSpecialTab()` már `async` — az `onMounted` már `async`, az `await` hozzáadása biztonságos.
- Tesztfájl: `packages/frontend/src/views/GroupDetailView.spec.ts`
- A teszt a `buildTestRouter` helper-t használja (a meglévő spec fájlok mintájára), a route-ot `{ query: { tab: 'special' } }` értékkel mock-olva.

### Hatókörön kívül

- Más query param értékek kezelése (`?tab=leaderboard`, `?tab=members`, stb.).
- Deep link a speciális tippek egy konkrét típusára.
- A query param törlése navigáció után (az URL-ben maradhat).
