# US-1002: Felhasználói felület lokalizációja (i18n)

## Leírás

Mint **felhasználó**, szeretnék **az alkalmazást a saját nyelvemen használni (magyar vagy angol)**, hogy **akkor is kényelmes legyen a platform, ha nem vagyok magyar anyanyelvű**.

## Elfogadási kritériumok

- [ ] Minden UI string ki van emelve a komponensekből egy `src/locales/hu.json` fájlba
- [ ] A `vue-i18n` v9 (Composition API mód) keretrendszer beállítva, a komponensek `$t()` / `useI18n()` hívásokkal használják a szövegeket
- [ ] Létezik `src/locales/en.json` az angol fordításokkal (a magyar fájl alapján)
- [ ] A login/regisztráció oldalon jobb felső sarokban `HU | EN` toggle látható (opcionálisan 🌐 ikonnal)
- [ ] A toggle váltáskor a login oldal azonnal az adott nyelven jelenik meg (újratöltés nélkül)
- [ ] A toggle állapota localStorage-ba mentődik — visszatérésnél is megmarad a bejelentkezés előtti nyelvválasztás
- [ ] Regisztrációkor a frontend az aktuálisan kiválasztott locale-t elküldi a backendnek → ez lesz a `preferred_locale` értéke a `users` táblában
- [ ] Bejelentkezéskor a backend visszaadja a `preferred_locale` értékét → a frontend erre vált
- [ ] A Profil oldalon egy dropdown-ból kiválasztható a nyelv (Magyar / English)
- [ ] A kiválasztott nyelv a `users` táblában tárolódik (`preferred_locale` mező, default: `hu`)
- [ ] A nyelv váltás azonnali, oldal-újratöltés nélkül
- [ ] Login után a backend által küldött `preferred_locale` értéke alapján töltődik be a megfelelő nyelv
- [ ] Meccsadatok (csapatnevek, eredmények) NEM fordítódnak — csak UI chrome szövegek

## UX specifikáció — Nyelvválasztó a login oldalon

### Elhelyezés

Jobb felső sarok (`fixed top-4 right-4` vagy `absolute` a wrapper-en belül).

```
┌──────────────────────────────────────────┐
│                                 🌐 HU|EN │
│                                          │
│               ⚽                          │
│          VB Tippjáték                    │
│                                          │
│     "Tippeld meg a VB meccseit           │
│      a barátaiddal!"                     │
│                                          │
│      ┌──────────────────────┐            │
│      │ 🔵 Sign in with Google│            │
│      └──────────────────────┘            │
│                                          │
└──────────────────────────────────────────┘
```

### Megjelenés és viselkedés

- Formátum: két szöveges link pipe-pal elválasztva, aktív nyelv félkövér (`HU | EN`)
- Stílus: `text-sm text-gray-500 hover:text-gray-800`, aktív: `font-semibold text-gray-900`
- Mobil: legalább 44x44px touch target (padding-gel biztosítva)
- Átváltás: azonnali, nincs animáció/reload
- NEM dropdown (2 nyelvnél felesleges) — mindkét opció mindig látszik
- NEM flag ikon (zászló = ország, nem nyelv)

### Viselkedési flow

```
User érkezik → HU az alapértelmezett (hacsak localStorage-ban nincs más)
  ├─ Magyar user: figyelmen kívül hagyja → belép → kész
  └─ Nemzetközi user: észreveszi → EN-re kattint
       → oldal azonnal angolra vált
       → localStorage menti "en"-t
       → Google-lel belép
       → backend létrehozza accountot preferred_locale = "en"-nel
       → következő sessionök angolul töltődnek
```

## Technikai megjegyzések

- `vue-i18n` v9+ runtime-only build (`vue-i18n/dist/vue-i18n.runtime.esm-bundler.js` alias a vite.config-ban)
- Pinia store NEM kell — `vue-i18n` saját reaktivitása elegendő, egy `useLocale()` composable kezeli
- JSON struktúra: 1 szint mélységű nested (pl. `nav.home`, `matches.title`, `common.save`)
- `$d()` és `$n()` formázók a dátum/szám lokalizációhoz
- Magyar pluralizáció egyszerű (számmal mindig egyes szám: "3 meccs")
- Backend: `users` tábla + `preferred_locale` mező (migration szükséges)
- Regisztrációs flow: a frontend a `POST /api/auth/me` hívásban küldi a `locale` mezőt — nincs szükség Supabase OAuth state paraméter hackelésre

## Nem tartalmazza

- Backend API hibaüzenetek fordítása (azok angolul maradnak)
- Meccsadatok / csapatnevek fordítása
- Browser locale auto-detektálás (a login toggle megoldja)
- Naptár widget / date picker lokalizáció

## Komplexitás

L (a string tokenizálás mechanikus de sok fájlt érint)
