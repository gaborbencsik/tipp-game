# QA Guide — VB Tippjáték (2026-04-24)

> Tesztelési útmutató az április 10 óta elkészült funkciókhoz.
> Előfeltétel: `docker compose up -d` fut, elérhető a `http://localhost:5173`

---

## 1. Landing page és auth flow

- [ ] `http://localhost:5173/` → landing page jelenik meg (nem az app)
- [ ] "Bejelentkezés" gomb → `/login` oldalra navigál
- [ ] Dev bypass: bejelentkezés után `/app/matches`-re kerül (nem a landing page-re)
- [ ] Kijelentkezés → vissza a landing page-re

## 2. Onboarding overlay (UX-008)

- [ ] Első bejelentkezés után megjelenik a 3 lépéses onboarding overlay
- [ ] Lépések: Üdvözlés → Határidő → Csoportok, "Tovább" gombbal léptetés
- [ ] "Kihagyás" bármely lépésben bezárja az overlay-t
- [ ] Bezárás után NEM jelenik meg újra (page refresh után sem)
- [ ] UserMenuButton → "Bemutató újranézése" → overlay újra megjelenik
- [ ] Escape billentyű bezárja az overlay-t

## 3. Meccsek nézet

- [ ] `/app/matches` — meccsek listája megjelenik napok szerint csoportosítva
- [ ] Lejátszott meccsek szekció a lista **tetején**, alapból **összecsukva**
- [ ] Kattintásra kinyitható/összecsukható
- [ ] Csapatlobogók megjelennek a csapatnevek mellett
- [ ] Tipp leadás: meccsre kattintás → eredmény beírás → "Mentés" → tipp megjelenik

## 4. Csoportok

### 4.1. Csoport létrehozás és csatlakozás
- [ ] `/app/groups` → "Csoport létrehozása" → név megadás → csoport létrejön
- [ ] Meghívó kóddal csatlakozás: "Csatlakozás" → kód beírás → tag lesz
- [ ] Meghívó link formátum: `http://localhost:5173/app/join/XXXXX`

### 4.2. Csoport részletek — Ranglista tab
- [ ] `/app/groups/:id` → Ranglista tab aktív
- [ ] Táblázat oszlopok: #, Játékos, Tipp, Helyes, **Stat**, Pont
- [ ] Saját sor kékkel kiemelve + "(te)" jelölés
- [ ] Avatar megjelenik (dicebear fallback)

### 4.3. Csoport részletek — Stat tippek tab
- [ ] "Stat tippek" tab megjelenik minden tagnak (a Ranglista után)
- [ ] Ha nincs stat tipp típus: "Ebben a csoportban még nincsenek stat tippek."
- [ ] Ha van: kártyák típusonként (név, leírás, max pont, határidő)
- [ ] Határidő előtt: input mező + "Leadás" gomb → "Mentve!" visszajelzés
- [ ] Meglévő tipp módosítás: "Módosít" gomb + "Jelenlegi: ..." szöveg
- [ ] Határidő után: "A határidő lejárt, kiértékelésre vár."
- [ ] Kiértékelés után: pontszám megjelenítés (zöld ha >0, szürke ha 0)

### 4.4. Csoport részletek — Tagok tab (admin)
- [ ] Csak admin felhasználónak jelenik meg
- [ ] Tagok listája: név, szerep badge (Admin/Tag), csatlakozás dátuma
- [ ] Saját sor gombjai disabled
- [ ] Más tag: "Admin" / "Admin visszavon" + "Eltávolít" gombok
- [ ] Eltávolítás: megerősítő dialógus → tag eltűnik a listából

### 4.5. Csoport részletek — Meghívó kód (admin)
- [ ] Tagok tab alatt: meghívó kód szekció
- [ ] Kód megjelenítés + "Kód" másolás gomb + "Link másolása" gomb
- [ ] Állapot: Aktív/Inaktív + toggle gomb
- [ ] "Újragenerálás" → megerősítő dialógus → új kód generálódik

### 4.6. Csoport részletek — Beállítások tab (admin)
- [ ] Csak admin/global admin felhasználónak jelenik meg
- [ ] **Egyedi pontrendszer szekció:**
  - [ ] 6 mezős form (Pontos találat, Helyes győztes + gólkülönbség, stb.)
  - [ ] "Mentés" → "Elmentve!" visszajelzés
- [ ] **Stat tipp típusok szekció:**
  - [ ] "Még nincs stat tipp típus." ha üres
  - [ ] "+ Új stat tipp típus" gomb → form megjelenik:
    - [ ] Név (kötelező), Leírás (opcionális), Típus (szöveg/legördülő), Pont, Határidő
    - [ ] "Legördülő" típusnál megjelenik az "Opciók" mező
    - [ ] "Létrehozás" → típus megjelenik a listában
  - [ ] Létező típus kártyán: inputType, pont, határidő, opciók (ha van)
  - [ ] "Szerkeszt" gomb → form előtöltve a meglévő adatokkal
  - [ ] "Törlés" gomb → megerősítő dialógus → típus eltűnik
  - [ ] "Kiértékel" gomb (ha nincs még helyes válasz) → helyes válasz megadása dialógus → pontok kiszámolva
  - [ ] Kiértékelés után "Helyes válasz: ..." megjelenik a kártyán

### 4.7. Csoport törlése
- [ ] Tagok tab alján "Csoport törlése" gomb (admin only)
- [ ] Megerősítő dialógus → törlés → `/app/groups`-ra navigál

## 5. Admin felület

### 5.1. Pontrendszer
- [ ] `/admin/scoring` → globális pontértékek szerkesztése
- [ ] Mentés → értékek frissülnek

### 5.2. Waitlist
- [ ] `/admin/waitlist` → feliratkozott email-ek listája
- [ ] Szűrés forrás alapján (hero/footer/admin)
- [ ] Keresés email-re
- [ ] "Törlés" gomb → sor eltűnik
- [ ] "Hozzáadás" → email megadás → új sor

### 5.3. Admin menü
- [ ] Sidebar: összecsukható admin szekció
- [ ] Pill tab navigáció: Mérkőzések / Csapatok / Felhasználók / Pontrendszer / Waitlist

## 6. Email waitlist (landing page)

- [ ] Landing page-en email feliratkozás mező (hero + footer)
- [ ] Email megadás → "Feliratkozva!" visszajelzés
- [ ] Duplikált email → hibaüzenet
- [ ] Honeypot mező: ha kitöltve → csendben elutasítva (anti-spam)

## 7. Biztonság (SEC-001)

- [ ] Supabase RLS bekapcsolva — közvetlen DB hozzáférés (Supabase dashboard) nem mutat adatot auth nélkül

## Ismert korlátok

- A Docker frontend konténer nem mindig frissíti a fájlokat HMR-rel → `docker compose restart frontend` szükséges lehet változtatások után
- Dev auth bypass: a mock user mindig admin — "sima tag" teszteléshez a csoport tagkezelésben kell admin jogot visszavonni
- A stat tipp pontok egyelőre csak a **csoport** ranglistában jelennek meg, a globális ranglistában nem (US-907 még nem implementált)
